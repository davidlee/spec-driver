## Scope

Conformance audit for DE-077: two CLI UX fixes.

1. **ISSUE-023**: Create commands should show schema inspection commands after creation
2. **ISSUE-043**: `--from-backlog` greedy token consumption fixed at root cause

## Findings

### F-001: Schema hints implemented via shared helper

<!-- disposition: aligned -->

`supekku/cli/hints.py` provides `ARTIFACT_SCHEMA_MAP` (named constant, POL-002) and `format_schema_hints()` (pure function). `print_schema_hints()` wired into 10 create commands: delta, plan, phase, revision, audit, spec, policy, standard, memory, and spec (prod variant). Backlog entries (issue/problem/improvement/risk), card, and drift intentionally excluded — no meaningful schemas.

**Outcome**: Matches DR-077 DEC-077-01. POL-001 (reuse) satisfied — single helper, no duplication.

### F-002: --from-backlog converted to boolean flag

<!-- disposition: aligned -->

`--from-backlog` changed from `str | None` with `callback=_validate_backlog_id` to `bool` flag. Backlog ID passed as positional `name` argument. `_validate_backlog_id` callback removed; inline validation in command body.

**Outcome**: Matches DR-077 DEC-077-02. Root cause (Click greedy parsing) eliminated. `create delta --from-backlog --help` now shows help.

### F-003: Existing create workflows unchanged

<!-- disposition: aligned -->

All 29 CLI tests pass. No regressions in backlog creation tests. Full suite (3619 tests) green.

### F-004: Skill guidance updated

<!-- disposition: aligned -->

`supekku/skills/spec-driver/SKILL.md` updated with `--from-backlog` boolean flag note and schema hints tip. Installed via `spec-driver install`. No stale `--from-backlog` references found in skills or memories.

### F-005: ADR not created for --from-backlog does not have frontmatter.adr schema

<!-- disposition: aligned -->

ADR create command intentionally has no schema hints — no `frontmatter.adr` schema exists in the registry. This is correct; ADR frontmatter uses the base schema only.
