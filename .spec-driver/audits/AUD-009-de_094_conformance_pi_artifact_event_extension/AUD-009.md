## Scope
- `supekku/pi.extensions/spec-driver-artifact-events.ts` — extension implementation
- `supekku/pi.extensions/spec-driver-artifact-events.test.ts` — 31 tests
- `supekku/scripts/install.py` — test file exclusion in `_collect_pi_sources()`
- `supekku/claude.hooks/artifact_event.py` — SYNC comment added

## Evidence
- 31 TS tests pass: classification (13 artifact types + 2 ordering + 7 non-artifact), event schema (5), JSONL write (2), socket delegation (2)
- 138 Python tests pass (103 install + 35 artifact_event)
- Lint clean (ruff)
- Install pipeline correctly includes extension, excludes test file

## Findings

### F-001: Pattern parity with Python
- **Outcome**: All 12 artifact patterns match identically between TS and Python
- **Disposition**: `aligned`

### F-002: Event schema v1 conformance
- **Outcome**: All v1 fields present (v, ts, session, cmd, argv, artifacts, exit_code, status, artifact_type). Session is null (pi doesn't expose session IDs).
- **Disposition**: `aligned`

### F-003: JSONL emission
- **Outcome**: Events appended to `.spec-driver/run/events.jsonl` with recursive dir creation. Same path as Python hook.
- **Disposition**: `aligned`

### F-004: Socket emission
- **Outcome**: Unix datagram sent via `pi.exec("python3", ...)` — async, fire-and-forget, fail-silent. Workaround for Node.js dgram lacking AF_UNIX.
- **Disposition**: `aligned`

### F-005: Fail-silent behaviour
- **Outcome**: Entire `tool_result` handler wrapped in try/catch per DEC-094-02. Socket errors caught independently.
- **Disposition**: `aligned`

### F-006: Install pipeline
- **Outcome**: `_collect_pi_sources()` filters `.test.` files. Extension installed to `.pi/extensions/` via existing DE-093 pipeline.
- **Disposition**: `aligned`

### F-007: Sync coupling documented
- **Outcome**: `mem.artifact-pattern-sync` memory + SYNC comments in both files.
- **Disposition**: `aligned`

### F-008: DR-094 socket design deviation
- **Outcome**: DR-094 proposed `dgram.createSocket("unix")` + `node:dgram`. Implementation uses `pi.exec("python3", ...)` instead because Node dgram lacks AF_UNIX.
- **Disposition**: `spec_patch` — DR-094 updated below.

## Reconciliation
- DR-094 socket section updated to reflect actual implementation approach.
