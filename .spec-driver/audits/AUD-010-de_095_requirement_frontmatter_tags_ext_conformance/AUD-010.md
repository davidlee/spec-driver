# AUD-010 – DE-095 Requirement Frontmatter Conformance

## Scope

Verify that DE-095 correctly wires `tags`, `ext_id`, `ext_url` through the requirement creation and sync pipelines per DR-095.

## Observations

### F-001: Creation emits frontmatter fields
- **Outcome**: aligned
- **Disposition**: aligned
- `create_requirement_breakout()` accepts `tags`, `ext_id`, `ext_url` and emits them in frontmatter when non-empty. Omits them when empty/None.
- Evidence: `test_create_requirement_breakout_with_tags_ext`, `test_create_requirement_breakout_without_tags_ext` — both pass.

### F-002: CLI surfaces expose new params
- **Outcome**: aligned
- **Disposition**: aligned
- Typer CLI: `--tag` (repeatable), `--ext-id`, `--ext-url`
- Argparse CLI: `--tags` (comma-separated), `--ext-id`, `--ext-url`
- Both pass through to `create_requirement_breakout()`.

### F-003: Sync reads breakout frontmatter
- **Outcome**: aligned
- **Disposition**: aligned
- New `_load_breakout_metadata()` scans `requirements/*.md` under spec dir.
- Applied in both `_records_from_frontmatter()` (spec_registry path) and fallback `spec_dirs` path.
- Evidence: 5 tests in `TestBreakoutFrontmatterSync` — all pass.

### F-004: Tag merge semantics
- **Outcome**: aligned
- **Disposition**: aligned
- Frontmatter tags ∪ inline `[tag]` tags → sorted set union.
- Consistent with `RequirementRecord.merge()` semantics.
- Evidence: `test_breakout_tags_merged_with_inline` — passes.

### F-005: No regressions
- **Outcome**: aligned
- **Disposition**: aligned
- 132 tests in touched modules pass (99 registry + 28 creation + 5 new).
- 47 CLI requirement tests pass.
- Pre-existing failure in `common_test.py::TestResolveArtifactBacklog` is unrelated (repo root detection in test environment).
