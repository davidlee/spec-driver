## Evidence

### Test results

- 62 frontmatter_writer tests (CompactDumper, quoting, list ops, backward-compat wrappers, prettier compat)
- 4044 full suite tests pass (4 skipped — pre-existing)
- Normalisation idempotency verified: second pass = 0 changes
- Prettier convergence verified: `prettier .spec-driver --check` = all files match

### Code coverage

- `frontmatter_writer.py`: CompactDumper, `_represent_str`, `_represent_list`, `increase_indent` override, `update_frontmatter`, `add/remove_frontmatter_list_items`, `update_frontmatter_status/fields`
- `edit.py`: `--tag`/`--untag` on all 17 subcommands via `_apply_tags` helper
- `scripts/normalise_frontmatter.py`: One-time migration script

### Key design decisions

- Double-quote strings that YAML would single-quote (prettier convention)
- Never use indentless sequences (prettier convention)
- Flow list width limit = 60 (leaves room for key prefix under prettier's 80-char width)
- `width=10000` prevents PyYAML mid-value wrapping

## Recommendations

- No spec patches required. PROD-010 (CLI UX) is the only referenced spec and the `--tag`/`--untag` flags are additive.
- IMPR-017 tracks client repo migration as future work.
