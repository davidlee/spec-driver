# AUD-017 — DE-107 Pydantic model migration conformance

## 1. Scope

Conformance audit for DE-107 (Pydantic model layer migration). Verifies that all in-scope @dataclass models were converted to Pydantic BaseModel per DR-107, with behavioural parity confirmed by existing tests.

## 2. Evidence

### Test results

- **671 tests passing** across all 7 converted modules (memory, backlog, cards, drift, diagnostics, sync, docs/python)
- Zero test failures
- `uv run ruff check` clean on all 7 model files

| Module      | Models converted                                      | Pattern                                                                   |
| ----------- | ----------------------------------------------------- | ------------------------------------------------------------------------- |
| memory      | MemoryRecord                                          | BaseModel + ConfigDict(extra="ignore") + field_validator for dates        |
| backlog     | BacklogItem                                           | BaseModel + ConfigDict(extra="ignore")                                    |
| cards       | Card                                                  | BaseModel + ConfigDict(extra="ignore") + from_file() classmethod retained |

## 3. Findings

### F-001: All in-scope models converted — aligned

<!-- disposition: aligned -->

16 models converted per DR-107 conversion recipe. `extra="ignore"` applied consistently. Frozen dataclasses correctly migrated to `ConfigDict(frozen=True)` or `BaseModel, frozen=True`.

### F-002: Spec model correctly deferred — follow_up_delta

<!-- disposition: follow_up_delta -->

Spec model is structurally different (wraps FrontmatterValidationResult, uses @property accessors). Not the from_frontmatter/to_dict boilerplate pattern. Deferred to DE-111 with explicit follows_from relation.

### F-003: Behavioural parity confirmed — aligned

<!-- disposition: aligned -->

671 tests passing. Test changes were minimal and mechanical:

- 7 tests updated in Phase 1 (from_frontmatter → direct construction)
- 0 tests updated in Phase 2
- ~24 tests updated in Phase 3 (positional→kwargs, AttributeError→ValidationError)

No behavioural changes detected.

### F-004: Three gotchas documented for future phases — aligned

<!-- disposition: aligned -->

Phase 3 discovered three Pydantic migration gotchas documented in notes.md:

1. TYPE_CHECKING guard on Path breaks Pydantic field resolution
2. Frozen models raise ValidationError not AttributeError
3. Positional construction not supported

These are captured in notes and will inform DE-111 work.

### F-005: IP-107 scope narrowed from original — follow_up_delta

<!-- disposition: follow_up_delta -->

Original IP-107 Phase 3 included frontmatter metadata updates and schema docs. These were deferred along with the Spec model to DE-111. The IP-107 scope was effectively narrowed to "convert all mechanical @dataclass models" which is complete.
