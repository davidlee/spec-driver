- **STD-002 (lint signal)**: pre-existing `revision → revision_metadata` cyclic-import warning retired in 4.1; no new pylint regressions on touched files.
- **STD-003 (utility placement)**: wrapper-alongside-metadata convention applied uniformly across 3 wrappers; OQ-HARNESS-PLACEMENT settled in P02 (harness stays in `blocks/metadata/` due to heavy import dependency).
- **STD-004 (script lifecycle)**: OQ-HARNESS-LIFECYCLE settled option (a) KEEP with documented owner + re-run trigger in the module docstring.
- **DR-136 §11.1**: validator stays current-schema only; no `migrations/` package imports.
- **DR-136 §11.3**: workflow.* unknown-key strict flip remains deferred to DE-137 per DEC-006; IMPR-035 is the load-bearing record.

- The DR-118 §2 "premise correction" (2026-05-09) — six of seven hand-rolled validators were lax on unknown keys, not strict — was caught by the harness exactly as DEC-007 intended. The P02.5 declaration-fidelity pass is the appropriate response: it widens the metadata declarations to match observed contract, preserving behaviour. Deliberate tightening is correctly owned by DE-137 per DR-136 §11.3.
- The P03 closure commits and P04 cleanup commits are independently revertible (R8 partial-stall policy honoured). HAND_ROLLED_ADAPTERS = {} confirms the unified state.
- Three new memory candidates surfaced during P03/P04 (`<source>_metadata_test.py` mirror rule, block-class taxonomy for `.data` cache vs `.parse()`, wrapper-signature divergence). Capture via `/capturing-memory` after close.
- One stale historical-analysis document (`YAML_BLOCKS_ANALYSIS.md`, dated 2025-11-01) references retired symbols. Discovery finding FIND-015; non-blocking backlog follow-up.
