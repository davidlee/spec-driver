## Observations

- **Migration complete**: 26 genuine core modules successfully moved from
  `supekku/scripts/lib/core/` to `spec_driver/core/` in 4 phases across 4
  dependency tiers. Module count: 27 non-test modules (26 migrated + 3 original;
  `slugify` merged into existing `string_utils.py`).
- **Layer contract truthful**: `import-linter` 3/3 contracts KEPT. Core is
  provably free of upward dependencies (core→orchestration, core→domain).
- **Decoupling successful**: `spec_utils` (P02) and `config` (P03) upward deps
  eliminated via DEC-128-004 (relocate) and DEC-128-005 (DI). Core now imports
  no orchestration or domain symbols.
- **Domain debt cleared**: Zero `supekku.scripts.lib.core` imports in
  `spec_driver/domain/`. All 3 legacy-core imports in `manager.py` and `graph.py`
  repointed to `spec_driver.core.*`.
- **Tests pass**: 370 core tests pass (excl. 6 test files with nix-env missing
  deps: yaml, jinja2, click, typer, tomlkit — pre-existing). 2 events_test.py
  template-dependency failures are pre-existing and unchanged.
- **Commit state**: P01+P02 committed (2 commits). P03+P04 uncommitted
  (~28 files modified + ~12 new).
- **Artefact drift**: DE-128.md and DR-128.md have pre-existing schema validation
  errors (invalid relation types, missing required fields) — cosmetic, no
  implementation impact. Phase-04.md has an inaccurate export name in a task
  template (`generate_agent_docs` vs actual `render_agent_docs`).
