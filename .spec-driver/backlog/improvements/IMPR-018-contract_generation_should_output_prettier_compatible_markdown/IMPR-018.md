# Contract generation should output prettier-compatible markdown

## Context

DE-096 made all `.spec-driver/**/*.md` frontmatter idempotent with prettier. However, Python contract files in `.contracts/` are generated by the sync/contract pipeline and currently require a manual `prettier --write` pass to conform.

## Problem

Generated `.contracts/**/*.md` files diverge from prettier formatting, creating noisy diffs and requiring manual formatting after every sync.

## Desired outcome

`spec-driver sync` should produce contract markdown that passes `prettier --check` without a separate formatting step. This likely means:

1. Using `CompactDumper` (or equivalent) for any frontmatter in contract files
2. Ensuring generated markdown body (tables, code blocks, lists) matches prettier conventions
