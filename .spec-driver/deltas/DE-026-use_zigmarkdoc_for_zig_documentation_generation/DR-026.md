# DR-026 – Use zigmarkdoc for Zig documentation generation

## 1. Executive Summary

- **Delta**: [DE-026](./DE-026.md)
- **Status**: draft
- **Owners / Team**: TBD
- **Last Updated**: 2026-02-16
- **Synopsis**: Replace manual Zig doc comment parsing with zigmarkdoc subprocess calls, following GoAdapter pattern

## 2. Problem & Constraints

- **Current Behaviour**: ZigAdapter uses manual regex-based parsing (`_parse_zig_file`, `_extract_declaration`) that:
  - Misses complex documentation patterns
  - Doesn't handle comptime, inline fn, or other Zig-specific constructs
  - Produces inconsistent output compared to standard Zig tooling
  - Duplicates effort that zigmarkdoc already solves
- **Drivers / Inputs**:
  - IMPR-004: Use zigmarkdoc for Zig documentation generation
  - GoAdapter provides proven pattern using gomarkdoc
  - zigmarkdoc available at ~/.local/bin/zigmarkdoc
- **Constraints / Guardrails**:
  - Must maintain backward compatibility with existing contract output locations
  - Must support `--check` mode for CI verification
  - Should handle gracefully when zigmarkdoc not available (either error or fallback)
- **Out of Scope**:
  - Changes to contract storage locations or frontmatter schema
  - Zig LSP integration improvements
  - Other language adapter changes

## 3. Architecture Intent

- **Target Outcomes**:
  - Generate accurate, comprehensive Zig documentation using official tooling
  - Match quality/consistency of GoAdapter using gomarkdoc
  - Support both public-only and all (including private) doc variants
- **Guiding Principles**:
  - Follow GoAdapter pattern for consistency across language adapters
  - Prefer external tools over manual parsing where available
  - Fail fast with clear error messages when dependencies missing
- **State Transitions / Lifecycle Impact**: None (contract generation only)

## 7. Design Decisions & Trade-offs

- **Decision**: Follow GoAdapter pattern exactly for consistency
  - **Rationale**: Proven pattern, easier maintenance, consistent behavior across adapters
  - **Trade-off**: Less flexibility for Zig-specific customization
- **Decision**: Generate both "public" and "all" variants (vs current single variant)
  - **Rationale**: Matches GoAdapter, provides complete API surface documentation
  - **Trade-off**: More disk space, longer generation time
- **Decision**: Raise ZigmarkdocNotAvailableError if zigmarkdoc unavailable (no fallback)
  - **Rationale**: Fail fast, clear feedback, prevents degraded behavior
  - **Trade-off**: Requires zigmarkdoc installation from https://github.com/davidlee/zigmarkdoc
- **Decision**: Update contract paths to contracts/interfaces.md and contracts/internals.md
  - **Rationale**: Match GoAdapter pattern for consistency
  - **Trade-off**: Breaking change from current contracts/zig/{slug}-public.md paths
