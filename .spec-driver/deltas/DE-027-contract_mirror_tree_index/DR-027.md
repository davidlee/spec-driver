# DR-027 – Contract mirror tree index

## 1. Executive Summary

- **Delta**: [DE-027](./DE-027.md)
- **Status**: draft
- **Last Updated**: 2026-02-20
- **Synopsis**: Add a `.contracts/` symlink tree at repo root that mirrors source paths to generated contract artefacts, enabling intuitive navigation and search.

## 2. Problem & Constraints

### Current Behaviour

Contracts live under `specify/tech/SPEC-*/contracts/` and are discovered via three supplementary index trees:

- `by-slug/<slug>` → `../SPEC-###` (spec-level, not contract-level)
- `by-package/<pkg>/spec` → `../../SPEC-###` (same)
- `by-language/<lang>/<id>/spec` → `../../SPEC-###` (same)

None of these provide direct paths to **contract artefacts**. A developer who knows their source path (`supekku/scripts/lib/specs/index.py`) must:

1. Know the SPEC ID (or look it up via `by-language`)
2. Navigate into `SPEC-123/contracts/`
3. Identify the right contract file among many (`supekku-scripts-lib-specs-index-public.md`)

### Drivers / Inputs

- PROD-014 — contract discovery ergonomics
- Agents cite contracts by source path, not SPEC ID

### Constraints / Guardrails

- Must not modify existing contract artefacts or `by-*` indices
- Must be safe to `.gitignore` entirely (NF-002)
- Rebuild must be interactive-speed: <2s for ~10k artefacts (NF-001)

### Out of Scope

- Relocating underlying contract files
- Removing existing `by-*` indices
- Aspirational/manual contracts
- Go/TypeScript mirror support beyond architecture (no contracts exist yet for these languages)

## 3. Architecture Intent

### Target Outcomes

- Source path → contract path with zero index knowledge (FR-001, FR-002)
- `rg .contracts/public/` behaves like searching the source tree (FR-001)
- Alias paths reduce idiom friction (FR-004)

### Guiding Principles

- Mirror-of-source beats taxonomy for daily discovery
- Same symlink mechanism as `SpecIndexBuilder` — proven, well-understood
- Language-specific rules isolated in pure mapping functions
- Discover artefacts from filesystem, not from aspirational metadata

### State Transitions / Lifecycle Impact

None — `.contracts/` is a derived, stateless view. No lifecycle state, no persistence.

## 7. Design Decisions & Trade-offs

### 7.1 Contract artefact discovery model

**Decision**: Scan `SPEC-*/contracts/` directories to discover actual artefacts; do not rely on spec frontmatter `variants[].path`.

**Rationale**: The spec frontmatter declares variant paths like `contracts/api.md`, but these files **do not exist** for Python. Python's `generate_docs()` produces per-module files via `PathNormalizer.get_output_filename()`:

```
# PathNormalizer.get_output_filename logic:
# module_name = "supekku.scripts.lib.specs.index"
# safe_name = "supekku-scripts-lib-specs-index"
# filename = f"{safe_name}-{variant}.md"
```

Actual files for SPEC-123 (`supekku/scripts/lib/specs`):

```
supekku-scripts-lib-specs-index-public.md
supekku-scripts-lib-specs-index-all.md
supekku-scripts-lib-specs-creation-public.md
supekku-scripts-lib-specs-creation-all.md
supekku-scripts-lib-specs-creation_test-tests.md
... (17 files total)
```

Go/Zig/TS produce simpler per-unit files (`interfaces.md`, `internals.md`, `api.md`, `internal.md`).

**Consequence**: The builder must parse contract filenames to extract module identity and variant. This is language-specific.

### 7.2 Python module identity from contract headers

**Problem**: Given a Python contract file, determine the source module path and variant for mirror placement.

**Previous approach (rejected)**: Strip a hyphenated registry-identifier prefix from the filename. This couples the mirror builder to the registry identifier format and breaks if a source unit is registered as a `.py` file or if a SPEC accumulates multiple Python sources.

**Chosen approach**: Read the first header line of the contract markdown.

All Python contracts produced by `generate_docs()` begin with a consistent header:

```markdown
# supekku.scripts.lib.specs.index
```

**Algorithm**:

```
Input:  contract file = "supekku-scripts-lib-specs-index-public.md"

1. Read first line: "# supekku.scripts.lib.specs.index"
2. Strip "# " prefix → dotted module name: "supekku.scripts.lib.specs.index"
3. Convert to path: "supekku/scripts/lib/specs/index.py"
4. Extract variant from filename suffix: strip ".md", match last -{variant}
   Known variants: {public, all, tests}
   "supekku-scripts-lib-specs-index-public" → variant = "public"
5. Mirror path: .contracts/public/supekku/scripts/lib/specs/index.py.md
```

**Robustness**: The module identity is derived from the contract's own content, not from registry metadata. This works regardless of how the source unit is registered.

**Variant extraction**: Still uses the filename suffix (the variant name is always the last hyphen-delimited segment before `.md`). Known Python variants: `public`, `all`, `tests`.

**Edge case**: Test files produce `creation_test-tests.md` → variant `tests`. The module name comes from the header, not the filename, so no ambiguity.

### 7.3 Contract filename parsing (Go / Zig / TypeScript)

These languages produce fixed-name contract files per source unit:

| Language       | Contract files                  | Variant → View mapping                       |
| -------------- | ------------------------------- | -------------------------------------------- |
| **Go**         | `interfaces.md`, `internals.md` | `public` → `public`, `internal` → `internal` |
| **Zig**        | `interfaces.md`, `internals.md` | `public` → `public`, `internal` → `internal` |
| **TypeScript** | `api.md`, `internal.md`         | `api` → `public`, `internal` → `internal`    |

Since these have one contract file per variant per source unit, parsing is a simple filename lookup — no prefix stripping needed.

### 7.4 Canonical view names and alias symlinks

**Decision**: Mirror uses canonical view directory names: `public`, `internal`, `all`, `tests`. Language-specific variant names are mapped to these.

| Canonical view | Python variant | Go variant | Zig variant | TS variant | Alias                    |
| -------------- | -------------- | ---------- | ----------- | ---------- | ------------------------ |
| `public/`      | `public`       | `public`   | `public`    | `api`      | `api` → `public`         |
| `internal/`    | —              | `internal` | `internal`  | `internal` | —                        |
| `all/`         | `all`          | —          | —           | —          | `implementation` → `all` |
| `tests/`       | `tests`        | —          | —           | —          | —                        |

### 7.5 Mirror path construction per language

**Python** (module-level, per-file contracts):

```
Contract:   SPEC-123/contracts/supekku-scripts-lib-specs-index-public.md
Header:     "# supekku.scripts.lib.specs.index"
Parsed:     module_path="supekku/scripts/lib/specs/index.py", variant="public"
Mirror:     .contracts/public/supekku/scripts/lib/specs/index.py.md
Symlink →   specify/tech/SPEC-123/contracts/supekku-scripts-lib-specs-index-public.md
```

**Go** (package-level, per-unit contracts):

```
Registry:   "internal/foo/bar" → SPEC-003
Contract:   SPEC-003/contracts/interfaces.md
Mirror:     .contracts/public/internal/foo/bar/interfaces.md
Symlink →   specify/tech/SPEC-003/contracts/interfaces.md
```

**Zig** (file-level, per-unit contracts):

```
Registry:   "src/domain/combat/agent.zig" → SPEC-007
Contract:   SPEC-007/contracts/interfaces.md
Mirror:     .contracts/public/src/domain/combat/agent.zig.md
Symlink →   specify/tech/SPEC-007/contracts/interfaces.md

Root package:
Registry:   "." → SPEC-001
Mirror:     .contracts/public/__root__/interfaces.md
```

**TypeScript** (file-level, per-unit contracts):

```
Registry:   "src/api.ts" → SPEC-050
Contract:   SPEC-050/contracts/api.md
Mirror:     .contracts/public/src/api.ts.md
Symlink →   specify/tech/SPEC-050/contracts/api.md
```

### 7.6 Symlink target computation

Symlinks must use **relative paths** (not absolute) so the tree is portable. The `SpecIndexBuilder` pattern computes depth-relative `../` chains.

For a mirror entry at `.contracts/public/supekku/scripts/lib/specs/index.py.md`, the symlink target must traverse:

```
.contracts/public/supekku/scripts/lib/specs/  ← 6 levels deep from repo root
specify/tech/SPEC-123/contracts/              ← target

Relative: ../../../../../../specify/tech/SPEC-123/contracts/supekku-scripts-lib-specs-index-public.md
```

Use `os.path.relpath(target, link_parent)` for correctness.

### 7.7 Conflict resolution

When two SPEC bundles produce contracts that map to the same mirror path:

1. **Detection**: Collect all `(mirror_path → [(spec_id, contract_path)])` mappings; flag entries with >1 source.
2. **Resolution**: Lexicographically smallest SPEC ID wins (e.g. `SPEC-004` over `SPEC-113`).
3. **Warning**: Emit `Warning: conflict at .contracts/public/foo.py.md — SPEC-004 wins over SPEC-113`.

### 7.8 Missing variant handling

If a contract artefact referenced by the mapping doesn't exist on disk:

- Skip the mirror entry for that variant
- Log a warning (not a hard failure)
- Continue processing remaining entries

## 9. Rollout & Operational Notes

- **Migration**: Run `uv run spec-driver sync` to populate. Recommend adding `.contracts/` to `.gitignore` to avoid churn from a generated index, but this is a project-level policy choice — some teams may prefer to commit the tree for tool-free browsing.
- **Phased delivery**: Python + Zig in phase 1. Go/TS mappers implemented architecturally but untestable until those languages have registered source units.
- **Recovery / Rollback**: `rm -rf .contracts/` and re-run sync. Stateless, no data migration.
