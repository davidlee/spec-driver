## 1. Summary
- **Delta**: DE-027 – Contract mirror tree index
- **Specs Impacted**: PROD-014 (primary), SPEC-110, SPEC-123 (collaborators)
- **Desired Outcome**: A `.contracts/` symlink tree at repo root that mirrors source paths to generated contract artefacts, enabling intuitive navigation and `rg`-based search.

## 2. Context & Constraints
- **Current Behaviour**: Contracts under `SPEC-*/contracts/` with `by-slug/`, `by-package/`, `by-language/` indices. None mirror source paths.
- **Target Behaviour**: `.contracts/<view>/<source-path>.md` → symlink to underlying contract artefact (PROD-014 FR-001–FR-007).
- **Dependencies**: Existing `registry_v2.json` and contract artefacts (already generated by sync adapters). Zig contracts available via hotlinked project.
- **Constraints**: Must not break existing indices; `.contracts/` should be `.gitignore`-safe; rebuild <2s.
- **Language scope**: Python + Zig testable in phase 1; Go/TS architecture only.

## 6. Testing & Verification Plan

- **New Cases**:
  - Python: header-based module identity → mirror path
  - Python: variant suffix extraction from filename
  - Zig: file-based identifier → mirror path (including root `"."` → `__root__/`)
  - Go/TS: path mapping functions (architecture, limited test fixtures)
  - Missing variant: skipped with warning, no error
  - Conflicting destinations: deterministic precedence, warning emitted
