# DR-029 – Canonical contract storage

## 1. Executive Summary

- **Delta**: [DE-029](./DE-029.md)
- **Status**: draft
- **Last Updated**: 2026-02-21
- **Synopsis**: Invert contract storage: `.contracts/` becomes canonical (real files), `SPEC-*/contracts/` becomes compat-only (symlinks pointing in). Adapters write directly to `.contracts/`-rooted paths.

## 2. Problem & Constraints

### Current Behaviour

Adapters hardcode `contracts_dir = spec_dir / "contracts"`, writing contract artefacts as real files under `SPEC-*/contracts/`. The `.contracts/` mirror tree is a post-hoc symlink index (DE-027) that points back into spec bundles.

This creates three problems:

1. **Spec-dependent generation**: contracts cannot be produced without a spec directory to write into. With spec auto-creation off (DE-028), sync generates nothing.
2. **Fragile indirection**: `.contracts/` entries are symlinks whose targets depend on spec bundle layout. Deleting or restructuring a spec breaks the mirror silently.
3. **Wrong ownership model**: canonical files live under a spec-centric namespace, but contracts are derived from source code (ADR-003). Ownership should be source-path-based.

### Drivers / Inputs

- [RE-015](../../change/revisions/RE-015-contracts_canonical_storage_sync_less_generation/RE-015.md) — canonical storage + sync-less generation revision
- [ADR-003](../../specify/decisions/ADR-003-separate_unit_and_assembly_specs.md) — observed contracts are the canonical interface truth
- [DE-027](../DE-027-contract_mirror_tree_index/DE-027.md) — v1 mirror tree; established path mapping rules
- [DE-028](../DE-028-sync_defaults_contracts_first_opt_in_spec_creation/DE-028.md) — contracts-first defaults; `generate_contracts` seam

### Constraints / Guardrails

- Must not break `sync --specs` workflows — spec creation, registry updates, frontmatter updates unchanged
- `.contracts/` remains `.gitignore`-safe (derived, deterministic, regenerable)
- DE-028's `--contracts/--no-contracts` flag continues to gate intent; this delta changes _where_, not _whether_
- Single callsite: `process_source_unit` is the only caller of `adapter.generate()`

### Out of Scope

- Spec-independent generation (DE-030/future) — this delta still uses the existing sync discovery pipeline
- Removing `SPEC-*/contracts/` directories — compat symlinks preserve them
- Removing `by-*` indices
- Python adapter internal refactoring — `generate_docs()` interface unchanged

## 3. Architecture Intent

### Target Outcomes

- `.contracts/<view>/<mirror-path>` contains real files as canonical storage (FR-008)
- `SPEC-*/contracts/<file>` → `.contracts/` compat symlinks for existing workflows (FR-011)
- Convention drift warnings when a SPEC has contracts but produces zero canonical entries (FR-012)
- All adapters accept `variant_outputs` instead of `spec_dir` (FR-006 update)

### Guiding Principles

- **Caller decides destination**: adapters are pure writers; the caller (`process_source_unit`) computes the output path
- **Reuse existing mapping logic**: path mapping functions from DE-027's `mirror.py` are reused, applied _before_ generation (Go/Zig/TS) or _after_ (Python)
- **Deterministic, stateless outputs**: `.contracts/` is entirely derived; `rm -rf .contracts/ && sync` reproduces everything
- **Central validation of adapter contracts**: `DocVariant.path` relativity enforced at the callsite, not inside adapters

### State Transitions / Lifecycle Impact

None — `.contracts/` and compat symlinks are derived views with no lifecycle state.

## 7. Design Decisions & Trade-offs

### 7.1 Adapter interface: `spec_dir` → `variant_outputs`

**Decision**: Change `LanguageAdapter.generate()` from `spec_dir: Path` to `variant_outputs: dict[str, Path]`, where each value is the exact canonical output file path for that variant.

**Current state**: All 4 adapters receive `spec_dir` and internally compute output paths:

```python
# Go/Zig (current)
contracts_dir = spec_dir / "contracts"
public_output = contracts_dir / "interfaces.md"

# Python (current)
output_root = spec_dir / "contracts"

# TypeScript (current)
output_root = spec_dir / "contracts"
```

**Problem with `contracts_dir`**: The canonical mirror paths from DE-027 are not uniform:

- Go: `.contracts/<view>/<identifier>/interfaces.md` — original adapter filename preserved
- Zig: `.contracts/<view>/<identifier>.md` — adapter filename (`interfaces.md`) discarded; leaf is `{identifier}.md`
- TS: `.contracts/<view>/<identifier>.md` — adapter filename (`api.md`) discarded; leaf is `{identifier}.md`

A `contracts_dir` parameter would produce the wrong filename for Zig and TS. The adapter writes `interfaces.md` but the canonical path expects `agent.zig.md`.

**Target state**: The caller computes per-variant canonical output file paths using mirror mapping functions and passes them as `variant_outputs`:

```python
# Go: original filenames preserved
variant_outputs = {
    "public": Path(".contracts/public/internal/foo/bar/interfaces.md"),
    "internal": Path(".contracts/internal/internal/foo/bar/internals.md"),
}
# Zig: adapter filename replaced by canonical leaf
variant_outputs = {
    "public": Path(".contracts/public/src/domain/agent.zig.md"),
    "internal": Path(".contracts/internal/src/domain/agent.zig.md"),
}
```

**Python exception**: Python can't use `variant_outputs` because it discovers N module files during AST walk. Python receives a staging directory via a separate code path (see §7.3).

**Consequence**: Adapters become pure writers — the caller owns all path decisions. Central validation after `adapter.generate()` ensures returned `DocVariant.path` values match the provided output paths.

### 7.2 `DocVariant.path` contract

**Decision**: Adapters MUST return `DocVariant.path` matching the output path they were given in `variant_outputs`. Centrally validated after `adapter.generate()`.

**Rules by language**:

- **Go/Zig/TS**: Must return the exact paths provided in `variant_outputs`. The caller already knows these paths — validation is a consistency check.
- **Python**: May return empty/placeholder paths. The caller uses filesystem scan of the staging directory instead, since module names are discovered during AST walk.

**Enforcement**: After `adapter.generate()` returns, `process_source_unit` validates each `DocVariant.path` against the provided `variant_outputs`. Mismatches are treated as errors.

One unit test per adapter locks the contract.

### 7.3 Python stage+distribute

**Decision**: Python uses stage+distribute. Adapter-aware views rejected.

**Flow**:

1. `process_source_unit` creates staging dir: `.contracts/.staging/<language>/<identifier-slug>/`
   - Staging key uses `unit.language` + identifier slug (not spec-id) — works even with `--no-specs`
2. Passes staging dir to `PythonAdapter.generate()` (Python-specific code path, not via `variant_outputs`)
3. Adapter writes N per-module files into staging dir (unchanged behaviour — just a different `output_root`)
4. Post-generation distribute step:
   - Scans staging dir
   - For each file: reads first header line via `read_python_module_name()` (from `mirror.py`) to get module identity
   - Extracts variant from filename suffix (known variants: `public`, `all`, `tests`)
   - Moves file to `.contracts/<view>/<module-path>.py.md`
5. Staging dir removed after distribution

**Why not adapter-aware views**: Would push `.contracts/**` path mapping logic into the Python adapter, breaking the principle that adapters are pure writers and the caller decides destination. Also harder to keep consistent across languages.

### 7.4 Compat symlinks (inverted mirror)

**Decision**: Invert the mirror builder. Instead of `.contracts/` → `SPEC-*/contracts/` (symlinks pointing out), create `SPEC-*/contracts/<file>` → `.contracts/<view>/<path>` (symlinks pointing in).

**Mechanics**:

- Only for source units with a registered spec (entries in `registry_v2.json`)
- Builder must `unlink()` existing files before creating symlinks
- Warn when replacing a non-symlink file (first migration run)
- Compat symlinks are derived — rebuilt on every sync, safe to delete

### 7.5 Convention drift warnings

**Decision**: Warn when a SPEC has a non-empty `contracts/` directory (≥1 `.md` file) but canonical generation produced zero `.contracts/**` entries for that spec's source unit.

**Rationale**: Catches mapper mismatches or adapter naming convention changes. Empty directories do not trigger warnings (no false positives).

### 7.6 Pre-generation path resolution (Go/Zig/TS)

**Decision**: For Go/Zig/TS, compute the canonical `.contracts/` output file paths _before_ calling `adapter.generate()`, using the mapping logic from `mirror.py`. Pass these as `variant_outputs`.

**Canonical path shapes** (from DE-027 `mirror.py` mappers):

| Language | Mapper               | Canonical path                                   | Adapter filename                            |
| -------- | -------------------- | ------------------------------------------------ | ------------------------------------------- |
| **Go**   | `go_mirror_entries`  | `.contracts/<view>/<identifier>/<contract_name>` | preserved (`interfaces.md`, `internals.md`) |
| **Zig**  | `zig_mirror_entries` | `.contracts/<view>/<identifier>.md`              | discarded; leaf = `{identifier}.md`         |
| **TS**   | `ts_mirror_entries`  | `.contracts/<view>/<identifier>.md`              | discarded; leaf = `{identifier}.md`         |

Examples:

```
Go:  variant_outputs = {"public": .contracts/public/internal/foo/bar/interfaces.md,
                        "internal": .contracts/internal/internal/foo/bar/internals.md}
Zig: variant_outputs = {"public": .contracts/public/src/domain/agent.zig.md,
                        "internal": .contracts/internal/src/domain/agent.zig.md}
TS:  variant_outputs = {"public": .contracts/public/src/api.ts.md,
                        "internal": .contracts/internal/src/api.ts.md}
```

**Key distinction**: For Zig and TS, the adapter internally generates content for `interfaces.md`/`api.md`, but writes to a differently-named output file. The adapter must use the provided `variant_outputs` path, not its internal filename convention.

**Difference from DE-027**: DE-027 computed mirror paths _after_ generation (post-hoc scan of `SPEC-*/contracts/`). DE-029 computes them _before_ generation, telling the adapter where to write directly. The mapping functions are the same; only the timing and direction change.

## 9. Rollout & Operational Notes

### Migration

- **Existing repos**: First `sync` after this delta writes real files to `.contracts/` and creates compat symlinks under `SPEC-*/contracts/`. Old real files in spec bundles are replaced by symlinks (with warning). Old symlinks in `.contracts/` are replaced by real files.
- **Fresh repos**: `.contracts/` populated directly once discovery is decoupled from specs (DE-030). This delta still requires the sync discovery pipeline.
- **Recovery**: `rm -rf .contracts/` + `sync` regenerates everything. No data loss risk.

### Observability

- Compat symlink builder warns on: non-symlink replacement, drift detection (non-empty `contracts/` with zero canonical entries)
- Sync summary output unchanged; contract generation counts still reported

### Recovery / Rollback

- Revert code changes → sync → `.contracts/` reverts to DE-027 symlink behaviour
- `SPEC-*/contracts/` real files still exist (not deleted by this delta)
