## 2. Context & Constraints

- **Current Behaviour**: Adapters write into `SPEC-*/contracts/`; `.contracts/` is a post-hoc symlink index pointing back in. Contracts cannot be generated without a spec directory.
- **Target Behaviour**: PROD-014.FR-008 (canonical storage), FR-011 (compat symlinks), FR-012 (drift warnings). PROD-012.FR-006 (adapter interface update).
- **Dependencies**: DE-028 (complete) — `generate_contracts` param exists on `process_source_unit`; `--contracts` flag wired.
- **Constraints**: Must not break existing `sync --specs` workflows. `.contracts/` remains `.gitignore`-safe. Single callsite for adapter interface (`process_source_unit`).

## 8. Open Questions & Decisions

- [x] Adapter interface: **per-variant output file paths** (`variant_outputs: dict[str, Path]`), not `contracts_dir`. Zig/TS canonical mirror paths use `{identifier}.md` as the leaf filename, not the adapter's internal filename (`interfaces.md`/`api.md`). A `contracts_dir` parameter can't reconcile this. Per-variant output paths let the caller dictate the exact canonical path.
- [x] Python multiplicity: **stage+distribute** confirmed. Staging location: `.contracts/.staging/<language>/<identifier-slug>/` (not spec-id — works with `--no-specs`). Distribute step reuses existing header/module-name parsing from `mirror.py`. Adapter-aware views rejected (pushes mirror mapping into adapters, harder to keep consistent across languages).
- [x] `DocVariant.path` contract: adapters MUST return `DocVariant.path` matching the provided output paths. Centrally validated after `adapter.generate()`. Unit test per adapter to lock the contract. Python may return empty/placeholder paths and rely on filesystem scan of staging dir; Go/Zig/TS must return the paths they were given.
