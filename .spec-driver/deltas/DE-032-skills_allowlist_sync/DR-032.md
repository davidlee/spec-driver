# DR-032 – skills_allowlist_sync

## 1. Executive Summary

- **Delta**: [DE-032](./DE-032.md)
- **Status**: draft (update when approved)
- **Owners / Team**: <names or teams accountable>
- **Last Updated**: 2026-03-01
- **Synopsis**: Add `spec-driver skills sync` to expose only allowlisted OpenSkills project skills in `AGENTS.md` without interactive selection.

## 2. Problem & Constraints

- **Current Behaviour**: OpenSkills can install project-local skills into `.agent/skills`, but `openskills sync` cannot be driven non-interactively to expose only a selected subset to agents.
- **Constraints / Guardrails**:
  - Idempotent output (minimal diffs).
  - Preserve global skills already declared in `AGENTS.md`.
  - Do not require OpenSkills runtime beyond the initial install; parse SKILL.md frontmatter directly.

## 3. Architecture Intent

- **Target Outcomes**:
  - Scriptable, allowlist-driven skills exposure for agents.
  - Stable file rewrite behavior focused on the `SKILLS_TABLE_START/END` region.
- **Guiding Principles**:
  - Spec-driver owns “exposure”; OpenSkills owns “distribution”.

| `supekku/cli/skills.py` | no skills commands                    | add `skills sync`                 |
