## 1. Summary & Context

spec-driver's skills system currently depends on an external npm tool (openskills) to install SKILL.md files into `.agent/skills/`, from which `skills sync` reads to generate AGENTS.md. This delta removes the openskills dependency entirely: skills are bundled in the spec-driver package (`supekku/skills/`) and installed directly to configurable agent-specific target directories.

## 2. Motivation

- openskills cannot CLI-select a subset of skills from a repo (only interactive or all)
- `--universal` installs to `.agent/` but Codex reads `.agents/` (plural) — "universal" isn't
- We are building workarounds for openskills limitations rather than solving the problem
- Skills are tightly coupled to spec-driver; an external npm dependency adds friction

## 3. Scope & Objectives

- **Primary Outcomes**:
  - `skills sync` copies allowlisted skills from package to configured agent target dirs
  - De-listed skills are pruned from target dirs (safely: only package-origin skills)
  - AGENTS.md generation reads from package source, not `.agent/skills/` cache
  - `workflow.toml` gains `[skills] targets` config for agent-specific dirs
  - Installer (`initialize_workspace`) includes skills installation
- **Dependencies**: None

## 4. Out of Scope

- Removing `.agent/skills/` or openskills artifacts (user cleanup)
- Per-skill CLI selection flags (allowlist is the mechanism)
- Manifest/tracking file (package skill names are pruning authority)
- `importlib.resources` (unnecessary — file paths are stable)

## 5. Approach Overview

- **System Touchpoints**: `core/paths.py`, `core/config.py`, `skills/sync.py`, `cli/skills.py`, `install.py`
- **Key Changes**:
  - Add `get_package_skills_dir()` to `paths.py`
  - Add `[skills]` config section with `targets = ["claude", "codex"]` default
  - Add `install_skills_to_target()`, `prune_skills_from_target()`, `get_package_skill_names()` to `sync.py`
  - Refactor `_collect_skills()` and `sync_skills()` to read from package source
  - Integrate `sync_skills()` into installer

## 6. Verification Strategy

- All existing tests updated and passing
- New tests for install, prune, config-driven targets
- Manual: `uv run spec-driver skills sync` installs to `.claude/skills/` and `.agents/skills/`
- `just test && just lint && just pylint`

## 7. Risks & Mitigations

- **Risk**: Existing tests hardcode `.agent/skills/` as source – _Likelihood_: certain – _Impact_: low – _Mitigation_: All fixtures updated to use `skills_source_dir=` parameter
- **Risk**: `.claude/` may not exist in fresh workspace – _Likelihood_: medium – _Impact_: low – _Mitigation_: `install_skills_to_target` creates target dirs with `mkdir(parents=True)`
