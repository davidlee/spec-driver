# DR-035 – assembly-only tech specs and unit-spec deprecation

## 1. Executive Summary

- **Delta**: [DE-035](./DE-035.md)
- **Status**: draft
- **Owners / Team**: TBD
- **Last Updated**: 2026-03-03
- **Synopsis**: Deprecate unit tech specs as a first-class concept and simplify to assembly-first specs + contracts.

## 2. Problem & Constraints

- **Current Behaviour**: Tooling still models a dual taxonomy where unit and assembly are first-class. This keeps UX and validation complexity for a low-value artifact class.
- **Drivers / Inputs**:
  - ADR-003 (accepted baseline)
  - PROD-015 taxonomy/navigation intent
  - RE-016 contracts-first sync defaults
  - RE-018 revision direction
- **Constraints / Guardrails**:
  - Non-breaking for existing SPEC IDs and repositories.
  - Warn-only migration behavior; do not hard fail legacy unit metadata.
  - Preserve contracts as canonical observed-code truth.
- **Out of Scope**:
  - One-shot migration rewriting all historical specs.
  - New ID prefixes or directory hierarchy changes.

## 3. Architecture Intent

- **Target Outcomes**:
  - New generated tech specs are assembly-oriented only.
  - Unit taxonomy becomes legacy compatibility, not target-state behavior.
  - CLI/index/validator behavior aligns with assembly-first workflows.
- **Guiding Principles**:
  - Simplify model surface area where value is not demonstrated.
  - Preserve compatibility during transitions.
  - Keep code-truth vs intent-truth separation explicit.
- **State Transitions / Lifecycle Impact**:
  - `category: unit` moves from supported-first-class to deprecated-but-supported.

## 4. Code Impact Summary

| Path                                          | Current State                                           | Target State                                                                    |
| --------------------------------------------- | ------------------------------------------------------- | ------------------------------------------------------------------------------- |
| `supekku/scripts/sync_specs.py`               | sync stubs may emit `category: unit` + `c4_level: code` | no new unit-tagged generation; assembly-first behavior                          |
| `supekku/scripts/lib/specs/creation.py`       | defaults centered on dual taxonomy                      | assembly-only default posture for authored/generated specs                      |
| `supekku/cli/list.py`                         | category UX includes unit as first-class target         | assembly-first UX; optional compatibility handling for explicit unit queries    |
| `supekku/scripts/lib/specs/index.py`          | by-category includes first-class unit views and aliases | deterministic assembly/unknown views; unit view compatibility scoped/deprecated |
| `supekku/scripts/lib/validation/validator.py` | validates unit/code consistency                         | warns that unit category is deprecated; avoids hard errors                      |
