# IP-040 – Separate template concerns: agents, artifacts, hooks

## 1. Summary

- **Delta**: DE-040 – Separate template concerns: agents, artifacts, hooks
- **Desired Outcome**: Templates directory cleanly separated into three concerns (artifacts, agents, hooks) with hooks installed to `.spec-driver/hooks/` and all references updated.

## 2. Context & Constraints

- **Current Behaviour**: `supekku/templates/` mixes artifact templates, agent templates, and hook files. `doctrine.md` is referenced by skills at `.spec-driver/doctrine.md` but was never installed by the installer.
- **Target Behaviour**: Three subdirectories (`agents/`, `hooks/`) under `supekku/templates/`; top-level remains artifact templates. Hooks installed to `.spec-driver/hooks/` with create-if-missing semantics.
- **Dependencies**: None
- **Constraints**: Must not break existing install flow or overwrite user-customized hook files on reinstall.

- [x] Delta scoped (DE-040)

- [x] All reference sites identified (4 skills × 3 locations)

## 4. Tasks

### T1: Move `boot.md` into `agents/` subdirectory

- Move `supekku/templates/boot.md` → `supekku/templates/agents/boot.md`
- Update `_render_boot_md` in `install.py:350` to use `"agents/boot.md"` template path
- Verify existing install tests still pass

### T2: Create hooks template directory and doctrine hook

- Create `supekku/templates/hooks/doctrine.md` (empty file — content is user-customizable)
- Create `supekku/templates/hooks/README.md` with brief explanation of hooks purpose

### T3: Add hooks install step to `initialize_workspace`

- New function `_install_hooks` in `install.py`:
  - Source: `package_root / "templates" / "hooks"`
  - Dest: `target_root / SPEC_DRIVER_DIR / "hooks"`
  - For each file: copy only if destination doesn't exist (create-if-missing)
  - Respect `dry_run` flag
- Call from `initialize_workspace` after template copy step
- Add `.spec-driver/hooks` to created directories list

### T4: Update skill references

- `supekku/skills/boot/SKILL.md`: `@.spec-driver/doctrine.md` → `@.spec-driver/hooks/doctrine.md`
- `supekku/skills/doctrine/SKILL.md`: `@.spec-driver/doctrine.md` → `@.spec-driver/hooks/doctrine.md`
