## 1. Summary

- **Delta**: DE-041 — CLI completeness — fill obvious command gaps
- **Specs Impacted**: PROD-010 (CLI UX), PROD-013 (CLI Artifact File Access)
- **Change Drivers**: DE-038 gaps-to-adoption §3, workflow friction
- **Desired Outcome**: Every artifact type with `create` also has `show/view/edit/find`; all `list` subcommands support baseline flags; plans have full CLI presence

## 2. Context & Constraints

- **Current Behaviour**: Several artifact types lack basic addressability (show/view/edit/find). Plans have zero CLI presence. List commands have inconsistent flag sets. Existing commands repeat ~30-line boilerplate patterns ~30 times.
- **Target Behaviour**: DR-041 §3 target outcomes 1-6. Shared helpers eliminate boilerplate; new commands are 3-5 lines each.
- **Dependencies**: None. All registries and formatters exist for most types.
- **Constraints**: Preserve existing command signatures (no breaking changes). Skinny CLI pattern. Shared helpers must earn their keep by eliminating real duplication.

## 6. Testing & Verification Plan

- **Test strategy**: Unit-test shared capabilities (`resolve_artifact`, `emit_artifact`, `find_artifacts`, `create_plan`, formatters, `find_backlog_items_by_id`) thoroughly with parameterized tests. Integration tests for CLI commands verify wiring correctness only.
- **Pre-migration requirement**: Write comprehensive regression tests for revision commands BEFORE migrating them to shared helpers (VT-migration).
- **New test files**: `supekku/cli/common_test.py` (helpers), additions to `show_test.py`, `view_test.py`, `edit_test.py`, `find_test.py`, `list_test.py`
- **Tooling**: typer.testing.CliRunner, pytest parameterize
- **Rollback**: Purely additive + one refactor with regression tests; revert is straightforward
