# DR-042 – Fix Go contract generation and respect calling shell environment

## 1. Executive Summary

- **Delta**: [DE-042](./DE-042.md)
- **Status**: draft
- **Synopsis**: Fix broken gomarkdoc invocation (wrong path format, missing cwd, silent errors), then add optional environment passthrough to all sync adapters.

## 2. Problem & Constraints

### Current Behaviour

**Bug A — gomarkdoc invocation (primary)**:

`GoAdapter.generate()` constructs a fully-qualified import path for gomarkdoc:

```python
# go.py:215-216
module = get_go_module_name(self.repo_root)  # e.g. "im"
module_pkg = f"{module}/{unit.identifier}"   # e.g. "im/cmd/im"
```

gomarkdoc rejects this: `invalid package at import path: im/cmd/im`. It expects `./`-prefixed relative paths (`./cmd/im`) when invoked from the module root.

Additionally:

- `subprocess.run()` calls at lines 277 and 298 omit `cwd`, so gomarkdoc doesn't know where the module root is
- The outer `except CalledProcessError` at line 319 catches the failure and returns `status="unchanged"`, making the failure completely invisible — directories are created but no files are written

**Bug B — PATH not forwarded**:

All adapters use `shutil.which()` for tool detection and `subprocess.run()` with inherited environment. When spec-driver is invoked from a context where the target project's tools aren't in PATH, the tools are invisible. This is correct by Python's design but unhelpful for the common case of nix-direnv projects.

### Constraints

- No new hard dependencies (direnv is not required)
- Must not break existing projects where tools are in system PATH
- Error handling must surface failures without aborting the entire sync run

## 3. Architecture Intent

### Target Outcomes

1. gomarkdoc receives `./`-prefixed relative package paths and runs with `cwd=repo_root`
2. gomarkdoc failures are surfaced (warning/error), not silently swallowed
3. All adapters can optionally receive an `env` dict that flows to `subprocess.run()` and tool availability checks
4. Engine constructs and passes `env` from the calling process environment

### Design Decisions

**DEC-042.1 — Use `./` relative paths for gomarkdoc**

`unit.identifier` is already a module-relative path (e.g. `cmd/im`, `internal/cli`). The `generate()` method should pass `f"./{unit.identifier}"` directly to gomarkdoc rather than reconstructing the fully-qualified import path. This eliminates the `get_go_module_name()` call in `generate()` entirely — it's only needed during `discover_targets()`.

**DEC-042.2 — Surface gomarkdoc errors as warnings**

The outer `except CalledProcessError` at line 319 must not return `status="unchanged"`. Instead:

- Log the error via `warnings.warn()` or return a new status value (e.g. `"error"`)
- The inner `except` at line 279 (check mode) is fine — gomarkdoc exits non-zero when docs differ, which correctly maps to `status="changed"`
- Re-raise from the outer except so the engine's `_process_source_unit` catches it and records in `outcome.errors`

**DEC-042.3 — Optional `env` on LanguageAdapter base class**

Add `env: dict[str, str] | None = None` to `LanguageAdapter.__init__()`. When set:

- Pass to all `subprocess.run(..., env=env)` calls
- Use for tool availability checks via a helper that wraps `shutil.which()` with PATH from env

When `None` (default), behaviour is unchanged — subprocess inherits the parent process environment, `which()` uses system PATH. This keeps the change backward-compatible.

## 4. Code Impact Summary

| Path                                             | Current State                                                            | Target State                                                                             |
| ------------------------------------------------ | ------------------------------------------------------------------------ | ---------------------------------------------------------------------------------------- |
| `sync/adapters/go.py` `generate()`               | Constructs `f"{module}/{identifier}"`, passes to gomarkdoc without `cwd` | Uses `f"./{identifier}"`, passes `cwd=self.repo_root`                                    |
| `sync/adapters/go.py` `_generate_variant()`      | Outer except swallows `CalledProcessError` silently                      | Re-raises; let engine record the error                                                   |
| `sync/adapters/go.py` `_generate_variant()`      | No `cwd` on subprocess calls                                             | `cwd=self.repo_root` on all gomarkdoc calls                                              |
| `sync/adapters/go.py` `generate()`               | Calls `get_go_module_name()`                                             | Removed — not needed for generation                                                      |
| `sync/adapters/go.py` `is_gomarkdoc_available()` | `which("gomarkdoc")` with system PATH                                    | Uses `self._which("gomarkdoc")` respecting adapter env                                   |
| `sync/adapters/base.py` `__init__()`             | `(repo_root)`                                                            | `(repo_root, env=None)`                                                                  |
| `sync/adapters/base.py`                          | No env-aware which helper                                                | Add `_which(cmd)` helper using `self.env` PATH                                           |
| `sync/adapters/base.py`                          | `_get_git_tracked_files()` uses bare subprocess                          | Passes `self.env` to subprocess                                                          |
| `sync/engine.py`                                 | Constructs adapters with `(repo_root)`                                   | Passes `env=os.environ.copy()` (or caller-provided)                                      |
| `core/go_utils.py`                               | Functions take no `env` param                                            | Add optional `env` param to `is_go_available()`, `get_go_module_name()`, `run_go_list()` |
| `sync/adapters/zig.py`                           | `which()` and subprocess with system env                                 | Uses adapter env for both                                                                |
| `sync/adapters/typescript.py`                    | `which("node")` with system PATH                                         | Uses adapter env                                                                         |

## 6. Supporting Context

- gomarkdoc docs confirm `./` relative path is the standard invocation pattern
- TypeScript adapter already sets `cwd=package_root` — Go adapter should follow the same pattern
- Zig adapter uses file paths (not import paths), so it's not affected by the path format bug but will benefit from env passthrough
