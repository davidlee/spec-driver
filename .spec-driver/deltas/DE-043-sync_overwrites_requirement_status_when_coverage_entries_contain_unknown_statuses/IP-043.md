## 1. Summary

- **Delta**: DE-043 – Verification coverage validation and lifecycle guidance
- **Specs Impacted**: SPEC-122 (requirements registry), PROD-008, PROD-009
- **Problems / Issues**: ISSUE-033
- **Desired Outcome**: Unknown coverage statuses can never corrupt derived
  requirement state; agents have discoverable lifecycle guidance.

## 2. Context & Constraints

- **Current Behaviour**: Unknown coverage statuses silently corrupt derived
  requirement status via `_compute_status_from_coverage()` mixed-status fallback.
  See DR-043 §2.
- **Target Behaviour**: Validation at sync boundary filters invalid entries
  with warning; derivation references canonical vocabulary. See DR-043 §4.
- **Dependencies**: None.
- **Constraints**: Minimal code surface. No schema changes. No new CLI commands.
