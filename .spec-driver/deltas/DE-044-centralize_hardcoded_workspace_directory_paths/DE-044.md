## 1. Summary & Context

- **Change Drivers**: PROB-003 (magic path strings), POL-002 (avoid magic strings)
- **Implementation Plan**: IP-044 (to be created)

## 2. Motivation

The workspace directory names `specify/`, `change/`, `backlog/`, and `memory/`
are hardcoded as string literals across ~120 references in ~70 Python files.
This violates POL-002, makes renaming brittle, and blocks future
configurability of install directory layout.

`paths.py` already centralizes `.spec-driver` — the workspace dirs were simply
never given the same treatment.

## 3. Scope & Objectives

- **Primary Outcomes**:
  - All workspace directory names defined as constants in `core/paths.py`
  - All production code uses constants instead of string literals
  - Test fixtures use constants or helpers built on them
- **Operational Constraints**: Pure refactor — no behaviour change, no new features
- **Dependencies**: None

## 4. Out of Scope

- Making directory names configurable via `workflow.toml` (follow-up delta)
- Renaming any directories
- Changing directory structure or nesting

## 5. Approach Overview

- **System Touchpoints**: `core/paths.py`, all registries, CLI commands, install script, test fixtures
- **Key Changes**:
  - Phase 1: Add constants and helper functions to `core/paths.py`
  - Phase 2: Replace literals in production code (registries, CLI, install)
  - Phase 3: Replace literals in test fixtures

## 6. Verification Strategy

- **Acceptance Criteria**:
  - `just` passes (tests + both linters)
  - `rg '"specify"' --type py` returns only test string content, not path construction
  - Same for `"change/"`, `"backlog"`, `"memory/"`
  - No behavioural changes — all existing tests pass unmodified (just with constant imports)

## 7. Risks & Mitigations

- **Risk**: Missed reference causes runtime path error – _Likelihood_: low – _Impact_: medium – _Mitigation_: grep verification + full test suite
- **Risk**: Test churn obscures review – _Likelihood_: medium – _Impact_: low – _Mitigation_: phase test changes separately

## 8. Follow-ups & Tracking

- **Future Deltas**: Directory name configurability via `workflow.toml` `[dirs]` section
- **Backlog Items**: PROB-003
