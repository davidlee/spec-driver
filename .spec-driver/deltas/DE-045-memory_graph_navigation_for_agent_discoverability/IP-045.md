# IP-045 — Memory graph navigation for agent discoverability

## Objectives

Turn the memory corpus from a forward-only linked list into a navigable graph.
Three capabilities: backlinks (`--links-to`), subgraph expansion (`--links-depth N`),
and body-only output (`--body-only/-b`).

## Design decisions (settled)

1. **Backlink data source**: parse all memory bodies at query time — no prerequisites,
   no frontmatter mutation. Keep computation in a pure function so a registry cache
   slots in later without touching callers.
2. **`--links-depth` output**: compact table (id, name, depth) by default; `--tree`
   for indented tree view; `--json` for structured.
3. **ADR-002 compliance**: no backlinks stored in frontmatter. Runtime computation only.

## Phase plan

### Phase 1: Graph domain + shared infrastructure

Build the pure functions and formatters. Also migrate `show_memory` to use
`emit_artifact` and add `--body-only` to the shared helper so all show
commands can adopt it later.

- Domain: `compute_backlinks()`, `expand_link_graph()` in `memory/links.py`
- Formatters: `format_link_graph_table()`, `format_link_graph_tree()` in `memory_formatters.py`
- Infrastructure: add `body_only` mode to `emit_artifact` in `cli/common.py`
- Migrate `show_memory` to use `emit_artifact` (consistency win, prerequisite for `--body-only`)
- Full unit test coverage for all new pure functions and formatters

**Entry**: DE-045 delta scoped and approved
**Exit**: all new domain functions + formatters tested, `emit_artifact` supports `--body-only`, `show_memory` migrated

### Phase 2: CLI wiring + integration tests

Wire the domain functions into CLI commands. Thin orchestration only.

- `list memories --links-to <id>` — calls `compute_backlinks()`, filters results
- `show memory --links-depth N [--tree]` — calls `expand_link_graph()`, formats output
- `show memory --body-only/-b` — already wired via `emit_artifact` migration
- CLI integration tests for all three flags
- Agent walkthrough (VA)
- Lint + full test suite pass

**Entry**: Phase 1 complete, all domain tests green
**Exit**: acceptance criteria from DE-045 §6 satisfied, VA walkthrough documented

## Reuse notes

- `emit_artifact` gains `body_only` parameter — available to all 15+ show commands for free
- `compute_backlinks` returns `dict[str, list[str]]` (target → list of source IDs) — reusable for any future graph query
- `expand_link_graph` is generic over any `known_artifacts` index — not memory-specific
- `parse_links` (existing) is the shared parser for both backlinks and depth expansion
