## 1. Summary & Context

- **Product Spec(s)**: PROD-002 (Delta Creation Workflow, draft)
- **Implementation Plan**: [IP-046](./IP-046.md)
- **Change Drivers**: DE-043 closure failure — `complete delta` exits 1 with no output

## 2. Motivation

`uv run spec-driver complete delta DE-043` silently fails. The user sees three
prompts, confirms all, then gets exit code 1 with zero diagnostic output. The
delta remains `in-progress`. This is a bad experience — silent failures
undermine trust in the tool.

Two code issues:
1. `complete_delta.py:307-308` catches `RevisionUpdateError` without printing
   the error message, returning `False` silently.
2. `updater.py:138-144` validates the *entire* revision block after a targeted
   status update. Pre-existing validation issues in other fields (unrelated to
   the change) cause the update to be rejected.

## 3. Scope & Objectives

- **Primary Outcomes**:
  - All error paths in `complete delta` produce descriptive stderr output
  - `update_requirement_lifecycle_status` tolerates pre-existing validation
    issues that are not caused by the current update
- **Operational Constraints**: Minimal change surface — bug fix, not refactor
- **Dependencies**: None

## 4. Out of Scope

- Fixing pre-existing data in RE-002.md (symptom, not cause)
- Overhauling the `RevisionBlockValidator` schema rules
- Adding specs for the completion/updater modules

## 5. Approach Overview

- **System Touchpoints**: `complete_delta.py`, `updater.py`
- **Key Changes**:
  - Print `RevisionUpdateError` message to stderr before returning False
  - In `update_requirement_lifecycle_status`, validate block both before and
    after update; only reject if the update *introduced* new validation errors

## 6. Verification Strategy

- **VT-046-001**: Unit test — `RevisionUpdateError` produces stderr output
- **VT-046-002**: Unit test — targeted update succeeds when block has
  pre-existing validation issues unrelated to the changed field
- **VT-046-003**: Regression — existing updater tests still pass

## 7. Risks & Mitigations

- **Risk**: Pre-update validation might mask real issues in the changed field
  – *Likelihood*: low – *Impact*: low
  – *Mitigation*: Compare error sets; only allow errors that existed before

## 8. Follow-ups & Tracking

- **Backlog Items**: Consider creating specs for completion and updater modules
- **Related**: DE-043 closure blocked by this bug
