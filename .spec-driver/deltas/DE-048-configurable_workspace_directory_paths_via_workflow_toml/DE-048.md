## 1. Summary & Context

- **Change Drivers**: PROB-003 (magic path strings — remaining configurability criterion), IMPR-008 (workspace layout consolidation), POL-002
- **Implementation Plan**: [IP-048](./IP-048.md)
- **Predecessor**: DE-044 (centralized constants in `core/paths.py`)

## 2. Motivation

DE-044 centralized workspace directory names as module-level constants. But the
values are still hardcoded literals — there's no mechanism for users or the
installer to override them. DE-049 (layout consolidation) depends on being able
to change these values via config.

This delta adds `[dirs]` config resolution to `workflow.toml` and wires it into
the existing `get_*_dir()` helpers via a one-time `init_paths(config)` call at
startup. Non-breaking — defaults match the current layout.

## 3. Scope & Objectives

- **Primary Outcomes**:
  - `workflow.toml` supports a `[dirs]` section for overriding workspace directory names
  - `paths.py` helpers resolve directory names from config when initialized
  - `init_paths(config)` / `reset_paths()` API for startup and test teardown
  - All existing tests pass without modification (defaults unchanged)
- **Operational Constraints**: Non-breaking — no default changes, no directory moves
- **Dependencies**: DE-044 (completed)

## 4. Out of Scope

- Changing default directory locations (DE-049)
- Migration tooling
- Convenience symlinks
- Installer changes

## 5. Approach Overview

- **System Touchpoints**: `core/paths.py`, `core/config.py`, CLI entry point, test base classes
- **Key Changes**:
  - Add `[dirs]` schema to `DEFAULT_CONFIG` in `config.py`
  - Add `init_paths(config)` to `paths.py` that overrides module-level constants from config dict
  - Add `reset_paths()` for test teardown
  - Call `init_paths()` in CLI startup (after config is loaded)
  - Update `RepoTestCase` and pytest fixtures to reset paths in teardown

## 6. Verification Strategy

- **VT-048-config**: Unit tests for `[dirs]` config parsing and `init_paths()` / `reset_paths()`
- **VT-048-custom**: Tests with non-default directory names verify helpers return custom paths
- **VT-048-regression**: Full test suite passes — defaults unchanged
- **Acceptance Criteria**: `just` green; custom `[dirs]` config produces correct paths

## 7. Risks & Mitigations

- **Risk**: `init_paths()` not called before helpers used — _Likelihood_: low — _Impact_: medium — _Mitigation_: helpers use current constant values as fallback (no-init = current behavior)
- **Risk**: Test isolation — config state leaks between tests — _Likelihood_: medium — _Impact_: medium — _Mitigation_: `reset_paths()` in tearDown/fixture finalizer
