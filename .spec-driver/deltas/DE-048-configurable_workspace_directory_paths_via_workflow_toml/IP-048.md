## 1. Summary

- **Delta**: DE-048 — Configurable workspace directory paths via workflow.toml
- **Problems / Issues**: PROB-003 (remaining configurability criterion), IMPR-008
- **Desired Outcome**: Workspace directory names configurable via `[dirs]`
  in `workflow.toml`, wired through `init_paths(config)` at startup, with
  `reset_paths()` for test isolation. Non-breaking — defaults unchanged.

## 2. Context & Constraints

- **Current Behaviour**: `paths.py` has 19 hardcoded string constants and 19
  helper functions. 44 production files import them. No config mechanism exists.
- **Target Behaviour**: Constants become overridable module state. CLI
  initializes from config at startup. Tests reset in teardown.
- **Dependencies**: DE-044 (completed — centralized constants)
- **Constraints**: No default changes, no directory moves, no installer changes.
