## 1. Summary

- **Delta**: DE-050 — Normalise registry API surface for consistent artifact access
- **Governing ADR**: ADR-009 — standard registry API convention
- **Design Revision**: DR-050
- **Issues**: ISSUE-019 (registry API drift)
- **Downstream**: IMPR-009 (TUI dashboard)
- **Desired Outcome**: Every class-based registry exposes `find()`, `collect()`,
  `iter()`, `filter()` per ADR-009; common.py resolvers simplified.

## 2. Context & Constraints

- **Current Behaviour**: 5+ incompatible patterns across 9 registries. See
  [research.md](./research.md) for full audit.
- **Target Behaviour**: ADR-009 standard surface on all class-based registries.
  DR-050 defines per-registry code impacts.
- **Dependencies**: ADR-009 (accepted). No code dependencies.
- **Constraints**: Additive only. No breaking constructor changes. Existing tests
  pass. Both linters pass.
