## 1. Summary & Context

- **Product Spec**: PROD-016 – agentic SDLC configurator (FR-001: single config file; FR-002: minimal knobs; FR-008: config-tailored guidance)
- **Implementation Plan**: [IP-051](./IP-051.md)
- **Change Drivers**:
  - DL-047.010 — spec auto-creation opt-in uses a marker file instead of `workflow.toml`
  - ISSUE-040 — no way to regenerate `workflow.md` from `workflow.toml` after post-install config changes

## 2. Motivation

- PROD-016.FR-001 requires a single workflow configuration file readable by both spec-driver and agents.
- The spec auto-creation preference currently uses a standalone marker file, creating config sprawl.
- ADR-003 is already neutral ("filesystem marker/config"); CLAUDE.md explicitly references the marker file.
- Moving this to `[sync] spec_autocreate = true|false` in `workflow.toml` eliminates a bespoke persistence mechanism and aligns with the config contract.

## 3. Scope & Objectives

- **Primary Outcomes**:
  - `spec_autocreate_enabled()` reads from `workflow.toml` `[sync]` section (FR-001)
  - `persist_spec_autocreate()` writes to `workflow.toml` using `tomlkit` (preserves comments/formatting)
  - Backward compat: if marker file exists and TOML key is absent, honour marker and migrate on next persist
  - Delete marker file after successful migration
  - Update CLAUDE.md reference from marker file to TOML config
  - Regenerate agent docs from current `workflow.toml` as an early step in `sync` (FR-008, ISSUE-040)
- **Operational Constraints**: No breaking changes to `sync` command behavior.

## 4. Out of Scope

- Scoping auto-creation to specific languages/globs (future work noted in DL-047.010)
- Other PROD-016 config migrations
- ADR-003 revision (already neutral wording)

## 5. Approach Overview

- **System Touchpoints**:
  - `supekku/scripts/lib/core/sync_preferences.py` — rewrite to read TOML + write via tomlkit
  - `supekku/scripts/lib/core/sync_preferences_test.py` — update tests
  - `supekku/scripts/lib/core/config.py` — add `[sync]` to DEFAULT_CONFIG
  - `CLAUDE.md` — update marker file reference
  - `supekku/scripts/install.py` — extract `_render_agent_docs` to shared location
  - `supekku/cli/sync.py` — call agent doc render early in sync flow
  - `pyproject.toml` — add `tomlkit` dependency
- **Key Changes**:
  1. Add `tomlkit` as a dependency for TOML read-modify-write
  2. Add `[sync] spec_autocreate = false` to DEFAULT_CONFIG
  3. Rewrite `spec_autocreate_enabled()` to check config, falling back to marker file
  4. Rewrite `persist_spec_autocreate()` to write TOML key via tomlkit, then delete marker if present
  5. Extract `_render_agent_docs` from installer to shared module; call early in `sync`
  6. Update CLAUDE.md

## 6. Verification Strategy

- **VT-051-001**: `spec_autocreate_enabled()` reads from TOML when key present
- **VT-051-002**: `spec_autocreate_enabled()` falls back to marker file when TOML key absent
- **VT-051-003**: `persist_spec_autocreate()` writes `[sync] spec_autocreate = true` to workflow.toml
- **VT-051-004**: `persist_spec_autocreate()` removes marker file after writing TOML
- **VT-051-005**: existing `sync` CLI behavior unchanged (integration-level)
- **VT-051-006**: `sync` re-renders agent docs from current workflow.toml early in its flow
- **Acceptance Criteria**: `just` passes; marker file no longer needed for new repos; ISSUE-040 resolved.

## 7. Risks & Mitigations

- **Risk**: tomlkit dependency adds weight – _Likelihood_: low – _Impact_: low – _Mitigation_: tomlkit is a well-maintained, pure-Python library; already standard for TOML round-trip editing.
- **Risk**: existing repos with marker file break – _Likelihood_: low – _Impact_: medium – _Mitigation_: backward-compat fallback reads marker; migration is automatic and idempotent.
