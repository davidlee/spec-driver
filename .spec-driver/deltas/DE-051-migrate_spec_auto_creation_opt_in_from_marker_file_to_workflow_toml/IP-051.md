## 1. Summary

- **Delta**: DE-051 — migrate spec auto-creation opt-in from marker file to workflow.toml
- **Specs Impacted**: PROD-016
- **Issues**: ISSUE-040 (no post-install agent doc regeneration), DL-047.010 (marker file config sprawl)
- **Desired Outcome**: All sync preferences live in workflow.toml; `sync` keeps agent docs fresh from config.

## 2. Context & Constraints

- **Current Behaviour**: `sync_preferences.py` reads/writes a marker file `.spec-driver/enable_spec_autocreate`. Agent docs are only rendered during install.
- **Target Behaviour**: Read/write `[sync] spec_autocreate` in workflow.toml (FR-001/FR-002). Re-render agent docs on every sync (FR-008).
- **Dependencies**: None.
- **Constraints**: Backward compat with existing marker files; no breaking changes to sync CLI.
