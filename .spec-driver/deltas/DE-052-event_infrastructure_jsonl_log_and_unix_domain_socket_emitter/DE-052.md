## 1. Summary & Context

- **Design Revision**: [DR-052](./DR-052.md)
- **Implementation Plan**: [IP-052](./IP-052.md)
- **Change Drivers**: IMPR-009 (TUI dashboard — track mode needs event stream)

## 2. Motivation

The TUI track mode (DE-054) needs structured events from CLI commands to follow
agent sessions in real time. But event infrastructure is independently useful —
JSONL logs provide audit trails and debugging visibility regardless of whether a
TUI is running.

This delta builds the emitter side: every leaf-command invocation records a
structured event. The TUI consumes these events later (DE-054).

## 3. Scope & Objectives

- **Primary Outcomes**:
  - `core/events.py` module with `emit_event()`, `record_artifact()`,
    `mark_command_invoked()` functions
  - JSONL event log at `.spec-driver/run/events.jsonl`
  - Unix domain socket datagram at `.spec-driver/run/tui.sock` (fire-and-forget)
  - `SPEC_DRIVER_SESSION` env var for session attribution, populated by
    `startup.sh` hook via `CLAUDE_ENV_FILE`
  - Process-boundary wrapper in `main()` that emits events after every
    leaf-command invocation, gated by a command-invocation flag
  - Artifact enrichment via `record_artifact()` calls in domain-layer
    creation/completion functions
  - `[events]` section in `workflow.toml` defaults (`enabled = true` only)
- **Operational Constraints**:
  - Stdlib only — no new dependencies
  - Zero-cost when no TUI is running (socket send fails silently)
  - Event emission must not slow CLI commands perceptibly
  - Non-blocking — never wait for socket delivery
  - Emission errors must never cause a CLI command to fail
- **Dependencies**: None (independent of DE-050, DE-053, DE-054)

## 4. Out of Scope

- TUI socket listener (DE-054)
- Event replay / log rotation
- Configurable event scope filtering (no `scope` config — log everything in v1)
- `"interrupted"` status (Click converts interrupts to `sys.exit(1)` before
  the wrapper sees them — indistinguishable from error)

## 5. Approach Overview

- **System Touchpoints**: `core/events.py` (new), `core/paths.py`,
  `core/config.py`, `cli/main.py`, `claude.hooks/startup.sh`,
  domain creation modules, `.gitignore`
- **Key Changes**:
  - New `core/events.py`:
    - `emit_event(argv, exit_code, status)` — appends JSONL line + sends
      socket datagram
    - `record_artifact(artifact_id)` — opt-in artifact enrichment
    - `mark_command_invoked()` / `command_was_invoked()` — flag for gating
    - `_detect_session()` — reads `SPEC_DRIVER_SESSION`, falls back to
      `CLAUDECODE` detection
    - Creates `.spec-driver/run/` directory lazily on first write
    - Socket send: `AF_UNIX`, `SOCK_DGRAM`, non-blocking, silent on failure
    - Runtime socket path length check (skip if > 104 chars)
  - `core/paths.py`: add `get_run_dir()` (hardcoded `.spec-driver/run/`,
    not configurable)
  - `core/config.py`: add `[events]` section to `DEFAULT_CONFIG`
    (`enabled = true` only)
  - `cli/main.py`: intercept `Command.invoke` to set invocation flag; wrap
    `app()` with process-boundary emission
  - `startup.sh`: read stdin JSON via Python, extract `session_id`, export
    via `CLAUDE_ENV_FILE`
  - `.gitignore`: add `.spec-driver/run/`

### Event schema (v1)

```jsonl
{"v":1,"ts":"2026-03-07T14:23:01Z","session":"abc123","cmd":"create delta","argv":["create","delta","--from-backlog","IMPR-009"],"artifacts":["DE-052"],"exit_code":0,"status":"ok"}
```

Fields:
- `v`: schema version (integer, always `1`)
- `ts`: ISO 8601 UTC timestamp
- `session`: `SPEC_DRIVER_SESSION` value, `"claude"` (type fallback), or `null`
- `cmd`: resolved command path
- `argv`: raw `sys.argv[1:]` (includes flags for full audit fidelity)
- `artifacts`: list of IDs touched (via `record_artifact()`; empty if unenriched)
- `exit_code`: integer exit code
- `status`: `"ok"` or `"error"`

## 6. Verification Strategy

- **VT**: Unit tests for `emit_event()` — JSONL format, file append, socket
  send (mock socket)
- **VT**: Silent failure when socket doesn't exist or path exceeds 104 chars
- **VT**: Session env var reads correctly / defaults to null / CLAUDECODE fallback
- **VT**: Config parsing for `[events]` section
- **VT**: Process-boundary wrapper emits correct exit_code for all exit paths
- **VT**: `record_artifact()` populates `artifacts` field
- **VT**: No event for `--help`, `--version`, bare `spec-driver`,
  `spec-driver create` (no subcommand), invalid command
- **VT**: Graceful no-op when no workspace exists
- **Acceptance Criteria**:
  - `spec-driver create delta "test"` produces a JSONL line in events log
  - `spec-driver create` (no subcommand) does NOT produce an event
  - No error when `.spec-driver/run/tui.sock` doesn't exist
  - `just` passes

## 7. Risks & Mitigations

- **Risk**: JSONL file grows unbounded — *Likelihood*: low (at current usage) —
  *Impact*: low — *Mitigation*: Defer rotation; log is append-only and
  trivially truncatable
- **Risk**: Socket path too long for `AF_UNIX` (108 char limit) —
  *Likelihood*: low — *Impact*: medium — *Mitigation*: Runtime length check;
  skip socket with one-time warning if exceeded; JSONL log unaffected
- **Risk**: Concurrent JSONL writes from multiple agents —
  *Likelihood*: medium — *Impact*: low — *Mitigation*: Best-effort local
  telemetry; small `O_APPEND` writes are atomic on common local filesystems;
  not guaranteed on NFS
