## 1. Summary
- **Delta**: DE-053 — TUI artifact browser MVP with Textual
- **Design Revision**: DR-053 (10 decided design decisions, 0 open questions)
- **Desired Outcome**: `spec-driver tui` launches a browsable 3-panel
  dashboard over all artifact types, with shared infrastructure (artifact
  view layer, column defs, style API) reusable by future consumers.

## 2. Context & Constraints
- **Dependencies**: DE-050 (completed). Textual 8.0.2, watchfiles available.
- **Constraints**: TUI is read-only presentation. No business-logic changes to
  registries. Shared formatter/theme changes are additive and
  backwards-compatible. `.tcss` uses Textual-native tokens only (POL-002).
- **Exit criteria**: Follow-up delta for BacklogRegistry normalisation must be
  scoped before closing DE-053.

## 5. Phase Detail

### Phase 1 — Shared infrastructure

Builds the reusable foundation with no TUI dependency. Three modules:

1. **`formatters/theme.py`** — add `resolve_style(name)` and
   `styled_text(value, style_name)` public API
2. **`formatters/column_defs.py`** — extract column metadata per artifact
   type; refactor existing formatters to consume it
3. **`core/artifact_view.py`** — `ArtifactEntry` dataclass, per-registry
   adapters (incl. BacklogRegistry shim), error-isolation boundary,
   snapshot caching

VT: VT-053-adapter, VT-053-column-defs, VT-053-styled-text

### Phase 2 — TUI core

Builds the Textual app consuming Phase 1 infrastructure:

1. **`tui/app.py`** — `App` subclass, keybindings, CSS_PATH
2. **`tui/browser.py`** — 3-panel screen composition (Horizontal + Vertical)
3. **`tui/widgets/`** — type selector (OptionList, colour-grouped),
   artifact list (DataTable + styled_text), preview panel (Markdown)
4. **`tui/theme.tcss`** — layout chrome, Textual-native tokens only

VT: VT-053-pilot, VT-053-tcss-lint

### Phase 3 — Integration & close

Wires the TUI into the CLI, adds watch/edge-case handling, runs VH:

1. **CLI entry point** — `tui` command in `main.py` with import guard
2. **File watching** — `watchfiles.awatch` integration, per-registry
   invalidation on change events
3. **Edge cases** — `$EDITOR` unset, empty registries, malformed artifacts
4. **VH smoke test** — manual verification gate
5. **Follow-up delta** — scope BacklogRegistry normalisation

VT: VT-053-edge-cases. VH: VH-053-smoke.

## 7. Risks & Mitigations

| Risk | Mitigation | Phase |
| --- | --- | --- |
| Registry errors corrupt TUI screen | Error-isolation boundary in adapter | P01 |
| Column logic drift CLI↔TUI | Shared `column_defs.py` | P01 |
| Startup cost (13 registries × filesystem scan) | Snapshot-on-load caching | P01 |
| Textual major version breakage | Bounded range `>=8.0,<9.0` | P03 |
| watchfiles/asyncio conflict | watchfiles async API compatible; test in P03 | P03 |
| Formatter refactor breaks existing output | VT for existing formatters after column_defs extraction | P01 |
