## 1. Summary

- **Delta**: DE-057 - BacklogRegistry class normalisation and backlog
  subsystem improvements
- **Design Revision**: DR-057 (all decisions resolved incl. adversarial review)
- **Problems / Issues**: ISSUE-009 (partial), ISSUE-026, ISSUE-034, ISSUE-043
  (partial)
- **Desired Outcome**: BacklogRegistry is ADR-009 conformant; artifact_view.py
  shim removed; per-kind status sets defined; operational gaps fixed

## 2. Context & Constraints

- **Current Behaviour**: Function-based backlog API with shim in
  artifact_view.py, no status validation, broken dry-run, missing link
  resolution, greedy --from-backlog flag
- **Target Behaviour**: Class-based BacklogRegistry with eager-cached
  collect/find/iter/filter; per-kind status frozensets; all consumers updated
- **Dependencies**: None (DE-050, DE-053 completed)
- **Constraints**: Preserve module-level function wrappers; no behaviour
  change to default list visibility (DEFAULT_HIDDEN_STATUSES = current);
  `just` must pass
