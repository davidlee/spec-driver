# DR-058 – govern pylint signal and document lint standard

## 1. Executive Summary
- **Delta**: [DE-058](./DE-058.md)
- **Status**: draft
- **Synopsis**: Improve pylint signal by fixing broken test-file ignore globs
  and codify a default Standard for how lint debt should be prioritized and
  discussed.

## 2. Problem & Constraints
- **Current Behaviour**:
  - Nested test files still emit `missing-function-docstring` and
    `protected-access` warnings because the configured glob only matches
    repo-root files.
  - Repo-level warning summaries are noisy enough to invite rationalizations
    like "these warnings already exist elsewhere".
- **Constraints / Guardrails**:
  - Keep repo-level pylint as the authoritative gate.
  - Do not broadly disable high-signal warnings.
  - Keep this delta narrow; broad debt cleanup is deferred.

## 3. Architecture Intent
- **Target Outcomes**:
  - Test-only docstring/protected-access noise is filtered where intended.
  - A default Standard states that new warnings need fixing or explicit
    justification, even when similar debt already exists.
- **Guiding Principles**:
  - Fix configuration bugs before disabling checks.
  - Prioritize warnings by engineering risk, not count.
  - Diagnostic sub-runs do not redefine the repo gate.

## 4. Code Impact Summary
| Path | Current State | Target State |
| --- | --- | --- |

## 6. Supporting Context
- **Related Specs / PRODs**: PROD-003
- **Research**: local inspection of installed `pylint-per-file-ignores` plugin
  shows it resolves patterns via `glob.glob(..., recursive=True)`, making
  recursive globs necessary.

## 7. Design Decisions & Trade-offs
- Fix the recursive glob bug instead of disabling `missing-function-docstring`
  globally.
- Encode review posture in a Standard so future changes are judged against a
  documented norm rather than ad hoc judgment.
