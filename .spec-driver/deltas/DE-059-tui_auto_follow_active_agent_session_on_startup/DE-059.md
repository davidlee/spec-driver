## 1. Summary & Context
- **Implementation Plan**: [IP-059](./IP-059.md)
- **Change Drivers**: DE-054 delivered track view; final UX gap is that the
  human must manually press `t` and find the session. Auto-detection removes
  that friction.

## 2. Motivation

When a human opens the TUI to watch an agent work, they want to see what the
agent is doing *right now*. Currently: open TUI → browser view → press `t` →
scan session list → click session. The agent's session is almost always obvious
from the replay data (most recent, highest event frequency). Auto-detecting and
following it saves multiple steps and makes the "watch the agent" use case
zero-friction.

## 3. Scope & Objectives
- **Primary Outcomes**:
  - On startup, if replay events contain a recently active session, auto-switch
    to TrackScreen and auto-filter to that session
  - "Recently active" = session with events in the last N minutes (reuse
    existing session metadata from SessionList)
  - If no recent session or multiple equally recent: stay on browser (no guess)
  - TrackScreen gains an artifact preview panel (bottom split) — shows the
    document for the currently selected event's artifact, updating as cursor
    moves or new events arrive
- **Dependencies**: DE-054 (completed)

## 4. Out of Scope
- CLI `--session` / `--follow` flags (can be added later if needed)
- Multi-session following
- Session naming / labelling

## 5. Approach Overview
- **System Touchpoints**: `supekku/tui/app.py`, `supekku/tui/track.py`,
  `supekku/tui/widgets/session_list.py`, `supekku/tui/theme.tcss`
- **Key Changes**:
  - Add PreviewPanel to TrackScreen layout (events top, preview bottom)
  - On cursor move / new event with artifact: resolve artifact → show preview
  - After replay, inspect session metadata for a single clearly-active session
  - If found: switch to TrackScreen, set session filter, update SessionList
    selection
  - Detection heuristic: single session with events in last 10 minutes

## 6. Verification Strategy
- **VT-059-01**: Auto-follow triggers when replay has one recent session
- **VT-059-02**: Auto-follow does NOT trigger when no sessions / multiple
  equally recent / no recent activity
- **VT-059-03**: Preview panel updates on event cursor change
- **VH-059-01**: Manual smoke — start agent, open TUI, verify track view
  opens showing agent's session with artifact preview

## 7. Risks & Mitigations
- **Risk**: False positive auto-follow on stale session — *Likelihood*: low
  — *Impact*: low (user can press `t` to go back) — *Mitigation*: conservative
  recency threshold, require single dominant session
