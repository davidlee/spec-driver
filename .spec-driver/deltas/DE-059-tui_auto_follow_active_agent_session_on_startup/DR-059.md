# DR-059 – TUI auto-follow active agent session on startup

Lightweight DR — single design decision, no architectural complexity.

## DEC-059-01: Auto-follow heuristic

**Decision**: After replay, inspect session metadata from `SessionList`.
If exactly one session has events within the last 10 minutes, auto-switch
to TrackScreen and filter to that session. Otherwise, stay on BrowserScreen.

**Rationale**: Conservative — only auto-follow when unambiguous. The 10-minute
threshold matches the existing "active" concept from DEC-054-04. Requiring
a single session avoids guessing when multiple agents are running.

**Implementation**:
1. `SessionList.detect_active_session() -> str | None` — scan `_sessions`,
   return session_id if exactly one has `_parsed_ts` within 10 minutes of now
2. `TrackScreen.auto_follow(session_id)` — set `_session_filter`, replay
   filtered, select session in list
3. `App._start_listener` — after replay + sync, call detect; if found,
   `switch_screen("track")` + `auto_follow`

**Fallback**: No trigger → browser view as before. False negatives are harmless.

### DEC-059-02: TrackScreen artifact preview

**Decision**: Add a `PreviewPanel` to TrackScreen in a vertical split — events
DataTable (2fr) on top, preview (3fr) on bottom. Same `PreviewPanel` widget
used by BrowserScreen. Preview border title shows the artifact file path.

**Trigger**: `DataTable.RowHighlighted` on the track panel. Resolve the
artifact ID from the highlighted row via `TrackPanel.artifact_for_row()`,
then `ArtifactSnapshot.find_entry()` → `preview.show_artifact(entry.path)`.
Also auto-update on new live events that have artifacts (preview follows the
latest touched artifact).

**Rationale**: Reuses existing widget. Matches BrowserScreen's preview
pattern. Gives the human reading context for what the agent is doing, not
just a list of command names.
