## 1. Summary
- **Delta**: DE-059 — auto-follow active agent session on TUI startup
- **Desired Outcome**: TUI detects a single active agent session from replay
  data and auto-switches to TrackScreen filtered to that session.

## 2. Context & Constraints
- **Current**: TUI opens on browser. User must press `t`, find session, click.
- **Target**: If replay shows one clearly active session, start on track view
  filtered to it. Otherwise start on browser as before.
- **Dependencies**: DE-054 (completed)

## 3. Gate Check
- [x] Dependencies completed (DE-054)
- [x] No DR needed (small, additive change to existing infrastructure)
- [x] Test strategy: unit tests for detection heuristic + pilot test + VH

## 5. Tasks (single phase, inline)

1. **TrackScreen: add PreviewPanel** — vertical split: events top (2fr),
   preview bottom (3fr). Reuse existing `PreviewPanel` widget. Wire
   `DataTable.RowHighlighted` → resolve artifact → `show_artifact(path)`.
   Show artifact file path in preview border title.
2. **TrackScreen CSS** — update `theme.tcss` with right-column vertical split
   for track panel + preview.
3. **SessionList.detect_active_session()** — inspect session metadata, return
   session_id if single clearly active session, else None. Heuristic:
   single session with events in last 10 minutes.
4. **TrackScreen.auto_follow(session_id)** — set filter + select session in list
5. **App._start_listener** — after replay, call detect; if found, switch to
   track + auto_follow
6. **Tests** — VT-059-01, VT-059-02, VT-059-03
7. **VH-059-01** — manual attestation

## 6. Risks

| Risk | Mitigation |
| --- | --- |
| False positive on stale session | 10-min recency threshold, require single session |
| Annoyance if user doesn't want track | One `t` press to return to browser |
