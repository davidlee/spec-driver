## 1. Summary
- **Delta**: DE-060 — Claude Code tool-use hooks emit spec-driver artifact events
- **Desired Outcome**: Agent file reads/writes of spec-driver artifacts are
  visible as events in the TUI track view.

## 2. Context & Constraints
- **Current**: TUI only sees `spec-driver` CLI command events.
- **Target**: PostToolUse async hook detects artifact paths and emits events
  to the existing JSONL + socket pipeline.
- **Dependencies**: DE-052 (event infrastructure), DE-054 (track view)

## 3. Gate Check
- [x] DR-060 drafted with design decisions
- [x] Event infrastructure understood (events.py, event_listener.py)
- [x] Claude Code hook API documented (PostToolUse, async, JSON schema)
- [x] Existing hook pattern established (startup.sh)

## 5. Tasks (single phase, inline)

1. **`artifact_event.py`** — self-contained Python script:
   - Read PostToolUse JSON from stdin
   - Extract `file_path` from `tool_input`
   - Classify against spec-driver artifact path patterns
   - On match: build v1 event, append JSONL, send socket
   - On no match or error: exit silently
2. **`.claude/settings.json`** — add PostToolUse hook entry
3. **Tests** — VT-060-01 (path classification), VT-060-02 (non-artifact
   ignored), VT-060-03 (event schema correctness)
4. **VH-060-01** — manual attestation

## 6. Risks

| Risk | Mitigation |
| --- | --- |
| Hook latency | Async mode, fire-and-forget |
| Import path unavailable | Self-contained script, no supekku imports |
| Run dir doesn't exist yet | Create lazily, same as events.py |
