## 1. Summary & Context
- **Implementation Plan**: [IP-061](./IP-061.md)
- **Change Drivers**: IMPR-011 (TUI polish, navigation, and relational display), DE-053 (TUI MVP)

## 2. Motivation

The TUI artifact browser (DE-053 MVP) displays one file per artifact. For
bundle-backed artifacts (deltas, specs with test companions, etc.), the bundle
directory contains additional documents — DR, IP, phase sheets, notes, handover
docs, and other markdown — that are invisible in the TUI. Users must leave the
TUI and use the filesystem to access them.

Target state: when a bundle artifact is selected, the TUI shows a file tree of
the bundle's contents and lets the user navigate between documents without
leaving the browser.

## 3. Scope & Objectives

- **Primary Outcomes**:
  - A `BundleTree` widget showing all files in the selected artifact's bundle directory
  - Tree appears below the TypeSelector when a bundle artifact is selected; TypeSelector
    returns to full height when a non-bundle (single-file) artifact is selected
  - Primary artifact file highlighted by default on selection
  - Standard tree navigation: up/down arrows, left/right to fold/unfold directories, Enter to open in preview
  - `f` keybinding to focus the bundle tree; Tab from tree focuses the artifact table
  - All files shown in tree; non-markdown files shown but preview displays a placeholder
  - Works for any bundle artifact type (deltas, specs, etc.), not delta-specific
  - **Track mode integration**: when an event references a file inside a bundle,
    track preview renders that specific file (not just the primary artifact).
    Navigate-to-browser from track selects the specific file in the BundleTree.
- **Dependencies**: None — builds on DE-053 (completed), DE-060 (event system)

## 4. Out of Scope

- Relational display / cross-artifact navigation (separate IMPR-011 concern)
- Structured rendering of phase data (tasks, gates) — raw markdown only
- Resizable split panes
- Frontmatter/content/both toggle
- Status/tag badges in artifact list

## 5. Approach Overview

- **System Touchpoints**:
  - `supekku/tui/widgets/` — new `BundleTree` widget
  - `supekku/tui/browser.py` — compose BundleTree into BrowserScreen layout
  - `supekku/tui/theme.tcss` — layout rules for tree pane visibility/sizing
  - `supekku/tui/widgets/preview_panel.py` — handle non-renderable files
  - `supekku/scripts/lib/core/artifact_view.py` — expose bundle path info on ArtifactEntry

- **Key Changes**:
  1. Add `bundle_dir: Path | None` to `ArtifactEntry` — populated when the artifact lives in a directory bundle
  2. New `BundleTree` widget using Textual's `Tree` — enumerates all files in bundle dir
  3. BrowserScreen conditionally shows/hides BundleTree based on selected artifact
  4. Wire tree node selection to `PreviewPanel.show_artifact()`
  5. Focus management: `f` → tree, Tab → cycle other panes
  6. Track mode: `_update_preview_for_event()` uses event `argv[1]` (file path) to
     render the touched file directly, not just the primary artifact via ID lookup.
     `action_navigate_artifact` gains an optional `file_path` parameter so track→browser
     navigation can select the specific file in the BundleTree.
  7. Fix `artifact_event.py` to use `hook_input["cwd"]` as relativization base
     instead of `Path.cwd()`, making event paths deterministically repo-root-relative.

## 6. Verification Strategy

- **VT**: Headless pilot tests for tree visibility, navigation, file selection, focus management
- **VA**: Agent verification that tree correctly reflects bundle contents for deltas with DR/IP/phases
- **VH**: Manual smoke test — navigate a delta bundle, open DR, IP, phase sheet, notes
- **Acceptance Criteria**:
  - Selecting a delta shows its bundle tree with all files
  - Selecting a single-file artifact shows no tree (full TypeSelector height)
  - Enter on a tree node renders that file in preview
  - Non-markdown files shown but not rendered

## 7. Risks & Mitigations

- **Risk**: Textual Tree widget styling may not integrate well with existing theme — *Likelihood*: low — *Impact*: low — *Mitigation*: Tree uses same colour tokens as TypeSelector
- **Risk**: Focus management with 4 regions becomes confusing — *Likelihood*: medium — *Impact*: medium — *Mitigation*: `f` keybinding keeps tree access explicit; tree is not in the Tab cycle by default
- **Risk**: Large bundle directories (unlikely) cause slow tree population — *Likelihood*: low — *Impact*: low — *Mitigation*: Bundles are small by convention

## 8. Follow-ups & Tracking

- **Backlog Items**: IMPR-011 — remaining items (relational display, badges, resizable split, etc.)

## 9. Implementation Notes

- Textual `Tree` widget: `from textual.widgets import Tree`
- `ArtifactEntry.path` already points to the primary file; `path.parent` gives the bundle dir when it's a directory bundle vs repo-root-level file
- Bundle detection heuristic: if `entry.path.parent` is a dedicated artifact directory (not a shared root like `.spec-driver/decisions/`), it's a bundle
