# DR-061 – TUI bundle file browser

## 1. Executive Summary
- **Delta**: [DE-061](./DE-061.md)
- **Status**: draft
- **Last Updated**: 2026-03-08
- **Synopsis**: Add a file tree widget to the TUI browser that shows all files
  in a bundle-backed artifact's directory, enabling navigation to DR, IP, phase
  sheets, notes, and other documents without leaving the TUI.

## 2. Problem & Constraints

- **Current Behaviour**: The TUI renders one file per artifact. Bundle
  directories (deltas with DR/IP/phases/notes, specs with test companions)
  contain additional documents that are invisible. Users must leave the TUI
  to access them.

- **Drivers**: IMPR-011 ("embedded sub-entities"), DE-053 (TUI MVP left this
  as follow-up), DE-060 (event system already emits file-level paths for
  bundle sub-files).

- **Constraints**:
  - Must not break existing single-file artifact display
  - Textual-native tokens only in CSS (POL-002)
  - Tree widget must not enter the Tab cycle (4 focusable regions is too many)
  - Generic to any bundle artifact type, not delta-specific
  - No hardcoded prefix lists in generic infrastructure (POL-002 spirit)

- **Out of Scope**:
  - Structured rendering of phase data (tasks/gates) — raw markdown only
  - Relational cross-artifact navigation
  - Resizable split panes

## 3. Architecture Intent

- **Target Outcomes**:
  - Selecting a bundle artifact shows a file tree of its directory contents
  - Users navigate the tree to preview any file in the bundle
  - Track mode previews the specific file an agent touched, not just the primary artifact
  - Track→browser navigation selects the specific file in the bundle tree

- **Guiding Principles**:
  - The tree is an optional enhancement — it appears only for bundles, disappears for single files
  - Data model change is minimal: one field on ArtifactEntry
  - One new message type: `BundleFileSelected` (tree selection → preview is a
    new data flow that doesn't fit existing messages)

## 4. Code Impact Summary

| Path | Current State | Target State |
| --- | --- | --- |
| `artifact_view.py` | `ArtifactEntry` has `path: Path` pointing to primary file | Add `bundle_dir: Path \| None` field; populated by `adapt_record` using the artifact's own ID to detect bundle dirs |
| `widgets/bundle_tree.py` | Does not exist | New `BundleTree(Tree)` widget. Enumerates files in bundle dir, emits `BundleFileSelected` message with `Path`. Symlink-safe, max-depth-limited |
| `browser.py` | `compose()` yields `TypeSelector` directly | Yields `Vertical(TypeSelector, BundleTree)` as `#left-column`. Handles `BundleFileSelected` to update preview. Shows/hides/clears tree on artifact/type selection. Refreshes tree on watch events |
| `theme.tcss` | `#type-selector` is a direct child of `BrowserScreen` | Add `#left-column` container (no border). Bundle tree and type selector each keep their own borders |
| `preview_panel.py` | Reads any path, strips frontmatter, renders markdown | Add guard: if path suffix is not `.md`, show a placeholder message instead of attempting markdown render |
| `track_panel.py` | `_row_artifacts: dict[str, str]` stores artifact ID per row | Add `_row_file_paths: dict[str, str]` to store event file paths per row. New `file_path_for_row()` accessor |

| `app.py` | `action_navigate_artifact(artifact_id)` | Add optional `file_path: str \| None` parameter. When provided, browser selects the artifact then highlights that file in the bundle tree |

## 5. Design Decisions

### DEC-061-01: Bundle detection via artifact ID

Add `bundle_dir: Path | None = None` to the frozen `ArtifactEntry` dataclass.

Detection uses the artifact's own ID, not a hardcoded prefix list:

```python
def _detect_bundle_dir(record_id: str, path: Path) -> Path | None:
  """Return parent if path lives in a directory named after this artifact."""
  parent = path.parent
  if parent.is_dir() and parent.name.startswith(record_id):
    return parent
  return None
```

Called in `adapt_record()`:

```python
record_id = getattr(record, id_attr, "")
bundle_dir = _detect_bundle_dir(record_id, Path(getattr(record, "path", ".")))
```

**Rationale**: This is strictly ID-derived — no magic prefix list, no
registry coupling, works for any artifact type whose bundle directory starts
with its own ID (DE-061-slug, SPEC-042-name, etc.). If a future artifact
type uses a different directory naming convention, the heuristic is extended
in one place.

**Why not registry-aware?** `adapt_record` is generic infrastructure that
normalizes any record model. It should not import or depend on individual
registry implementations. The ID-based check achieves the same result
without coupling.

### DEC-061-02: Left pane layout — no double borders

Wrap `TypeSelector` and `BundleTree` in a borderless `Vertical#left-column`.
Each child keeps its own border:

```
BrowserScreen (horizontal)
├── Vertical#left-column               ← no border, just a layout container
│   ├── TypeSelector#type-selector     ← keeps existing border
│   └── BundleTree#bundle-tree         ← own border, hidden by default
└── Vertical#right-column
    ├── ArtifactList#artifact-panel
    └── PreviewPanel#preview-panel
```

CSS:
```css
#left-column {
  width: 1fr;
  min-width: 20;
  max-width: 30;
  height: 100%;
}

/* Type selector keeps its existing border/styling */
#type-selector {
  height: 100%;                        /* full height when no bundle */
  border: round $surface-lighten-2;
  /* ... existing rules ... */
}

#bundle-tree {
  display: none;
}

#left-column.has-bundle #type-selector {
  height: 1fr;                         /* share space */
}

#left-column.has-bundle #bundle-tree {
  display: block;
  height: 2fr;

  border-title-color: $text-muted;
}

  border: round $accent;
  border-title-color: $accent;

```

`BrowserScreen.on_mount` still sets `#type-selector` border title as before.
`BundleTree` border title is set to "Files" when populated.

The width rules move from `#type-selector` to `#left-column` since the
container now owns the column sizing.

**Rationale**: `#left-column` is purely structural. Each widget owns its own
border and title. No double chrome, no title-setting confusion. Existing
`query_one("#type-selector")` calls remain valid — the ID and widget type
are unchanged.

### DEC-061-03: Focus model — screen-scoped `f`, explicit Tab target

The `f` binding is on `BrowserScreen` (screen scope). This is safe because
Textual `Input` widgets consume character keys before they reach screen
bindings — so `f` never fires while the search input has focus.

```python
# BrowserScreen
BINDINGS = [
  Binding("f", "focus_files", "Files", show=False),
]

def action_focus_files(self) -> None:
  """Focus the bundle tree if visible."""
  tree = self.query_one("#bundle-tree", BundleTree)
  if tree.display:
    tree.focus()
```

Tab from the tree goes explicitly to `#artifact-table`:

```python
# BundleTree
def key_tab(self) -> None:
  """Leave tree, focus artifact table."""
  table = self.screen.query_one("#artifact-table")
  table.focus()
```

Repeated `f` when tree is already focused: no-op (focus is already there).

### DEC-061-04: Track integration — per-row file paths

**Problem identified by review**: `TrackPanel._row_artifacts` only stores
artifact IDs, not file paths. Row highlight/select in `track.py` calls
`panel.artifact_for_row()` which only returns an ID. File-level navigation
would only work for the live preview event, not for table interaction.

**Fix**: `TrackPanel` stores file paths alongside artifact IDs:

```python
class TrackPanel(DataTable):
  def __init__(self, **kwargs) -> None:
    super().__init__(cursor_type="row", **kwargs)
    self._row_counter = 0
    self._row_artifacts: dict[str, str] = {}
    self._row_file_paths: dict[str, str] = {}   # NEW

  def file_path_for_row(self, row_key_value: str) -> str:
    """Return the file path associated with a row key, or empty string."""
    return self._row_file_paths.get(row_key_value, "")

  def append_event(self, event: dict) -> None:
    # ... existing code ...
    argv = event.get("argv", [])
    file_path = argv[1] if len(argv) > 1 else ""
    if file_path:
      self._row_file_paths[row_key] = file_path
    # ... row pruning also cleans _row_file_paths ...
```

`TrackScreen` uses both accessors:

```python
def _on_event_row_highlighted(self, event):
  panel = self.query_one("#track-panel", TrackPanel)
  file_path = panel.file_path_for_row(event.row_key.value)
  if file_path:
    resolved = self._resolve_event_path(file_path)
    if resolved and resolved.is_file():
      preview.show_artifact(resolved)
      preview.border_title = resolved.name
      return
  # Fallback to artifact ID resolution (existing)
  artifact_id = panel.artifact_for_row(event.row_key.value)
  # ...

def _on_event_row_selected(self, event):
  panel = self.query_one("#track-panel", TrackPanel)
  artifact_id = panel.artifact_for_row(event.row_key.value)
  file_path = panel.file_path_for_row(event.row_key.value)
  if artifact_id:
    self.app.action_navigate_artifact(artifact_id, file_path=file_path or None)
```

**Path resolution**: Event `argv[1]` is made relative to `Path.cwd()` in
the hook (`artifact_event.py:87`). The TUI resolves it against
`self._snapshot._root`. These are the same when both run from the repo root,
which is the normal case. However, the hook also receives `cwd` in its JSON
input (`hook_input["cwd"]`) — the correct fix is to use that as the
relativization base in `artifact_event.py` instead of `Path.cwd()`, and to
emit a `repo_root`-relative path. This is a small fix to `artifact_event.py`
that DE-061 should include:

```python
# artifact_event.py — build_event
cwd = kwargs.get("cwd")  # from hook_input
base = Path(cwd) if cwd else Path.cwd()
with contextlib.suppress(ValueError):
  rel_path = str(Path(file_path).relative_to(base))
```

This makes the event path deterministically repo-root-relative regardless of
where the hook process runs.

**Deleted/misresolved files**: If `resolved.is_file()` is False, the fallback
shows the primary artifact via ID lookup. This is a conscious tradeoff: the
user sees *something relevant* rather than nothing, but the previewed file
is not what was touched. Acceptable because deleted-file events are rare and
transient — the next event will likely reference a valid file. The
alternative (showing an error or empty preview) is worse UX for the common
case where deletion is intentional (e.g. removing a draft phase sheet).

### DEC-061-06: Tree refresh and state lifecycle

**Tree refresh on file watch**: When `BrowserScreen.refresh_snapshot()` fires
for a type matching the currently selected artifact, re-resolve the entry
from the refreshed snapshot (in case paths changed), then repopulate the tree
if it's a bundle:

```python
def refresh_snapshot(self, art_type: ArtifactType) -> None:
  # ... existing registry refresh + list refresh ...

  # Re-resolve selected entry from refreshed snapshot
  if (
    self._selected_entry is not None
    and self._selected_entry.artifact_type == art_type
  ):
    fresh = self._snapshot.find_entry(self._selected_entry.id)
    if fresh is not None:
      self._selected_entry = fresh

    # Refresh bundle tree if applicable
    if fresh is not None and fresh.bundle_dir is not None:
      tree = self.query_one("#bundle-tree", BundleTree)
      tree.show_bundle(fresh.bundle_dir, fresh.path)
```

**State clear on type change**: When `on_type_selected` fires, clear the tree
and remove the bundle class:

```python
def on_type_selected(self, message: TypeSelected) -> None:
  # ... existing code ...
  left = self.query_one("#left-column")
  left.remove_class("has-bundle")
  tree = self.query_one("#bundle-tree", BundleTree)
  tree.clear_bundle()
```

**State clear on non-bundle artifact selection**: When `on_artifact_selected`
fires for an artifact without a bundle dir, same clearing. When it fires for
a bundle artifact, populate and show:

```python
def on_artifact_selected(self, message: ArtifactSelected) -> None:
  self._selected_entry = message.entry
  preview = self.query_one("#preview-panel", PreviewPanel)
  preview.show_artifact(message.entry.path)
  preview.border_title = message.entry.id

  left = self.query_one("#left-column")
  tree = self.query_one("#bundle-tree", BundleTree)
  if message.entry.bundle_dir:
    tree.show_bundle(message.entry.bundle_dir, message.entry.path)
    tree.border_title = "Files"
    left.add_class("has-bundle")
  else:
    tree.clear_bundle()
    left.remove_class("has-bundle")
```

**Initial tree selection**: When `show_bundle` populates the tree, cursor
the primary file node so it is visually highlighted. The tree's `cursor_line`
is set to the node index matching `primary_path`.

## 6. BundleTree Widget Design

```python
class BundleFileSelected(Message):
  """Posted when a file is selected in the bundle tree."""
  def __init__(self, path: Path) -> None:
    super().__init__()
    self.path = path


_MAX_TREE_DEPTH = 3  # Prevent runaway recursion


class BundleTree(Tree[Path]):
  """File tree for bundle-backed artifact directories."""

  def __init__(self, **kwargs) -> None:
    super().__init__("Bundle", **kwargs)
    self._bundle_dir: Path | None = None
    self._primary_path: Path | None = None

  def show_bundle(self, bundle_dir: Path, primary_path: Path) -> None:
    """Populate tree with bundle directory contents."""
    self._bundle_dir = bundle_dir
    self._primary_path = primary_path
    self.clear()
    self.root.expand()
    self._populate(self.root, bundle_dir, depth=0)
    self._select_primary()

  def _populate(self, node: TreeNode, directory: Path, depth: int) -> None:
    """Add files and directories, skipping hidden files and symlinks."""
    if depth >= _MAX_TREE_DEPTH:
      return
    entries = sorted(directory.iterdir(), key=self._sort_key)
    for entry in entries:
      if entry.name.startswith(".") or entry.is_symlink():
        continue
      if entry.is_dir():
        branch = node.add(entry.name, data=entry)
        self._populate(branch, entry, depth + 1)
        branch.expand()
      else:
        label = entry.name
        node.add_leaf(label, data=entry)

  def _sort_key(self, path: Path) -> tuple[int, int, str]:
    """Sort: directories first, primary file first among files, then alpha."""
    is_dir = 0 if path.is_dir() else 1
    is_primary = 0 if path == self._primary_path else 1
    return (is_dir, is_primary, path.name.lower())

  def _select_primary(self) -> None:
    """Cursor the primary file node after population."""
    for idx, node in enumerate(self.root.children):
      if node.data == self._primary_path:
        self.cursor_line = idx
        return

  def on_tree_node_selected(self, event: Tree.NodeSelected) -> None:
    """Emit BundleFileSelected when a leaf node is selected."""
    path = event.node.data
    if path is not None and isinstance(path, Path) and path.is_file():
      self.post_message(BundleFileSelected(path))

  def clear_bundle(self) -> None:
    """Reset to empty state."""
    self._bundle_dir = None
    self._primary_path = None
    self.clear()
```

Key safeguards:
- **Symlinks skipped**: `entry.is_symlink()` check prevents escaping the
  bundle or creating cycles
- **Depth limited**: `_MAX_TREE_DEPTH = 3` prevents runaway recursion in
  pathological cases
- **Hidden files skipped**: dotfiles excluded
- **Empty bundle**: tree shows an empty root node. No special placeholder —
  the tree pane itself with "Files" border title is sufficient context
- **Primary file styling**: sorted first (no `*` prefix — sort position
  is the visual signal, avoiding label mutation that could confuse users
  expecting filesystem names). Cursor is placed on the primary node.
