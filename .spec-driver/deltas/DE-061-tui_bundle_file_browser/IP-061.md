## 1. Summary

- **Delta**: DE-061 — TUI bundle file browser
- **Design Revision**: DR-061
- **Desired Outcome**: Bundle-backed artifacts show a file tree in the TUI;
  track mode previews specific touched files; navigation flows from track to
  specific bundle files.

## 2. Context & Constraints

- **Current Behaviour**: TUI shows one file per artifact; bundle sub-docs invisible
- **Target Behaviour**: Per DR-061 — BundleTree widget, browser integration, track integration
- **Dependencies**: DE-053 (completed), DE-060 (completed)
- **Constraints**: No breaking changes to existing artifact display; POL-002 (no magic strings in CSS)

## 6. Testing & Verification Plan

- **Phase 1**: Unit tests for `_detect_bundle_dir`, BundleTree widget (populate, select, clear, sort, symlinks, depth, empty), PreviewPanel non-md guard
- **Phase 2**: Integration tests via Textual pilot — tree visibility, class toggling, preview updates, refresh, state lifecycle
- **Phase 3**: TrackPanel file path storage/pruning, TrackScreen preview with file paths, navigate-with-path, regression on existing track tests
- **Phase 4**: TrackScreen inline tree — compose BundleTree into TrackScreen layout, wire row highlight to tree show/hide/populate, file selection updates track preview
- **Cross-phase**: `just check` at each phase exit
