## 1. Summary & Context

- **Technical Spec(s)**: SPEC-129 – sync installer-component (stub)
- **Implementation Plan**: [IP-062](./IP-062.md)
- **Change Drivers**: Fresh installs get a workflow.toml with only `[tool] exec`, leaving users unaware of ~30 config options across 14 sections.

## 2. Motivation

A fresh `spec-driver init` writes a workflow.toml containing only:

```toml
[tool]
exec = "uv run spec-driver"
```

The full config surface (ceremony mode, cards, policies, sync, contracts, dirs, integration, events, skills, etc.) is invisible. Users and agents cannot discover options without reading source code. There is also no agent memory documenting the config surface, so agents cannot advise on configuration.

## 3. Scope & Objectives

- **Primary Outcomes**:
  1. `install.py` writes a richly-commented workflow.toml showing all default options (commented out) with explanatory comments
  2. New agent memory `mem.reference.spec-driver.workflow-config` documents the full config surface
- **Operational Constraints**: No behavioural change — deep-merge means commented-out lines are inert

## 4. Out of Scope

- Schema validation for workflow.toml (future work)
- Config migration tooling
- Changes to DEFAULT_CONFIG values

## 5. Approach Overview

- **System Touchpoints**: `supekku/scripts/install.py`, memory layer
- **Key Changes**:
  1. Extract default template generation into a function that renders DEFAULT_CONFIG as commented TOML with section explanations
  2. Replace the inline 2-line write in `install.py` with a call to that function
  3. Create memory record for config reference

## 6. Verification Strategy

- **VT**: Test that generated template contains all DEFAULT_CONFIG keys, all commented out
- **VT**: Test that the generated template is valid TOML when comments are stripped
- **VA**: Verify memory record is queryable and accurate
- **Acceptance Criteria**: `just` passes; fresh init produces documented workflow.toml

## 7. Risks & Mitigations

- **Risk**: Template drifts from DEFAULT*CONFIG — \_Likelihood*: medium — _Impact_: low — _Mitigation_: Generate template from DEFAULT_CONFIG rather than hand-writing it
