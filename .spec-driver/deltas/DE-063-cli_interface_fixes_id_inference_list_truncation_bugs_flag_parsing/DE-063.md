## 1. Summary & Context

- **Implementation Plan**: [IP-063](./IP-063.md)
- **Change Drivers**: ISSUE-045, ISSUE-038, ISSUE-041, ISSUE-042

## 2. Motivation

Three categories of CLI friction that affect daily usage:

1. **ID inference** (ISSUE-045): `show` and `view` commands require both
   the subcommand and full ID prefix (`spec-driver show delta DE-061`).
   Users and agents should be able to write `spec-driver show 61` or
   `spec-driver show DE-061` and have the CLI infer the artifact type.
   `find` already supports numeric shorthand — `show`/`view` should too.

2. **List truncation rendering** (ISSUE-038, ISSUE-041): `list --truncate`
   renders the status column with a raw file path (ISSUE-038 for memories)
   or raw enum path instead of the display value (ISSUE-041 for other
   types). Both are display bugs in the truncated table formatter.

3. **Flag parsing** (ISSUE-042): `-ri "pattern"` breaks because `-r`
   consumes `i` as its regex value. Documented in skills as a workaround
   (`-ir` or `-r "pattern" -i`), but fixable at source — either by making
   `-i` a boolean flag that doesn't consume a value, or by documenting the
   limitation more prominently.

## 3. Scope & Objectives

- **Primary Outcomes**:
  - `spec-driver show DE-061` works without specifying `delta` subcommand
  - `spec-driver show 61` infers the artifact type from the numeric ID
  - `spec-driver view DE-061` likewise
  - `list --truncate` renders status as display value, not path/enum
  - `-ri` flag combination works correctly, or fails with a clear error
- **Dependencies**: None

## 4. Out of Scope

- Full CLI verb-noun taxonomy overhaul (ISSUE-030 — separate effort)
- Adding ID inference to `edit`, `complete`, or other mutation commands
  (could follow up if show/view works well)

## 5. Approach Overview

- **System Touchpoints**: `supekku/cli/show.py`, `supekku/cli/view.py`,
  `supekku/cli/common.py` (ID inference), `supekku/scripts/lib/formatters/`
  (truncation bugs)
- **Key Changes**:
  1. ID inference layer: parse bare IDs and numeric shorthand, resolve to
     artifact type via registry lookup
  2. Fix truncated table status rendering
  3. Investigate and fix `-ri` flag parsing

## 6. Verification Strategy

- **VT**: Unit tests for ID inference (various prefixes, numeric, ambiguous)
- **VT**: Tests for truncated list output (status column values)
- **VT**: Tests for flag combination parsing
- **Acceptance Criteria**:
  - `show`/`view` accept bare IDs and numeric shorthand
  - `list --truncate` shows human-readable status values
  - `-ri` either works or produces a clear error

## 7. Risks & Mitigations

- **Risk**: Ambiguous numeric IDs (e.g. `61` matches both DE-061 and
  SPEC-061) – _Likelihood_: low – _Impact_: low – _Mitigation_: print
  disambiguation prompt listing matches
- **Risk**: ID inference changes argument parsing in ways that break
  existing usage – _Likelihood_: low – _Impact_: medium – _Mitigation_:
  inference is additive, existing `show delta DE-061` still works
