# DR-063 – CLI interface fixes: ID inference, list truncation bugs

## 1. Executive Summary

- **Delta**: [DE-063](./DE-063.md)
- **Status**: draft
- **Last Updated**: 2026-03-08
- **Synopsis**: Add type-inferring ID resolution to `show`/`view` commands and
  fix Rich markup corruption in truncated list output.

## 2. Problem & Constraints

### 2a. ID inference (ISSUE-045)

- **Current**: `show` and `view` require explicit entity subcommand:
  `spec-driver show delta DE-061`. The entity type is redundant when the ID
  prefix already encodes it (`DE-` → delta, `ADR-` → adr, etc.).
- **Constraint**: Existing `show delta DE-061` must continue to work unchanged.
  Inference is additive only.
- **Constraint**: Numeric-only IDs (`61`) may match multiple entity types.
  Must disambiguate, not silently pick one.

### 2b. List truncation (ISSUE-038, ISSUE-041)

- **Current**: `truncate_text()` in `table_utils.py` uses `len(text)` on
  strings that contain Rich markup (e.g.
  `[memory.status.active]active[/memory.status.active]` is 47 chars, not 6).
  Truncation cuts into markup tags, producing garbage like `[memory.st...`.
- **Constraint**: Fix must not change the `prepare_row()` contract — callers
  should continue returning markup strings.

### 2c. Flag parsing (ISSUE-042) — wontfix

- `-ri "pattern"` fails because `-r` consumes `i` as its regex value. This is
  standard POSIX short-option clustering behaviour. Already documented in skill
  tips. Close the issue as wontfix.

## 3. Architecture Intent

- **Outcome 1**: `spec-driver show DE-061` and `spec-driver show 61` resolve
  to the correct artifact without specifying the entity subcommand.
- **Outcome 2**: `list --truncate` renders status as human-readable display
  values, not raw markup fragments.
- **Guiding Principles**:
  - Additive: existing `show <type> <id>` commands unchanged
  - Surgical: truncation fix is localized to `table_utils.py`
  - Reuse: leverage existing `_build_artifact_index()` (POL-001)
  - No new abstractions beyond what's needed

#### `PREFIX_TO_TYPE` mapping

```python
PREFIX_TO_TYPE: dict[str, str] = {
    "ADR": "adr",
    "DE": "delta",
    "RE": "revision",
    "POL": "policy",
    "STD": "standard",
    "SPEC": "spec",
    "PROD": "spec",
    "IP": "plan",
    "AUD": "audit",
    "ISSUE": "issue",
    "PROB": "problem",
    "IMPR": "improvement",
    "RISK": "risk",
    "T": "card",
}
```

Notes on coverage:

- `IMPR` (not `IMP`) — matches actual improvement IDs (e.g. `IMPR-001`).
- `RISK` included — matches actual risk IDs.
- `T` for cards — matches `T123-slug` format.
- **Out of scope**: Memory IDs (`mem.pattern.foo`) use dot-separation, not
  dash-separation. The prefix parser splits on `-`, so memory IDs cannot be
  inferred from prefix alone. Users must use `show memory <id>`.
- **Out of scope**: Requirement IDs are composite (`SPEC-009.FR-001`). Bare
  `FR-001` or `NF-001` are not first-class IDs. Users must use
  `show requirement <id>`.

#### `resolve_by_id()` function — using `_build_artifact_index()`

```python
def resolve_by_id(raw_id: str, root: Path) -> list[tuple[str, ArtifactRef]]:
    """Resolve artifact type from a bare ID (prefixed or numeric).

    Uses _build_artifact_index() for O(1) lookup across all registries
    (POL-001: reuse existing cross-registry index from resolve.py).

    Returns list of (artifact_type, ArtifactRef) tuples.
    - Prefixed IDs (e.g. "DE-061"): returns 0 or 1 matches.
    - Numeric IDs (e.g. "61"): checks all prefixed expansions, returns all matches.
    """
```

Logic:

1. Build artifact index via `_build_artifact_index(root)` — single scan of
   all registries, returns `dict[str, tuple[str, str]]` mapping
   `artifact_id → (relative_path, kind)`.
2. **Prefixed ID** (e.g. `DE-061`): Look up `raw_id.upper()` directly in index.
   If found, resolve via `resolve_artifact(kind, raw_id, root)`.
3. **Numeric-only** (e.g. `61`): For each `(prefix, kind)` in `PREFIX_TO_TYPE`,
   construct candidate ID `f"{prefix}-{int(raw_id):03d}"` and check the index.
   Collect all matches.
4. **Card numeric** (e.g. `123`): Also try `T{raw_id}` prefix pattern (cards
   use `T123` not `T-123`).
5. If no matches → return empty list.

Performance: `_build_artifact_index()` does one pass across all registries.
This is the same cost as the existing `resolve links` command. For a CLI
command that runs once, this is acceptable.

#### `InferringGroup` — custom Click Group

```python
import click

class InferringGroup(click.Group):
    """Click Group that infers artifact type from bare IDs.

    When resolve_command() encounters an unknown subcommand name, it stores
    the name as an inferred artifact ID and routes to the hidden "inferred"
    command for resolution.
    """

    def resolve_command(self, ctx, cmd_name):
        # Normal subcommand resolution
        rv = click.Group.resolve_command(self, ctx, cmd_name)
        if rv[1] is not None:
            return rv

        # cmd_name is not a known subcommand — treat as artifact ID
        ctx.obj = ctx.obj or {}
        ctx.obj["inferred_id"] = cmd_name
        return "inferred", self.get_command(ctx, "inferred")
```

#### Typer integration

```python
app = typer.Typer(
    help="Show detailed artifact information",
    cls=InferringGroup,
    no_args_is_help=True,  # preserved — Click handles this before resolve_command
)
```

`no_args_is_help=True` is kept alongside `cls=InferringGroup`. Click processes
`no_args_is_help` in `Group.invoke()` before calling `resolve_command()`, so
`spec-driver show` (no arguments) still prints help.

#### Hidden "inferred" command

Each of `show.py` and `view.py` registers a hidden `"inferred"` subcommand:

```python
@app.command("inferred", hidden=True)
def show_inferred(
    ctx: typer.Context,
    json_output: Annotated[bool, typer.Option("--json", ...)] = False,
    path_only: Annotated[bool, typer.Option("--path", ...)] = False,
    raw_output: Annotated[bool, typer.Option("--raw", ...)] = False,
    root: RootOption = None,
) -> None:
    """Resolve and show an artifact by inferred type."""
    raw_id = ctx.obj["inferred_id"]
    root = root or find_repo_root()
    matches = resolve_by_id(raw_id, root)

    if not matches:
        typer.echo(
            f"Error: no artifact found matching '{raw_id}'",
            err=True,
        )
        raise typer.Exit(EXIT_FAILURE)

    if len(matches) > 1:
        typer.echo(f"Ambiguous ID '{raw_id}' matches:", err=True)
        for kind, ref in matches:
            typer.echo(f"  {ref.id} ({kind})", err=True)
        typer.echo("Specify the type: e.g. 'show delta ...'", err=True)
        raise typer.Exit(EXIT_FAILURE)

    kind, ref = matches[0]
    # Delegate to emit_artifact with resolved ref and output flags
    emit_artifact(ref, json_output=json_output, path_only=path_only, raw_output=raw_output)
```

The hidden command:

- Accepts all output flags (`--json`, `--path`, `--raw`, `--root`) so
  remaining CLI tokens parse correctly after `resolve_command()` consumes
  the ID as `cmd_name`.
- Uses `hidden=True` to avoid appearing in `--help` output.
- For `view.py`, the inferred command calls `open_in_pager(ref.path)` instead
  of `emit_artifact()`.

#### Error messages

- **Prefixed ID not found**: `Error: no artifact found matching 'DE-999'`
- **Numeric ID not found**: `Error: no artifact found matching '999'`
  (The message is the same — the index lookup is exhaustive, so no need to
  list which types were tried.)
- **Ambiguous numeric**: Lists all matches with their types.

#### Shell completion

Out of scope. Typer generates completions from registered subcommands. The
hidden `"inferred"` command won't contribute completions. Completing arbitrary
artifact IDs would require a custom completer, which is a separate concern.

### 4b. Truncation fix

| Path                                            | Current State                                                                                                                                           | Target State                                                                                                                                                     |
| ----------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `supekku/scripts/lib/formatters/table_utils.py` | `add_row_with_truncation()` calls `truncate_text(value, max_width)` on raw markup strings; `truncate_text()` uses `len(text)` which counts markup chars | Use `Text.from_markup(value)` to measure display width. Truncate via `Text.truncate()`. Pass `Text` objects to `table.add_row()`. Remove dead `truncate_text()`. |

Target implementation for `add_row_with_truncation`:

```python
from rich.text import Text

def add_row_with_truncation(table, row_data, max_widths=None, no_truncate=False):
    if no_truncate or max_widths is None:
        table.add_row(*row_data)
        return

    truncated = []
    for i, value in enumerate(row_data):
        max_width = max_widths.get(i, 40)
        text = Text.from_markup(value)
        if max_width <= 3:
            text.truncate(max_width)
        elif len(text) > max_width:
            text.truncate(max_width - 3)
            text.append("...")
        truncated.append(text)

    table.add_row(*truncated)
```

Key points:

- `Text.from_markup()` parses Rich markup into styled spans. Handles plain
  strings (no markup) correctly — returns unstyled `Text`.
- `len(Text)` returns display width (visible characters only).
- `Text.truncate()` preserves styling on remaining characters.
- `Table.add_row()` accepts `Text` objects natively.
- **Edge case guard**: `max_width <= 3` → truncate without ellipsis (no room
  for `...` suffix).
- **Dead code**: `truncate_text()` has no callers after this change (confirmed
  by grep — only `table_utils.py` and `table_utils_test.py` reference it).
  Remove it and update tests accordingly.

## 7. Design Decisions & Trade-offs

### DEC-063-01: Custom Click Group for ID inference

**Decision**: Override `Click.Group.resolve_command()` via `InferringGroup`
rather than adding a top-level catch-all command or using
`invoke_without_command=True`.

**Rationale**: `resolve_command()` is the natural interception point — it fires
before argument parsing, so bare IDs don't conflict with subcommand argument
schemas. Alternative approaches (callback with `invoke_without_command`) would
require changes to every subcommand's argument handling.

**Consequences**: Adds a Click-level customization. Typer's `cls=` parameter
makes this clean. The `InferringGroup` is ~20 lines.

### DEC-063-02: Markup-aware truncation via `rich.text.Text`

**Decision**: Use `Text.from_markup()` + `Text.truncate()` in
`add_row_with_truncation` rather than stripping markup before truncation or
changing the `prepare_row()` contract.

**Rationale**: This is the most surgical fix. `Text` correctly handles display
width, preserves styling on truncated content, and is already a Rich primitive.
No caller changes needed.

**Consequences**: `add_row_with_truncation` now passes `Text` objects (not
`str`) to `table.add_row()`. This is supported by Rich. `truncate_text()`
becomes dead code and is removed.

### DEC-063-03: ISSUE-042 wontfix

**Decision**: Close ISSUE-042 (flag parsing). `-ri` breaking is standard POSIX
short-option clustering: value-taking options consume the rest of the cluster.

**Rationale**: Fixing this would require either non-standard option parsing or
renaming flags. The workaround (`-ir` or `-r "pattern" -i`) is already
documented in skill tips.

### DEC-063-04: Reuse `_build_artifact_index()` for `resolve_by_id`

**Decision**: Use the existing cross-registry index builder from `resolve.py`
instead of iterating individual resolvers for numeric ID resolution.

**Rationale**: `_build_artifact_index()` already scans all registries and
builds an `artifact_id → (path, kind)` mapping. Reusing it gives O(1) lookup
after a single scan, avoids N sequential registry instantiations, and
satisfies POL-001.

**Consequences**: `resolve.py`'s `_build_artifact_index()` may need to become
a public function (rename or re-export). The function is already well-tested
via the `resolve links` command path.

### DEC-063-05: Remove dead `truncate_text()`

**Decision**: Remove `truncate_text()` from `table_utils.py` after migrating
`add_row_with_truncation` to `Text`-based truncation.

**Rationale**: Grep confirms no callers outside `table_utils.py` and its test
file. Keeping dead code violates POL-001.
