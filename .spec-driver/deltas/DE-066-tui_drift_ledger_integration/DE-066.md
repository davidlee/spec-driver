## 1. Summary & Context
- **Dependency**: DE-065 (drift ledger primitive — completed)
- **Implementation Plan**: [IP-066](./IP-066.md)
- **Change Driver**: DE-065 delivered CLI support for drift ledgers but no TUI integration

## 2. Motivation
Drift ledgers are a first-class primitive (DE-065) with CLI commands, but the
TUI browser does not know they exist. Users browsing artifacts in the TUI
cannot discover, select, or preview drift ledgers.

## 3. Scope & Objectives
- **Primary Outcomes**:
  - Drift ledgers appear as a selectable artifact type in the TUI type selector
  - Drift ledger list shows id, name, status, entry count, delta ref
  - Drift ledger preview renders detail view (header + entries)
  - File watcher refreshes drift ledger list on `.spec-driver/drift/` changes
- **Dependencies**: DE-065 (completed)

## 4. Out of Scope
- Drift entry-level navigation (expanding entries within a ledger in the tree)
- Inline editing of drift entries from the TUI
- Filtering entries by status/severity within the preview

## 5. Approach Overview
- **System Touchpoints**: `artifact_view.py`, TUI widgets, `drift/registry.py`
- **Key Changes**:
  - Add `DRIFT_LEDGER` to `ArtifactType` enum (Operational group)
  - Add metadata to `ARTIFACT_TYPE_META`, `_TITLE_ATTR`, `_STATUS_ATTR`, `_ID_ATTR`
  - Add registry factory for `DriftLedgerRegistry`
  - Add `DRIFT_SUBDIR` to `path_to_artifact_type()` subdir map
  - Ensure `ArtifactSnapshot` can load drift ledger records

## 6. Verification Strategy
- **VT-066-artifact-type**: Unit tests for new enum member, metadata, registry factory
- **VT-066-path-mapping**: Unit tests for `path_to_artifact_type()` with drift paths
- **VA-066-tui-e2e**: Manual verification that TUI shows drift ledgers
- **Acceptance**: TUI type selector includes "Drift Ledger", list renders DL-047, preview shows detail

## 7. Risks & Mitigations
- **Risk**: `DriftLedger` model interface differs from other registry records — *Likelihood*: low — *Impact*: low — *Mitigation*: `ArtifactSnapshot` already handles diverse record types via `_TITLE_ATTR`/`_ID_ATTR` mappings
