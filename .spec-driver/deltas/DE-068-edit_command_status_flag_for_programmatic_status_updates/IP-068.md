## 1. Summary

- **Delta**: DE-068 — edit command --status flag for programmatic status updates
- **Specs Impacted**: PROD-013 (CLI Artifact File Access)
- **Problems / Issues**: ISSUE-014 (registry edit commands), ISSUE-009 (status enums)
- **Desired Outcome**: `spec-driver edit <entity> <id> --status <value>` updates frontmatter status in-place with per-kind validation, no editor launch

## 2. Context & Constraints

- **Current Behaviour**: All edit subcommands unconditionally open `$EDITOR`; no `edit drift` subcommand exists; two near-duplicate frontmatter update functions in completion code
- **Target Behaviour**: `--status` flag on all edit subcommands performs validated, non-interactive status update (DEC-068-02); shared primitive consolidates duplicates (DEC-068-01)
- **Dependencies**: None
- **Constraints**: Line-level replacement only (no YAML round-trip); strict validation where enums exist (DEC-068-03); unquoted status values to match convention
