## 1. Summary & Context

- **Change Drivers**: IMPR-006 — complete the formatter migration started by DE-041
- **Design Revision**: [DR-069](./DR-069.md)
- **Implementation Plan**: [IP-069](./IP-069.md)

## 2. Motivation

DE-041 introduced `format_list_table()` and migrated 6 of 10 formatters. The remaining 4 (spec, change, backlog, requirement) still use the old inline pattern, duplicating ~60 lines of boilerplate each. Additionally, cell-level formatting (tags, dates) is repeated 8–10x across all formatters with no reuse, and the `no_truncate` naming inconsistency creates a readability footgun.

With the TUI artifact browser landing (DE-066), cell-level formatting helpers become a shared concern across presentation layers — extracting them now prevents duplication between CLI and TUI.

## 3. Scope & Objectives

- **Primary Outcomes**:
  - All 10 list-table formatters use `format_list_table` — one dispatch pattern
  - Reusable `format_tags_cell` / `format_date_cell` helpers in `cell_helpers.py`
  - `no_truncate` eliminated — `truncate` is the sole parameter name
  - Identical `_calculate_column_widths` consolidated to `governance_5col_widths`
- **Operational Constraints**: Pure refactoring — no functional changes, no CLI interface changes
- **Dependencies**: None; DE-041 is already complete

## 4. Out of Scope

- TUI integration (future work that will benefit from this)
- Changes to `format_list_table` signature
- New formatter tests beyond helpers (existing tests provide regression coverage)

## 5. Approach Overview

- **System Touchpoints**: `supekku/scripts/lib/formatters/`, `supekku/cli/list.py`
- **Key Changes**:
  1. Create `cell_helpers.py` with `format_tags_cell`, `format_date_cell`
  2. Add `governance_5col_widths` to `table_utils.py`, remove `no_truncate` from `add_row_with_truncation`
  3. Migrate 4 remaining formatters to `format_list_table`
  4. Backport cell helpers + consolidated widths into 4 already-migrated formatters
  5. Fix CLI call sites (remove `not` inversion)

## 6. Verification Strategy

- **VT-069-cell-helpers**: Unit tests for new helpers (edge cases: empty, None, single tag, styled output)
- **VT-069-migrations**: All existing formatter tests pass after migration
- **VA-069-no-truncate**: `grep -r no_truncate` returns zero hits in `supekku/`
- **Acceptance**: `just check` passes; no new lint warnings

## 7. Risks & Mitigations

- **Risk**: Spec formatter's dynamic columns (`show_external`, `include_packages`) don't fit the pattern — _Likelihood_: low — _Impact_: low — _Mitigation_: closure approach validated in DR-069; no signature change needed
