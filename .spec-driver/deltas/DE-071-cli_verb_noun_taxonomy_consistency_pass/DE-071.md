## 2. Motivation

The CLI command structure has grown organically. Several commands violate the
established VERB NOUN grammar, hurting discoverability for both humans and
agents. Three niche maintenance commands (`compact`, `resolve`, `backfill`)
sit at the top level despite being rarely used. The `skills` group duplicates
functionality already covered by `install`. The `schema` group uses NOUN VERB
ordering contrary to the rest of the CLI.

Target state: a consistent VERB NOUN grammar with niche operations grouped
under `admin`, redundant commands removed, and agent-facing guidance updated.

## 3. Scope & Objectives

- **Primary Outcomes**:
  1. Remove `skills` command group (redundant with `install`; improve skill-change feedback in `install` output)
  2. Move `schema list` â†’ `list schemas` and `schema show` â†’ `show schema`; remove top-level `schema` group
  3. Create `admin` command group housing `compact`, `resolve links`, `backfill spec`
  4. Evaluate `sync` internals for structural cleanup opportunities (without changing the UX â€” `sync` stays as one command)
  5. Update all agent guidance (memories, skills, docs) to reference new command shapes
- **Operational Constraints**: backward-compat aliases not required; agent guidance update is the migration path
