# DR-071 ŌĆō CLI verb-noun taxonomy: consistency pass

## 1. Executive Summary

- **Delta**: [DE-071](./DE-071.md)
- **Status**: draft
- **Last Updated**: 2026-03-08
- **Synopsis**: Enforce VERB NOUN grammar across the CLI by moving, grouping,
  and removing commands that violate the established convention.

## 2. Problem & Constraints

- **Current Behaviour**: Three grammar violations exist:
  - `skills sync` ŌĆö NOUN VERB, and redundant with `install`
  - `schema list` / `schema show` ŌĆö NOUN VERB group
  - `compact`, `resolve`, `backfill` ŌĆö niche maintenance commands cluttering
    the top-level namespace
- **Drivers**: ISSUE-030
- **Constraints**: No backward-compat aliases required. Agent guidance update
  is the migration mechanism. `sync` UX stays as one command.
- **Out of Scope**: `validate`/`doctor` consolidation, `sync` decomposition,
  ADR for CLI grammar conventions, `sync --memory-links` deprecation.

## 3. Architecture Intent

- **Target Outcomes**:
  1. All commands follow VERB NOUN grammar
  2. Niche maintenance commands grouped under `admin`
  3. No redundant command surfaces
  4. Agent guidance references only current command shapes
- **Guiding Principles**: Consistent grammar > command count reduction.
  Group by frequency of use: core verbs at top level, niche ops under `admin`.

## 4. Code Impact Summary

### 4.1 Remove `skills` group (DEC-071-03)

| Path                             | Current                                      | Target                       |
| -------------------------------- | -------------------------------------------- | ---------------------------- |
| `supekku/cli/main.py`            | Registers `skills` typer group               | Remove registration          |
| `supekku/cli/skills.py`          | Standalone CLI module with `sync` subcommand | Delete file                  |
| `supekku/cli/skills_test.py`     | Tests for `skills sync`                      | Delete file                  |
| `supekku/scripts/install.py:745` | Calls `sync_skills()`, discards result       | Capture result, echo summary |

**Install feedback design**: Reuse the reporting logic currently in
`skills.py:skills_sync()`. Extract a `format_skills_sync_summary(result) -> str`
helper (or just inline ŌĆö it's ~15 lines). Print under an "Skills:" heading
during install output.

### 4.2 Move `schema` into `show`/`list` (DEC-071-01, DEC-071-04)

| Path                         | Current                                      | Target                                                                            |
| ---------------------------- | -------------------------------------------- | --------------------------------------------------------------------------------- |
| `supekku/cli/schema.py`      | Typer app with `list` and `show` subcommands | Keep module as library; remove `app`; export `list_schemas()` and `show_schema()` |
| `supekku/cli/show.py`        | 18 subcommands                               | Add `show schema` calling `schema.show_schema()`                                  |
| `supekku/cli/list.py`        | 16 subcommands                               | Add `list schemas` calling `schema.list_schemas()`                                |
| `supekku/cli/main.py`        | Registers `schema` typer group               | Remove registration                                                               |
| `supekku/cli/schema_test.py` | Tests using `schema list` / `schema show`    | Adapt to `list schemas` / `show schema`                                           |

**Key detail**: The schema functions are self-contained (they use their own
`Console` instance, handle their own error exits). They can be called directly
from `show.py`/`list.py` subcommand handlers as thin wrappers. The existing
private helpers (`_render_markdown`, `_render_json`, etc.) stay in `schema.py`
as a library module.

The `show schema` handler in `show.py` accepts the same arguments
(`block_type`, `--format`) and delegates to `schema.show_schema()`.
The `list schemas` handler in `list.py` accepts `schema_type` and delegates to
`schema.list_schemas()`.

### 4.3 Create `admin` group (DEC-071-02)

| Path                      | Current                                               | Target                                                           |
| ------------------------- | ----------------------------------------------------- | ---------------------------------------------------------------- |
| `supekku/cli/admin.py`    | Does not exist                                        | New Typer group assembling subcommands                           |
| `supekku/cli/compact.py`  | Top-level Typer app                                   | Keep module; `admin.py` imports and registers `compact_deltas`   |
| `supekku/cli/resolve.py`  | Top-level Typer app                                   | Keep module; `admin.py` imports and registers `resolve_links`    |
| `supekku/cli/backfill.py` | Top-level Typer app                                   | Keep module; `admin.py` imports and registers `backfill` command |
| `supekku/cli/main.py`     | Registers `compact`, `resolve`, `backfill` separately | Replace with single `admin` group                                |

**Structure**:

```
spec-driver admin compact [delta-id] [--dry-run]
spec-driver admin resolve links [--dry-run] [--link-mode]
spec-driver admin backfill spec [SPEC_ID] [--dry-run]
```

**Implementation approach**: `admin.py` creates a Typer app and either:

- (a) Imports subcommand functions directly and registers them, or
- (b) Adds the existing sub-Typers as nested groups

Option (a) is simpler for `compact` (single command, flat) but `resolve` has
its own group with `links` as a subcommand. For consistency and to preserve
the existing `admin resolve links` path, use option (b): mount `compact.app`,
`resolve.app`, and `backfill.app` as sub-Typers of admin.

This means:

- `admin compact delta [id]` ŌĆö same depth as today's `compact delta [id]`
- `admin resolve links` ŌĆö same depth as today's `resolve links`
- `admin backfill spec` ŌĆö same depth as today's `backfill spec`

Each existing module keeps its own `app` and tests. `admin.py` is purely
assembly ŌĆö no business logic.

## 6. Design Decisions

### DEC-071-01: Schema commands move into show/list

Schema is an artifact type. `show schema X` and `list schemas` follow the same
grammar as every other artifact. The `schema.py` module stays as a library;
the `app` Typer is removed.

### DEC-071-02: Admin group for niche maintenance commands

`compact`, `resolve`, `backfill` share the character of "infrequent workspace
maintenance." Grouping under `admin` cleans the top-level namespace and provides
a natural home for future operational commands (`config`, `settings`).

### DEC-071-03: Remove skills group entirely

`install` already calls `sync_skills()`. The only missing piece is surfacing
the result. Adding ~15 lines of output formatting to install eliminates an
entire command group.

### DEC-071-04: Schema module becomes a library

`schema.py` retains all rendering logic but drops its Typer `app`. The public
functions `list_schemas()` and `show_schema()` become the stable interface,
called by thin wrappers in `show.py` and `list.py`.
