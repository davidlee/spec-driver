# DR-072 – remove installer backward-compat symlinks

## 1. Executive Summary

- **Delta**: [DE-072](./DE-072.md)
- **Status**: draft
- **Last Updated**: 2026-03-08
- **Synopsis**: Stop the installer from creating backward-compatibility workspace symlinks and make canonical `.spec-driver/` layout the only install output.

## 2. Problem & Constraints

- **Current Behaviour**: `supekku/scripts/install.py` creates root-level compatibility paths after laying down canonical workspace directories. Fresh installs therefore still populate legacy top-level views.
- **Drivers / Inputs**: User request to remove those compatibility symlinks; ADR-006 provides the current rationale and makes the existing behavior explicit.
- **Constraints / Guardrails**: Preserve canonical directory creation and avoid adding new installer flags or migration behavior.
- **Out of Scope**: Existing repository cleanup, migration tooling, and broad documentation/path canonicalization outside installer touchpoints. ADR-006 reconciliation remains in scope because the installer-default behavior is changing.

## 3. Architecture Intent

- **Target Outcomes**:
  - Fresh installs create only the canonical `.spec-driver/` workspace directories.
  - Installer tests and user-facing output reflect the new canonical-only behavior.
- **Guiding Principles**: Keep installer logic explicit and minimal; prefer deleting no-longer-needed compatibility code over introducing toggles or dead abstractions.
- **State Transitions / Lifecycle Impact**: No runtime lifecycle changes beyond installer output.

## 4. Code Impact Summary

| Path                              | Current State                                                           | Target State                                                      |
| --------------------------------- | ----------------------------------------------------------------------- | ----------------------------------------------------------------- |
| `supekku/scripts/install.py`      | Creates compatibility symlinks and advertises them in installer output. | Leaves compatibility paths uncreated and updates output to match. |
| `supekku/scripts/install_test.py` | Encodes compatibility symlink expectations for fresh installs.          | Verifies canonical install layout without compatibility symlinks. |

## 5. Verification Alignment

| Verification              | Impact     | Notes                                                   |
| ------------------------- | ---------- | ------------------------------------------------------- |
| Installer pytest coverage | update     | Update expected fresh-install filesystem shape.         |
| Touched-file lint         | regression | Ensure removed code does not introduce new lint issues. |

## 6. Supporting Context

- **Related Specs**: SPEC-129 – installer component.
- **Related ADRs**: ADR-006 – records the currently implemented compatibility-symlink posture.

## 7. Design Decisions & Trade-offs

- Remove the compatibility helper instead of leaving dead compatibility code behind an unused branch.
- Scope this as an installer behavior change only; do not reinterpret all remaining legacy path references in the same delta.

## 8. Open Questions

- None currently.

## 9. Rollout & Operational Notes

- **Migration / Backfill**: None for existing repos. Fresh installs stop creating compatibility views, and ADR-006 must be updated so the documented default no longer claims installer-provided backward-compat symlinks.
- **Observability / Alerts**: Not applicable.
- **Recovery / Rollback**: Reinstating the helper is straightforward if downstream tooling proves to still require installer-created compatibility views.
