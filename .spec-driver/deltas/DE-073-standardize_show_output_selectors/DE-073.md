## 1. Summary & Context
- **Implementation Plan**: [IP-073](./IP-073.md)
- **Change Drivers**: ISSUE-035 (remaining items), ISSUE-036, plus view pager/alias UX

## 2. Motivation
- `show` commands use bespoke flags (`--raw`, `--json`) with no unified
  content-type selector, increasing cognitive load for agents and users.
- `resolve links` lacks verbose/scoped modes, making missing-target debugging
  slow and whole-corpus-only.
- `view` defaults to pager mode, which is disruptive for quick lookups; pager
  should be opt-in.
- `read` is a natural synonym for `view` that users reach for but doesn't exist.
- Bundling these delivers a coherent CLI UX improvement across output selection,
  viewing ergonomics, and link diagnostics.

## 3. Scope & Objectives
- **Primary Outcomes**:
  - Unified `--content-type/-c` selector (`markdown|frontmatter|yaml`) on
    `show` subcommands; `--raw` preserved as synonym or soft-deprecated.
  - `resolve links --verbose`: report missing targets with containing files.
  - `resolve links --path <file>|--id <mem-id>`: scoped resolution.
  - `view` default output without pager; `--pager/-p` for opt-in paged display
    (prefer `glow --pager`, then `rich --pager`, then `$PAGER`).
  - `read` as a command alias for `view`.
- **Dependencies**: None. DE-045 covers `--links-to`/`--links-depth`/`--body-only`
  independently.

## 4. Out of Scope
- Backlog-item link resolution (ISSUE-034 — separate concern).
- `--links-to` / `--links-depth` / `--body-only` (DE-045).
- `list memories --truncate` column rendering bug (noted in ISSUE-035, separate fix).

## 5. Approach Overview
- **System Touchpoints**: CLI `show` subcommands, `resolve links` command,
  shared CLI option helpers.
- **Key Changes**:
  - Add `--content-type/-c` option to `show` commands (start with `show memory`,
    extend to other artifact types).
  - Add `--verbose` flag to `resolve links`.
  - Add `--path`/`--id` scoping to `resolve links`.
  - Remove default pager from `view`; add `--pager/-p` flag with tool
    preference cascade (`glow --pager` → `rich --pager` → `$PAGER`).
  - Register `read` as a Typer alias for the `view` command group.
- **Migration**: `--raw` remains functional; `view` stops launching a pager
  by default (behavioural change, but the current default is the annoyance).

## 6. Verification Strategy
- VT: unit tests for each new flag/option.
- VA: manual CLI smoke test confirming output formats and scoped resolution.
- **Acceptance Criteria**:
  - At least one `show` subcommand implements `--content-type/-c`.
  - `resolve links --verbose` reports missing targets with file locations.
  - `resolve links --path <file>` resolves only links in the given file.
  - `view` outputs directly without pager by default; `--pager` invokes pager.
  - `read` works identically to `view`.
  - All existing tests pass; lint clean.

## 7. Risks & Mitigations
- **Risk**: `-c` flag conflicts with existing options on some `show` subcommands –
  *Likelihood*: low – *Impact*: medium – *Mitigation*: audit existing `-c` usage
  before implementation.
