## 1. Summary & Context

- **Product Spec(s)**: PROD-008 – Requirements Lifecycle Coherence
- **Technical Spec(s)**: SPEC-122 – `supekku/scripts/lib/requirements`
- **Implementation Plan**: [IP-076](./IP-076.md)
- **Change Drivers**: ISSUE-016 — requirements defined in backlog items are invisible to registry and tooling

## 2. Motivation

- Requirements defined in backlog issues (e.g. ISSUE-013) are properly structured but invisible to `spec-driver list requirements`, coverage tracking, and filtering
- Creates discovery gaps, traceability holes, and coverage blind spots
- Target: unified requirements view across all sources (specs, issues, problems, improvements)

## 3. Scope & Objectives

- **Primary Outcomes**:
  - Requirements registry syncs from backlog sources (FR-016.001)
  - `list requirements` supports `--source-kind` filtering (FR-016.002)
  - Coverage tracking works for backlog-sourced requirements (FR-016.004)
  - Backward compatibility preserved (NF-016.001)
- **Dependencies**: None

## 4. Out of Scope

- Requirement migration tooling (`migrate requirement` command — FR-016.003): deferred to future delta
- Automatic promotion rules, maturity lifecycle dashboards
- Bulk migration

## 5. Approach Overview

- **System Touchpoints**: `requirements/registry.py`, `requirements/models.py`, `backlog/registry.py`, CLI `list.py`
- **Key Changes**:
  - Extend `RequirementsRegistry.sync()` to discover requirements in backlog artifacts
  - Add `source_kind` / `source_type` metadata to requirement records
  - Add `--source-kind` filter to `list requirements` CLI
  - Extend coverage validation to handle all source kinds

## 6. Verification Strategy

- **Requirements Coverage**: VT (unit tests) for each FR; regression suite for NF-016.001
- **Acceptance Criteria**:
  - `spec-driver list requirements` includes backlog-sourced requirements
  - `--source-kind issue` filtering works
  - Existing tests pass (backward compat)

## 7. Risks & Mitigations

- **Risk**: Backlog items have inconsistent requirement formatting – _Likelihood_: medium – _Impact_: low – _Mitigation_: graceful skip with warning for unparseable items
- **Risk**: Registry performance with many backlog sources – _Likelihood_: low – _Impact_: low – _Mitigation_: lazy loading, measure before optimizing

## 8. Follow-ups & Tracking

- **Future Delta**: FR-016.003 (requirement migration command)
- **Backlog Items**: ISSUE-016 — parent issue
