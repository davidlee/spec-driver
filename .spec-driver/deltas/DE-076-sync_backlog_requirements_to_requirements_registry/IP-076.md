## 1. Summary

- **Delta**: DE-076 — Sync backlog requirements to requirements registry
- **Specs Impacted**: SPEC-122 (requirements), SPEC-110 (CLI), PROD-008 (lifecycle coherence)
- **Issues**: ISSUE-016
- **Desired Outcome**: `spec-driver list requirements` shows requirements from backlog items; `--source-kind` filtering works; coverage tracking is uniform across all sources.

## 2. Context & Constraints

- **Current Behaviour**: Requirements registry only syncs from specs and change artifacts. Backlog-sourced requirements are invisible.
- **Target Behaviour**: Registry syncs from backlog items (issues, problems, improvements) using heading+dotted regex. Source provenance tracked via `source_kind`/`source_type` fields.
- **Dependencies**: None. DE-075 (in-progress) touches different code surfaces — no conflict.
- **Constraints**: Pure additive — no breaking changes to existing serialization or workflows.
