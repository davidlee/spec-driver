## 1. Summary & Context

- **Product Spec(s)**: PROD-010 – CLI UX
- **Technical Spec(s)**: SPEC-110 – `supekku/cli`
- **Implementation Plan**: [IP-077](./IP-077.md)
- **Change Drivers**: ISSUE-023 (create commands lack schema hints), ISSUE-043 (`--from-backlog` greedy flag parsing)

## 2. Motivation

- `create` commands output the file path but no guidance on inspecting the YAML schemas within generated templates — users must separately discover `schema show`
- `--from-backlog` greedily consumes the next token as its value, even flags like `--help` — partially mitigated in DE-057 with validation callback, but root cause remains
- Both are CLI discoverability/polish issues that share the `supekku/cli` surface

## 3. Scope & Objectives

- **Primary Outcomes**:
  - All `create` commands output relevant `schema show` commands after creation (ISSUE-023)
  - `--from-backlog` flag parsing fixed at root cause — either boolean flag + positional, or Click eager-option handling (ISSUE-043)
- **Dependencies**: None (DE-057 partial mitigation already landed)

## 4. Out of Scope

- Broader CLI help system redesign
- Schema command enhancements beyond discoverability hints

## 5. Approach Overview

- **System Touchpoints**: `supekku/cli/` create commands, template metadata, `--from-backlog` option definition
- **Key Changes**:
  - Add template-to-schema mapping (central or embedded) so create commands can emit schema hints
  - Add `print_schema_hints()` helper to CLI output after successful creation
  - Restructure `--from-backlog` option to avoid Click's greedy token consumption

## 6. Verification Strategy

- **Acceptance Criteria**:
  - `create delta`, `create phase`, `create revision`, `create audit` all show schema inspection commands
  - `create delta --from-backlog --help` shows help (not "backlog item '--help' not found")
  - Existing create workflows unchanged
- **Planned Artefacts**: VT (CLI output tests), VA (manual smoke test)

## 7. Risks & Mitigations

- **Risk**: Click/Typer limitation may require CLI shape change for `--from-backlog` – _Likelihood_: high – _Impact_: low – _Mitigation_: accept minor breaking change in flag shape if needed, document in release notes
- **Risk**: Template-to-schema mapping maintenance burden – _Likelihood_: medium – _Impact_: low – _Mitigation_: prefer parsing templates over maintaining a parallel mapping
