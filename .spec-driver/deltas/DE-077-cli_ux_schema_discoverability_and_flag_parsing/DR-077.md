# DR-077 – CLI UX: schema discoverability and flag parsing

## 1. Executive Summary

- **Delta**: [DE-077](./DE-077.md)
- **Status**: draft
- **Last Updated**: 2026-03-09
- **Synopsis**: Add schema inspection hints to create command output; fix `--from-backlog` greedy token consumption by converting to boolean flag

## 2. Problem & Constraints

### ISSUE-023: No schema discoverability after create

- **Current Behaviour**: `create` commands output artifact ID and file path, then exit. Users must independently discover `schema show` / `schema list` commands to understand the YAML block schemas embedded in templates.
- **Impact**: New users and agents don't know how to inspect the structure of generated artifacts.

### ISSUE-043: `--from-backlog` greedy flag parsing

- **Current Behaviour**: `--from-backlog` is a `typer.Option` of type `str | None`. Click's parser greedily consumes the next token as the option's value — even when that token is another flag like `--help` or `--spec`.
- **DE-057 mitigation**: Added `_validate_backlog_id` callback that rejects non-ID-shaped values with a clear error. This improves the error message but does not fix the root cause.
- **Impact**: `create delta --from-backlog --help` shows a validation error instead of help.

### Constraints

- **POL-001**: Schema hint logic must be shared, not duplicated per command.
- **POL-002**: Artifact-to-schema mapping must use named constants, not inline strings.
- **Backward compatibility**: The happy-path invocation `create delta --from-backlog ISSUE-123` should continue to work.

### Out of Scope

- Broader CLI help system redesign
- Schema command enhancements beyond discoverability hints
- Changes to `schema show` / `schema list` commands themselves

## 3. Architecture Intent

### Target Outcomes

1. Every `create` command that produces an artifact containing YAML blocks prints relevant `schema show` commands after creation (ISSUE-023)
2. `create delta --from-backlog --help` shows help output, not a validation error (ISSUE-043)

### Guiding Principles

- Pure function for hint generation — no side effects, easily testable
- Minimal CLI shape change for `--from-backlog` — preserve the positional UX
- Centralized artifact→schema mapping — single source of truth

### Design Detail: Schema Hints

```python
# supekku/cli/hints.py

"""Schema hint output for create commands."""

# Artifact kind → list of relevant schema types to suggest
# POL-002: named constant, not inline strings
ARTIFACT_SCHEMA_MAP: dict[str, list[str]] = {
    "delta": ["delta.relationships", "frontmatter.delta"],
    "plan": ["plan.overview", "frontmatter.plan"],
    "phase": ["phase.overview", "phase.tracking"],
    "revision": ["revision.change", "frontmatter.revision"],
    "audit": ["verification.coverage", "frontmatter.audit"],
    "spec": ["spec.relationships", "spec.capabilities", "frontmatter.spec"],
    "prod": ["frontmatter.prod"],
    "policy": ["frontmatter.policy"],
    "standard": ["frontmatter.standard"],
    "memory": ["frontmatter.memory"],
}


def format_schema_hints(artifact_kind: str) -> list[str]:
    """Return schema inspection commands relevant to an artifact kind.

    Pure function: artifact_kind -> list of hint strings.
    Returns empty list for artifact kinds with no registered schemas.
    """
    schemas = ARTIFACT_SCHEMA_MAP.get(artifact_kind, [])
    return [
        f"  spec-driver schema show {s} -f yaml-example"
        for s in schemas
    ]
```

Usage in create commands (example for delta):

```python
hints = format_schema_hints("delta")
if hints:
    typer.echo("\nInspect embedded schemas:")
    for hint in hints:
        typer.echo(hint)
```

### Design Detail: `--from-backlog` Fix

Convert `--from-backlog` from string option to boolean flag. The backlog ID is passed via the existing `name` positional argument.

**Before:**

```
create delta --from-backlog ISSUE-123           # works
create delta --from-backlog ISSUE-123 "title"   # fails (name consumed by --from-backlog)
create delta --from-backlog --help              # fails (--help consumed by --from-backlog)
```

**After:**

```
create delta --from-backlog ISSUE-123           # works: flag=True, name="ISSUE-123"
create delta --from-backlog --help              # works: shows help
create delta "My delta title"                   # works: flag=False, name="My delta title"
```

Logic change in `create_delta_cmd`:

```python
from_backlog: Annotated[
    bool,
    typer.Option("--from-backlog", help="Treat name argument as backlog item ID"),
] = False,

# In command body:
if from_backlog:
    if not name or not _BACKLOG_ID_RE.match(name):
        typer.echo(
            "Error: --from-backlog requires a backlog item ID as the name argument "
            "(e.g., ISSUE-123, PROB-001)",
            err=True,
        )
        raise typer.Exit(EXIT_FAILURE)
    # ... lookup and pre-populate as before, using `name` as backlog ID
```

The `_validate_backlog_id` callback is removed — its purpose is subsumed by the inline validation which now runs without greedy token issues.

## 7. Design Decisions & Trade-offs

### DEC-077-01: Schema hint helper as shared pure function in `cli/hints.py`

- **Rationale**: POL-001 (reuse) demands a single helper, not per-command duplication. Pure function is testable without CLI framework. Separate module keeps `create.py` thin.
- **Alternative rejected**: Embedding the mapping in each create command — violates POL-001/002, harder to maintain.
- **Alternative rejected**: Putting mapping in `schema.py` — `schema.py` is a library module for schema display, not creation-time hints. Different responsibility.

### DEC-077-02: Convert `--from-backlog` to boolean flag

- **Rationale**: Root cause is Click's greedy option parsing for string-typed options. A boolean flag does not consume the next token. The backlog ID naturally fits the existing positional `name` argument — when `--from-backlog` is set, `name` is interpreted as a backlog ID instead of a title. This preserves the exact same invocation syntax for the happy path.
- **Alternative rejected**: `is_eager=True` — still consumes the next token; doesn't fix the issue.
- **Alternative rejected**: Separate `--backlog-id` option — verbose, poor UX, and the greedy parsing problem would just move.
- **Trade-off**: Slight semantic overload of `name` argument (title vs backlog ID depending on flag). Acceptable because the two interpretations are mutually exclusive and the flag makes intent explicit.
