# DR-078 – Add list audits CLI subcommand

## 1. Executive Summary

- **Delta**: [DE-078](./DE-078.md)
- **Status**: draft
- **Synopsis**: Add `list audits` (+ `list audit` alias) to close the sole parity gap among first-class change primitives.

## 2. Problem & Constraints

- **Current Behaviour**: Audits are only listable via `list changes -k audit`. Every other change primitive (`deltas`, `revisions`) and every other first-class artifact has a dedicated `list` subcommand. This is non-discoverable and breaks user expectations.
- **Drivers**: SPEC-110 (consistent CLI surface), PROD-016.FR-010 (first-class command coverage for change primitives).
- **Constraints**: Must follow the established `list_revisions` pattern per POL-001 (maximise reuse). Must conform to SPEC-110.FR-002 (delegate to registries/formatters), FR-005 (consistent flags), FR-006 (table/json/tsv output).
- **Out of Scope**: Audit-specific formatters, `complete audit`, any changes to audit domain model or registry.

## 3. Architecture Intent

- **Target Outcome**: `spec-driver list audits` behaves identically to `list revisions` but loads `ChangeRegistry(kind="audit")`.
- **Guiding Principle**: Zero new abstractions — pure pattern replication.

## 4. Code Impact Summary

| Path                  | Current State               | Target State                                                                                       |
| --------------------- | --------------------------- | -------------------------------------------------------------------------------------------------- |
| `supekku/cli/list.py` | No `list audits` subcommand | Add `list_audits` function (~50 lines) mirroring `list_revisions`; register singular alias `audit` |

## 6. Supporting Context

- **Pattern source**: `list_revisions` (list.py:1347–1446) — identical structure, identical flags, identical formatter.
- **Registry**: `ChangeRegistry(kind="audit")` — already used by `list changes -k audit` and proven.
- **Formatter**: `format_change_list_table()` — already handles audit `ChangeArtifact` records.

## 7. Design Decisions & Trade-offs

- **DEC-078-001**: Clone `list_revisions` pattern exactly rather than attempting to extract a shared helper. Rationale: POL-001 says "start specific, generalize only when patterns proven across 3+ cases." This is the third change-kind list command (`deltas`, `revisions`, `audits`), but the `list_deltas` command has significantly different flags (--paths, --relations, --applies, --plan) so a shared helper would require complex conditional logic. The duplication is mechanical and stable. If a fourth change kind ever appears, extraction would be warranted.
