## 1. Summary
- **Delta**: DE-079 — Implement canonical audit reconciliation contract
- **Specs Impacted**: PROD-008, PROD-011, SPEC-116, SPEC-122, SPEC-125
- **Desired Outcome**: Machine-checkable per-finding disposition in audits, audit-gating in delta completion, and validation surfacing of unresolved audit drift.

## 2. Context & Constraints
- **Current Behaviour**: Audits are advisory with no structured disposition. `complete delta` gates only on coverage. Audit template uses non-canonical statuses and deprecated fields.
- **Target Behaviour**: Per-finding inline disposition with derived closure effects. Qualifying deltas gated by audit completeness. Deprecated fields removed.
- **Dependencies**: DE-055 (design source, in-progress). DE-065 (drift ledger, completed — compatibility target).
- **Constraints**: Additive where possible. Only 2 existing audits (1 needs manual edit for deprecated field removal). Shared change lifecycle from `lifecycle.py`.
