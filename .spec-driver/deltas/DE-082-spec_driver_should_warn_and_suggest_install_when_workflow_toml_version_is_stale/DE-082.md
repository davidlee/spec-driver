## 1. Summary & Context

- **Change Driver**: ISSUE-046 — no warning when package version drifts from installed version

## 2. Motivation

After upgrading the spec-driver package, `workflow.toml` retains the old
`spec_driver_installed_version`. Users get no indication that re-running
`spec-driver install` is needed to update agent docs, skills, and workspace
structure.

## 3. Scope

1. Extract `_get_package_version()` from `install.py` to `core/version.py`
2. Add version-staleness check to `_app_callback` in `main.py` (stderr warning, skip for `install` command)
3. Add version-staleness diagnostic to `diagnostics/checks/config.py`
4. Tests for all three

- Unit tests for version helper, callback warning, and doctor check
