## 1. Summary
- **Delta**: DE-083 - strengthen audit-to-spec reconciliation and spec/revision authorship skills
- **Specs Impacted**: `PROD-002`, `PROD-011`, `SPEC-151`
- **Problems / Issues**: post-audit authorship path is still underspecified even after `DE-055` and `DE-079`; revision authorship remains thin; no explicit spec-authoring branch exists yet
- **Desired Outcome**: a concrete design and implementation path for getting audit findings back into authoritative specs through stronger revision/spec authorship guidance

## 2. Context & Constraints
- **Current Behaviour**: `DE-079` gives audit findings a stronger runtime contract, but the authorship path from those findings into specs/revisions/new specs is still too ambiguous.
- **Target Behaviour**: audit findings should default toward authoritative spec reconciliation rather than loose narrative follow-up.
- **Dependencies**: `DE-055` is the settled doctrine source; `DE-079` is the audit-runtime dependency.
- **Constraints**: design-first; no implementation should start until `DR-083` settles the authorship-skill boundary.
