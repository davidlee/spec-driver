## 1. Summary & Context

- **Product Spec(s)**: PROD-010 (CLI UX), PROD-013 (CLI Artifact File Access)
- **Implementation Plan**: [IP-084](./IP-084.md)
- **Change Drivers**: ISSUE-037 (show commands bypass emit_artifact)

## 2. Motivation

- ISSUE-037 identified 7 show commands with duplicated output dispatch. All 7
  still use inline `bool_flags = sum([...])` or `if sum([...])` patterns
  instead of `emit_artifact`:
  - `show spec`, `show delta`, `show adr`, `show policy`, `show standard`
    (use `bool_flags = sum(...)` pattern)
  - `show requirement`, `show card` (use `if sum(...)` pattern)
- IMPR-006's formatter migration and DRY helpers are already complete.

## 3. Scope & Objectives

- **Primary Outcomes**:
  - All 7 remaining show commands migrated to `emit_artifact`
  - Zero inline output dispatch remaining in `cli/show.py`
- **Operational Constraints**: Pure refactor — no user-visible behaviour change

## 4. Out of Scope

- New output modes — ride on consolidated dispatch after this lands
- Changes to `emit_artifact` internals
- IMPR-006 (already complete)

## 5. Approach Overview

- **System Touchpoints**: `cli/show.py`
- **Key Changes**:
  - Migrate `show spec`, `show delta`, `show adr`, `show policy`, `show standard`
    to `resolve_artifact` + `emit_artifact` pattern
  - Migrate `show requirement` to `resolve_artifact` + `emit_artifact`
  - Migrate `show card` to `emit_artifact` (note: `--anywhere` flag needs
    manual resolve then `emit_artifact` for output dispatch)

## 6. Verification Strategy

- **Acceptance Criteria**:
  - Zero inline `--json`/`--path`/`--raw` dispatch in `cli/show.py`
  - All existing tests pass; no behaviour change in CLI output
  - Linters pass (`just lint`, `just pylint-report`)

## 7. Risks & Mitigations

- **Risk**: `show card` has `--anywhere` flag that changes resolution –
  _Likelihood_: low – _Impact_: low – _Mitigation_: resolve manually with
  `CardRegistry`, construct `ArtifactRef`, then delegate to `emit_artifact`
- **Risk**: `show spec` has `to_dict(repo_root)` needing root param –
  _Likelihood_: low – _Impact_: low – _Mitigation_: capture root in `json_fn` closure
