## 1. Summary
- **Delta**: DE-085 — Surface relation and backlog metadata in CLI and skills
- **Design Revision**: [DR-085](./DR-085.md)
- **Specs Impacted**: PROD-010 (CLI UX), PROD-002 (Delta Creation), PROD-004 (Frontmatter Metadata)
- **Context Inputs**: IMPR-006 (formatter migration), IMPR-011 (TUI relational display)
- **Desired Outcome**: Generic relation query layer with CLI integration; relation/backlog metadata discoverable via `--related-to`, `--relation`, `--refs` flags; `--from-backlog` auto-populates metadata; skills guide consistent usage.

## 2. Context & Constraints
- **Current Behaviour**: Fragmented relation queries (`find_by_implements`, `find_by_informed_by`). No generic query. No `--related-to` flag. `--from-backlog` omits `context_inputs`/`relations`.
- **Target Behaviour**: DR-085 §5 — generic query functions, CLI flags, formatter support, skill updates.
- **Dependencies**: None blocking.
- **Constraints**: `--implements` semantics must not change (DEC-085-002). DecisionRecord and BacklogItem models out of scope.
