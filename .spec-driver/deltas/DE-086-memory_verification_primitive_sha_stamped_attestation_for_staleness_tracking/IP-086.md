## 1. Summary

- **Delta**: DE-086 — Memory verification primitive
- **Design Revision**: [DR-086](./DR-086.md) — fully reviewed, all decisions closed
- **Specs Impacted**: SPEC-132 (memory), SPEC-116 (frontmatter metadata), SPEC-110 (CLI)
- **Issues**: ISSUE-047 (source), ISSUE-020 (general SHA tracking horizon)
- **Desired Outcome**: Memories carry tool-enforced verification SHAs; staleness is mechanically computable; confidence is always present; skills guide verification behaviour.

## 2. Context & Constraints

- **Current Behaviour**: Memories have `verified` date and `scope` but no SHA checkpoint. `confidence` is optional and rarely set. No staleness ranking.
- **Target Behaviour**: Per DR-086 — `verified_sha` (full 40-char) stamped on explicit `--verify`; `confidence` required; `list --stale` with batched git and three-tier sort; skills updated.
- **Dependencies**: None.
- **Constraints**: Backward compatible — existing memories must continue to work.
