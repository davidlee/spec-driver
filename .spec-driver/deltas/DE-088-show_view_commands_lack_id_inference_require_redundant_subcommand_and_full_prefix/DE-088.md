## 1. Summary & Context

- **Implementation Plan**: [IP-088](./IP-088.md)
- **Change Drivers**: ISSUE-045 — CLI usability gap in show/view/edit commands

## 2. Motivation

- ISSUE-045 reported three gaps: (a) ID-only dispatch, (b) bare numeric IDs, (c) `show backlog` alias.
- Preflight testing shows **(a) already works** via `InferringGroup` + hidden `inferred` subcommand.
- Remaining pain: `show issue 045` fails; `show backlog ISSUE-045` fails with extra-arg error.

## 3. Scope & Objectives

- **Primary Outcomes**:
  - Bare numeric IDs normalised per subcommand context: `show issue 045` → `ISSUE-045`
  - `show backlog` / `view backlog` aliases accept any backlog item type (ISSUE, IMPR, PROB, RISK)
  - Same treatment for `edit` if it shares the pattern
- **Dependencies**: None — builds on existing `normalize_id()` and `PREFIX_TO_TYPE` in `common.py`

## 4. Out of Scope

- ID-only dispatch (`show ISSUE-045`) — already works
- Changing `find`, `list`, or other command groups
- Broader CLI restructuring

## 5. Approach Overview

- **System Touchpoints**: `supekku/cli/show.py`, `supekku/cli/view.py`, `supekku/cli/common.py`, possibly `supekku/cli/edit.py`
- **Key Changes**:
  - Add numeric ID normalisation in each artifact subcommand (or a shared decorator)
  - Add `backlog` subcommand to `show` and `view` that dispatches to the correct artifact handler based on prefix

## 6. Verification Strategy

- **Acceptance Criteria**:
  - `show issue 045` → shows ISSUE-045
  - `show backlog ISSUE-045` → shows ISSUE-045
  - `view backlog ISSUE-045` → renders ISSUE-045
  - `show delta 88` → shows DE-088
  - All existing commands unchanged
- **Planned Artefacts**: VT (automated tests), VA (agent CLI verification)

## 7. Risks & Mitigations

- **Risk**: Numeric-only args collide with Typer/Click parsing – _Likelihood_: low – _Impact_: low – _Mitigation_: test edge cases (single-digit, zero-padded, large numbers)
