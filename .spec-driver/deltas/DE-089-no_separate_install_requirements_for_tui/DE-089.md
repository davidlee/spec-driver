## 1. Summary & Context
- **Implementation Plan**: [IP-089](./IP-089.md)
- **Change Driver**: ISSUE-049 — optional TUI extras add friction without meaningful benefit

## 2. Motivation
- The TUI optional-extras split creates install complexity (users must know to `pip install supekku[tui]`) for negligible dependency-size savings.
- Consolidating TUI deps into the main install simplifies onboarding and removes a recurring support friction.

## 3. Scope & Objectives
- **Primary Outcome**: TUI dependencies are part of the default install; no separate extras group required.
- **Operational Constraints**: Must not break existing `supekku[tui]` installs (extras group can remain as a no-op alias if needed).

## 4. Out of Scope
- Restructuring TUI code or changing TUI functionality.

## 5. Approach
- Move TUI dependencies from `[project.optional-dependencies]` to `[project.dependencies]` in `pyproject.toml`.
- Remove or alias the `tui` extras group.
- Update any install documentation or conditional imports that gate on the extras.

## 6. Verification Strategy
- **VT**: `uv run spec-driver --help` and TUI commands work from a clean install without extras.
- **VA**: Confirm `pyproject.toml` no longer requires `[tui]` extras for TUI functionality.

## 7. Risks & Mitigations
- **Risk**: Slightly larger default install — *Likelihood*: certain — *Impact*: low — *Mitigation*: TUI deps are small; acceptable trade-off.
