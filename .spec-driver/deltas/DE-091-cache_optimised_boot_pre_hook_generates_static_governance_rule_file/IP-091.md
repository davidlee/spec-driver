## 1. Summary

- **Delta**: DE-091 — Cache-optimised boot: pre-hook generates static governance rule file
- **Specs Impacted**: SPEC-129 (sync installer-component)
- **Design Revision**: [DR-091](./DR-091.md)
- **Desired Outcome**: All boot-sequence governance content loaded into Claude Code's cacheable prefix via a pre-generated static file, eliminating per-session tool call overhead.

## 2. Context & Constraints

- **Current Behaviour**: Boot skill triggers ~6 Read tool calls + 3 CLI tool calls per session, each busting token cache
- **Target Behaviour**: `startup.sh` runs `spec-driver admin preboot` → writes `.agents/spec-driver-boot.md` → Claude Code loads via `.claude/rules/` symlink in cacheable prefix
- **Dependencies**: None
- **Constraints**: Idempotent; installer manages symlink; preboot shells out to CLI for listings (respects workflow.toml exec)
