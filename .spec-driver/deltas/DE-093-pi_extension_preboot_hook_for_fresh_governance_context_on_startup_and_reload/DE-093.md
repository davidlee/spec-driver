## 1. Summary & Context

- **Related Delta**: DE-092 (parked) — full harness; this is the immediate-value extract
- **Prior Work**: DE-091 — introduced `spec-driver admin preboot` which generates static boot context
- **Change Drivers**: pi is replacing Claude Code as primary agent. Claude Code has a startup hook that runs preboot; pi needs the equivalent to ensure AGENTS.md contains fresh governance context.

## 2. Motivation

- `spec-driver admin preboot` generates `.agents/spec-driver-boot.md` containing active ADRs, policies, standards, memories, and workflow configuration
- This file is inlined into AGENTS.md, which pi loads into the system prompt at startup
- Without a hook, the preboot content may be stale (new policies added, delta completed, memories updated since last run)
- On `/reload`, pi re-reads AGENTS.md — preboot must run before that re-read to pick up changes

## 3. Scope & Objectives

- **Primary Outcomes**:
  - Nix wrapper runs preboot before pi launches (belt)
  - pi extension runs preboot on `session_shutdown` event, which fires before `/reload` re-reads AGENTS.md (suspenders)
  - Preboot also runs on actual exit (harmless — idempotent, ~100ms)
- **Deliverables**:
  - `.pi/extensions/spec-driver-preboot.ts` — pi extension (~20 LOC)
  - Nix flake update — wrapper script for jailed-pi
  - Documentation

## 4. Out of Scope

- Dynamic per-turn context injection (system prompt must be stable for cache efficiency)
- Artifact observation / event hooks (future work)
- Full agent harness (DE-092, parked)

## 5. Approach Overview

- **Belt (Nix wrapper)**: `spec-driver admin preboot "$PWD"` runs on host before `jailed-pi` launches
- **Suspenders (extension)**: hooks `session_shutdown` → `pi.exec("spec-driver", ["admin", "preboot", ctx.cwd])`
- **Why `session_shutdown`**: on `/reload`, the event sequence is:
  1. `session_shutdown` fires → extension runs preboot → **awaited**
  2. `_resourceLoader.reload()` → re-reads AGENTS.md (now fresh)
  3. `session_start` fires
- **Cache safety**: preboot output is deterministic given same project state — if nothing changed, AGENTS.md is byte-identical, cache prefix holds

## 6. Verification Strategy

- **VH**: run `jailed-pi` in a spec-driver project, confirm boot context appears in system prompt
- **VH**: modify a policy, run `/reload`, confirm updated policy appears
- **VH**: confirm preboot on exit doesn't cause errors or noticeable delay

## 7. Risks & Mitigations

- **Risk**: `spec-driver` not on PATH inside jail – _Likelihood_: low (already in extraPkgs) – _Mitigation_: extension checks exec result, warns on failure
- **Risk**: preboot is slow – _Likelihood_: low (~100ms) – _Mitigation_: already measured in DE-091
- **Risk**: extension not loaded (missing .pi/extensions/) – _Likelihood_: medium – _Mitigation_: Nix wrapper is the belt; extension is suspenders only
