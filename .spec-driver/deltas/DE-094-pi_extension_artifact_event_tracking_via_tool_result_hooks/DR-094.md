# DR-094 – artifact event tracking for pi

## Current Behaviour

- `artifact_event.py` (Claude Code `PostToolUse` hook) classifies file paths
  and emits v1 events to JSONL + Unix socket.
- pi sessions produce no artifact events — TUI cannot track them.

## Target Behaviour

- A pi extension hooks `tool_result` events, classifies file paths using the
  same artifact patterns, and emits identical v1 events to the same JSONL log
  and Unix socket.

## Design

### Extension: `spec-driver-artifact-events.ts`

Location: `supekku/pi.extensions/spec-driver-artifact-events.ts`

**Hook**: `tool_result` — fires after every tool execution with typed input.

```typescript
pi.on("tool_result", (event, ctx) => {
  // event.toolName: "read" | "edit" | "write" | ...
  // event.input: { path: string, ... } for file tools
  // event.isError: boolean
});
```

**Relevant tools**: `read`, `edit`, `write` — all have `path` in input.
Other tools (bash, grep, find, ls) don't carry a single canonical file path
or don't mutate artifacts.

### Path classification

Port `_ARTIFACT_PATTERNS` from `artifact_event.py`:

```typescript
const ARTIFACT_PATTERNS: [RegExp, string][] = [
  [/\.spec-driver\/deltas\/(DE-\d+)[^/]*\/phases\//, "phase"],
  [/\.spec-driver\/deltas\/(DE-\d+)[^/]*\/DR-\d+/, "design_revision"],
  [/\.spec-driver\/deltas\/(DE-\d+)[^/]*\/IP-\d+/, "plan"],
  [/\.spec-driver\/deltas\/(DE-\d+)/, "delta"],
  [/\.spec-driver\/tech\/(SPEC-\d+)/, "spec"],
  [/\.spec-driver\/decisions\/(ADR-\d+)/, "adr"],
  [/\.spec-driver\/revisions\/(RE-\d+)/, "revision"],
  [/\.spec-driver\/audits\/(AUD-\d+)/, "audit"],
  [/\.spec-driver\/backlog\//, "backlog"],
  [/\.spec-driver\/product\/(PROD-\d+)/, "product_spec"],
  [/\.spec-driver\/policies\/(POL-\d+)/, "policy"],
  [/\.spec-driver\/standards\/(STD-\d+)/, "standard"],
];
```

### Event emission

Same v1 schema as `artifact_event.py`:

```typescript
{
  v: 1,
  ts: new Date().toISOString(),
  session: null,              // pi doesn't expose session ID yet
  cmd: `artifact.${action}`,  // "artifact.read" | "artifact.edit" | "artifact.write"
  argv: [`artifact.${action}`, relPath],
  artifacts: artifactId ? [artifactId] : [],
  exit_code: 0,
  status: "ok",
  artifact_type: type,
}
```

**JSONL**: `fs.appendFileSync(runDir/events.jsonl, JSON.stringify(event) + "\n")`

**Socket**: Node.js `dgram` lacks `AF_UNIX` support. Socket datagram sent via
`pi.exec("python3", ["-c", ...])` using a one-liner that opens
`AF_UNIX SOCK_DGRAM`, sends the JSON payload, and closes. Async,
fire-and-forget, fail-silent.

### Run directory resolution

`path.join(ctx.cwd, ".spec-driver", "run")` — same as `artifact_event.py`.
`mkdirSync(runDir, { recursive: true })` on first write.

### DEC-094-01: In-process TypeScript, no shell-out

- Zero per-tool-call overhead (no Python startup)
- Pattern list duplicated but stable; cross-referenced via comments

### DEC-094-02: Fail-silent wrapper

- Entire handler wrapped in try/catch
- Extension must never interfere with pi operation

### DEC-094-03: Same v1 event schema

- TUI consumer is already format-agnostic
- No changes needed to the TUI side
