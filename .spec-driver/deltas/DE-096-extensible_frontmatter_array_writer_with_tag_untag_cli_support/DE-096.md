## 1. Summary & Context

Replace line-level frontmatter manipulation with a full YAML round-trip writer using `CompactDumper`. Wire `--tag`/`--untag` on all edit commands. Normalise all existing `.spec-driver` frontmatter.

- **Product Spec**: PROD-010 – CLI UX
- **Prior work**: DE-095 added tags to requirement creation; exposed the writer gap
- **Design Revision**: [DR-096](./DR-096.md)
- **Implementation Plan**: [IP-096](./IP-096.md)

## 2. Motivation

The frontmatter writer only handles scalars. Users must open an editor to manage tags. A YAML round-trip writer with `CompactDumper` gives us:

- Correct, idempotent frontmatter output
- Trivial array operations (`--tag`/`--untag` and future `--owner` etc.)
- Canonical formatting enforced by tooling
- Foundation for pre-commit formatting hooks

## 3. Scope & Objectives

**Phase 1 — Writer + CompactDumper + dump_markdown_file alignment**
**Phase 2 — CLI `--tag`/`--untag` on all edit commands**
**Phase 3 — One-time normalisation of all `.spec-driver` frontmatter**

## 4. Out of Scope

- `--tag` on remaining `create` commands (follow-up)
- Other array fields (`--owner`, `--relation`) — future deltas, same writer
- Pre-commit hook (future — uses this writer)

## 5. Risks & Mitigations

- **Risk**: Normalisation produces large diff – _Mitigation_: One atomic commit, clearly labelled; all output is deterministic and idempotent
- **Risk**: CompactDumper formatting doesn't match expectations – _Mitigation_: Tested against real files; heuristic tuned to match project convention
