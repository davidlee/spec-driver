# DR-096 – Extensible frontmatter array writer with tag/untag CLI

## 1. Executive Summary
- **Delta**: [DE-096](./DE-096.md)
- **Status**: approved
- **Last Updated**: 2026-03-14
- **Synopsis**: Replace line-level frontmatter manipulation with a YAML round-trip writer using a `CompactDumper` that produces deterministic, idempotent, canonical output. Wire `--tag`/`--untag` on all edit commands as first consumer. Normalise all existing `.spec-driver` frontmatter in a one-time commit.

## 2. Problem & Constraints

**Current**: `frontmatter_writer.py` does line-level string replacement for scalar fields. No array support exists. This blocks `--tag`/`--untag` on edit commands and any future array field CLI operations.

**Options evaluated**:
- **A. Line-level mini-parser**: Fragile, reimplements YAML subset badly. Rejected.
- **B. Full YAML round-trip with CompactDumper**: Correct, idempotent, extensible. **Selected.**
- **C. Hybrid (YAML read, line-level write)**: Unnecessary complexity given B works well.

**Key insight**: Formatting churn from YAML round-trip is a one-time cost if we normalise all files and enforce canonical output from all tooling paths.

## 3. CompactDumper Design

Custom `yaml.SafeDumper` subclass with one override:

```python
class CompactDumper(yaml.SafeDumper):
    """YAML dumper producing compact, deterministic frontmatter."""

def _represent_list(dumper, data):
    """Flow-style for short scalar lists, block-style otherwise."""
    if (
        all(isinstance(x, (str, int, float, bool)) for x in data)
        and sum(len(str(x)) for x in data) + 2 * len(data) < 80
        and not any(isinstance(x, dict) for x in data)
    ):
        return dumper.represent_sequence(
            "tag:yaml.org,2002:seq", data, flow_style=True
        )
    return dumper.represent_sequence(
        "tag:yaml.org,2002:seq", data, flow_style=False
    )
```

**Behaviour**:
- `tags: [auth, security]` — flow (short scalars)
- `relations:` with nested dicts — block
- `guiding_principles:` with long strings — block (exceeds 80-char heuristic)
- `aliases: []` — flow (empty)
- Key order preserved (`sort_keys=False`)
- `allow_unicode=True` for `⇄` etc.
- `width=120` for summary line wrapping

**Verified**: Idempotent. `load → dump → load → dump` produces identical output.

## 4. Writer API

### Core primitive

```python
def update_frontmatter(
    path: Path,
    mutator: Callable[[dict[str, Any]], None],
) -> dict[str, Any]:
    """Load frontmatter, apply mutation, write back with canonical formatting.

    Bumps `updated` date automatically. Returns the mutated frontmatter dict.
    """
```

All operations become trivial callers:

```python
# Tag add
update_frontmatter(path, lambda fm: fm.setdefault("tags", []).extend(new_tags))

# Status change
update_frontmatter(path, lambda fm: fm.__setitem__("status", "active"))
```

### Convenience functions

```python
def add_frontmatter_list_items(path, field, items, *, sort=True) -> ListUpdateResult
def remove_frontmatter_list_items(path, field, items) -> ListUpdateResult
```

These call `update_frontmatter` internally and return a result describing what changed.

### Existing functions

`update_frontmatter_status` and `update_frontmatter_fields` are retained (callers exist throughout) but reimplemented as thin wrappers around `update_frontmatter`.

## 5. dump_markdown_file alignment

`dump_markdown_file` in `spec_utils.py` currently uses `yaml.safe_dump`. Switch to `CompactDumper` so all creation paths produce canonical output matching the writer.

## 6. CLI Design

### Shared types in `edit.py`

```python
TagOption = Annotated[
    list[str] | None,
    typer.Option("--tag", "-t", help="Add tag (repeatable)"),
]
UntagOption = Annotated[
    list[str] | None,
    typer.Option("--untag", help="Remove tag (repeatable)"),
]
```

### Shared helper

```python
def _apply_tags(artifact_id, path, tags, untags) -> None:
    # Calls add/remove_frontmatter_list_items, echoes results
```

### Edit commands

Each gains `tag: TagOption = None, untag: UntagOption = None`. When either provided, apply and skip editor (same as `--status`).

## 7. Normalisation

One-time commit: script walks all `.spec-driver/**/*.md` files, loads frontmatter, re-dumps with `CompactDumper`. Body content untouched. This makes the entire corpus canonical so subsequent edits produce minimal diffs.

## 8. Verification

- CompactDumper: idempotency, flow/block heuristic, unicode, date quoting
- update_frontmatter: mutation, updated-date bump, body preservation, error cases
- add/remove list items: add, remove, dedup, sort, create-when-absent, empty-after-remove
- CLI: --tag, --untag, combined, artifact-without-tags
- Existing writer and edit tests adapted or green
- Normalisation: before/after round-trip stability on real files
