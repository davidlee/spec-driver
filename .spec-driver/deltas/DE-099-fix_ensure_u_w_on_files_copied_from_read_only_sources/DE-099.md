## 1. Summary & Context

- **Technical Spec(s)**: SPEC-129 – sync installer-component
- **Implementation Plan**: [IP-099](./IP-099.md) – complete

## 2. Motivation

`shutil.copy` preserves source permissions. When the installer runs from a
read-only source (Nix store, installed wheel), all copied files inherit
read-only permissions. This causes:

1. **Fresh install**: `.claude/settings.json` (and other files) are created
   `0444` — any tool that later tries to write them gets `PermissionError`.
2. **Re-install over prior bad install**: `shutil.copy` cannot overwrite a
   read-only destination, so the next `spec-driver install` also fails.

Observed in the wild on `.claude/settings.json` in a Nix-managed devshell.

## 3. Scope & Objectives

- **Primary Outcomes**:
  - All files written by the installer are user-writable (`u+w`).
  - Re-install over a previously broken (read-only) destination succeeds.
- **Operational Constraints**: Minimal, bug-fix only.

## 4. Out of Scope

- Broader permission hardening (group/other bits).
- File ownership concerns — out of scope for the copy helper.

## 5. Approach Overview

- **System Touchpoints**: `supekku/scripts/lib/file_ops.py`, `supekku/scripts/install.py`, `supekku/scripts/lib/skills/sync.py`
- **Key Changes**:
  1. New helpers in `file_ops.py`: `copy_with_write_permission`, `copytree_with_write_permission`.
  2. All `shutil.copy` calls in the installer replaced with `copy_with_write_permission`.
  3. `shutil.copytree` in skills sync replaced with `copytree_with_write_permission`.
  4. Pre-copy `u+w` on existing dest handles re-install over broken state.

## 6. Verification Strategy

- Unit tests for both helpers (read-only source, read-only dest overwrite).
- Integration test: `TestInstallClaudeConfig.test_readonly_source_produces_writable_dest`.
- Full existing test suites pass (183 tests).

## 7. Risks & Mitigations

- **Risk**: `chmod` fails on filesystem that doesn't support it – _Likelihood_: very low – _Impact_: low – _Mitigation_: same risk as existing hook `chmod`; no regression.
