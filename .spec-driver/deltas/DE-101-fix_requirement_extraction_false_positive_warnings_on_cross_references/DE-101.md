## Summary

The requirement-like line heuristic in `_records_from_content()` uses a broad
`\b(FR|NF)-\d{3}\b` scan to detect potential requirement definitions. When it
finds matches but the formal `_REQUIREMENT_LINE` / `_REQUIREMENT_HEADING` regexes
extract zero, it warns about possible format issues.

Cross-references — `per FR-007`, `(per PROD-004.FR-007)`, heading titles like
`# Implementation per NF-001` — trigger the heuristic but are not definitions.
This produces false-positive warnings on every sync/complete-delta.

## Fix

Tighten the requirement-like detection to only count lines where the FR/NF-xxx
pattern appears in a definition-like position (line-initial after bullet/heading
markers), excluding cross-reference contexts (`per`, parenthetical mentions).
