## 2. Motivation

- Design a workflow/orchestration layer for spec-driver that reduces manual handover work and repeated review bootstrap cost, while preserving the existing artifact model and operator-facing workflow.
- Add a small machine-readable control plane that allows the CLI to detect workflow state reliably, emit structured handovers, support persistent reviewer sessions, recover cleanly, and amortize reviewer bootstrap effort.

## 3. Scope & Objectives

- **Primary Outcomes**:
  - Scope and design the artifact schemas for workflow state, handoff, review bootstrap cache, review findings, and session map.
  - Define the CLI surface for orchestration (handoff emission, review priming/resumption, approval/change transitions, and status).
  - Design the refit of the continuation skill to use structured handoff state.
  - Detail optional bridge sections in `notes.md` or phase sheets.
  - Establish lifecycle and invalidation rules for reviewer state.
  - Propose a migration/compatibility strategy for existing workflows.
- **Operational Constraints**: The design must be additive. Do not replace existing DE/IP/phases/notes.
- **Dependencies**: None.

## 4. Out of Scope

- A general-purpose workflow engine unrelated to spec-driver artifacts.
- A transcript-centric agent memory system.
- A subagent framework that assumes fresh agents for every review round.
- A breaking redesign of DE/IP/phase schemas.
- Implementation of the orchestration layer (this delta is for scoping and design only).

## 5. Approach Overview

- **System Touchpoints**: CLI commands, agent skills, artifact schemas.
- **Key Changes**:
  - Produce a design revision document detailing the approach for adding structured YAML state to a `workflow/` directory.
