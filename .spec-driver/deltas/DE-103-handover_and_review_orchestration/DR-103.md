# DR-103 – implement workflow orchestration control plane

## Design Authority

This delta implements the design defined in
[DR-102](../DE-102-handover_and_review_orchestration/DR-102.md) (approved).

DR-102 is the complete and authoritative design document covering:

- Artifact schemas (§3)
- State machine (§4)
- CLI command surface (§5)
- Continuation skill refit (§6)
- Bridge sections (§7)
- Lifecycle and invalidation rules (§8)
- Global policy configuration (§9)
- Validation rules (§10)
- Migration and compatibility (§11)

This DR does not duplicate that content. It records only the two design
questions that were left open in DR-102 §13 and are now resolved.

## Resolved Open Questions

### 1. Finding ID format (DR-102 §13.1)

**Decision:** Round-scoped (`R3-001`).

Round-scoped IDs are simpler and avoid a global counter. Cross-artifact
aggregation is speculative and belongs to IMPR-020 territory. If aggregation
is ever needed, a composite key (`DE-xxx:R3-001`) is trivial to derive.

### 2. `workflow.toml` location (DR-102 §13.2)

**Decision:** Global (`.spec-driver/workflow.toml`).

Consistent with the existing config pattern where `workflow.toml` is already
the project-level configuration file. Per-delta policy override adds complexity
for a use case that doesn't exist yet and can be introduced later as a
non-breaking addition.
