# DR-107 – Spike: evaluate Pydantic for model layer

## 1. Executive Summary

- **Delta**: [DE-107](./DE-107.md)
- **Status**: draft
- **Last Updated**: 2026-03-22
- **Synopsis**: Investigate replacing hand-rolled `@dataclass` + manual parsing/validation/serialization with Pydantic `BaseModel` across the model layer.

## 2. Problem & Constraints

- **Current Behaviour**: Every model module follows the same pattern: `@dataclass` → `from_frontmatter()` classmethod with field-by-field `fm.get()` + defaults → custom type coercion (`_parse_date`) → manual `to_dict()` with conditional inclusion → ad-hoc validation functions. This is duplicated across memory, drift, docs/python, and any new model type.
- **Drivers**: POL-001 (maximise reuse). The pattern violates DRY — each new entity type requires reimplementing the same parsing/serialization boilerplate.
- **Constraints**: Spike only. Must not break existing tests. Must evaluate real-world corpus compatibility.
- **Out of Scope**: Full migration, CLI changes, config/settings management.

## 3. Architecture Intent

- **Target Outcomes**:
  - Quantified assessment of boilerplate reduction per model
  - Proven behavioural parity on `MemoryRecord` prototype
  - Clear migration path for remaining models
- **Guiding Principles**: Replace convention with declaration. Validation at construction time. One pattern for all models.

## 4. Code Impact Summary

| Path                                        | Current State                                                                    | Target State (prototype)                                |
| ------------------------------------------- | -------------------------------------------------------------------------------- | ------------------------------------------------------- |
| `supekku/scripts/lib/memory/models.py`      | `@dataclass` + `from_frontmatter()` + `_parse_date()` + `to_dict()` (~160 lines) | `BaseModel` with type annotations (~60 lines estimated) |
| `supekku/scripts/lib/drift/models.py`       | `@dataclass` + manual frozenset validation functions                             | Audit only — catalogue for future delta                 |
| `supekku/scripts/lib/docs/python/models.py` | `@dataclass` pattern                                                             | Audit only                                              |

## 5. Spike Deliverables

1. **Model inventory**: catalogue of all `@dataclass` models, their `from_*`/`to_*` methods, and custom validators
2. **Prototype**: `MemoryRecord` as Pydantic `BaseModel` on a branch, passing existing tests
3. **Measurements**: lines before/after, validation coverage gained, import-time delta
4. **Compatibility check**: test against real `.spec-driver/memories/` corpus
5. **Go/no-go recommendation** with migration design sketch if positive
6. **ADR draft** if recommendation is "go"
