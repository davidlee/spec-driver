## 1. Summary

- **Delta**: DE-107 — Pydantic model layer migration
- **Design Revision**: DR-107
- **Specs Impacted**: SPEC-132, SPEC-159, SPEC-116
- **Desired Outcome**: All model files use Pydantic BaseModel. Hand-rolled parsing/serialization boilerplate eliminated. Kind-aware frontmatter validation enabled.

## 2. Context & Constraints

- **Current Behaviour**: 8 model files with duplicated @dataclass + from_frontmatter() + to_dict() + \_parse_date() pattern (~875 lines)
- **Target Behaviour**: Pydantic BaseModel with type annotations, ConfigDict, automatic coercion (~estimated 40-50% line reduction)
- **Dependencies**: DE-106 (completed — PhaseSheet reference)
- **Constraints**: Incremental migration, behavioural parity gated by existing tests
