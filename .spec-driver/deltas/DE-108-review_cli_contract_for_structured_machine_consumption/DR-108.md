# DR-108 – Review CLI contract for structured machine consumption

## 1. Executive Summary

- **Delta**: [DE-108](./DE-108.md)
- **Status**: draft
- **Last Updated**: 2026-03-22
- **Synopsis**: Add structured JSON output mode and granular exit codes to all review CLI commands, plus new `review status` and `review finding` commands, giving autobahn a stable machine-consumable contract for programmatic review orchestration.

## 2. Problem & Constraints

- **Current Behaviour**: Review commands (`prime`, `complete`, `teardown`) emit human-readable text via `typer.echo()`. Exit codes are binary (0/1). No `status` or `finding` commands exist. autobahn cannot programmatically parse output or distinguish error categories.
- **Drivers / Inputs**:
  - autobahn DE-001/DR-001 OQ-001 — needs structured CLI contract for reviewer session orchestration
  - DR-109 — formalised review state machine (bootstrap/judgment/findings) that the CLI must expose
  - DR-102 — review orchestration design, atomic writes, review-index/findings schemas
- **Constraints / Guardrails**:
  - Must not break existing interactive CLI usage (default output unchanged)
  - Must align naming with DR-109 terminology (bootstrap_status, judgment_status, disposition)
  - Output schemas must be stable — changes break autobahn (versioned envelope mitigates)
  - POL-001: maximise code reuse, no new modules where existing ones suffice
  - POL-002: no magic strings — use enums/constants for exit codes, action values, error kinds
- **Out of Scope**:
  - Changing review business logic or state machine (DE-109)
  - Non-review CLI commands (workflow status, phase start, etc.)
  - autobahn-side implementation

## 3. Architecture Intent

- **Target Outcomes**:
  - All review commands accept `--format json` and produce valid, parseable JSON with consistent envelope
  - Granular exit codes distinguish success / unexpected error / precondition failure / guard violation
  - New `review status` command exposes full review state for programmatic queries
  - New `review finding` subcommands enable programmatic finding management (add/dispose/list)
  - Output shapes documented, versioned, and discoverable via existing `show schema` infrastructure
- **Guiding Principles**:
  - Single envelope shape for all commands — autobahn parses one structure
  - Error information always structured in JSON mode — no stderr, no partial output
  - Naming follows DR-109 terminology exactly — no spec-driver-local synonyms
  - Finding-level mutations go through IO layer — atomic write guarantee preserved
- **State Transitions / Lifecycle Impact**: No new states. Commands expose existing DR-109 lifecycle (bootstrap 5-state, judgment 4-state, finding disposition) through structured output.

## 4. Exit Codes

| Code | Constant | Meaning | autobahn mapping |
|------|----------|---------|------------------|
| `0` | `EXIT_SUCCESS` | Completed successfully | Normal result processing |
| `1` | `EXIT_FAILURE` | Unexpected error (bug, crash, unhandled) | `ToolInvocationError` |
| `2` | `EXIT_PRECONDITION` | Precondition not met (missing state, wrong workflow status, invalid input) | `PreconditionError` |
| `3` | `EXIT_GUARD_VIOLATION` | Business rule prevented operation (approval guard, undispositioned findings) | `ToolContractError` / `OperationResult.error` |

Constants added to `supekku/cli/common.py` alongside existing `EXIT_SUCCESS`/`EXIT_FAILURE`.

## 5. JSON Envelope

All review commands in `--format json` mode emit a single JSON object to stdout.

### Success

```json
{
  "version": 1,
  "command": "review.prime",
  "status": "ok",
  "exit_code": 0,
  "data": { ... }
}
```

### Error

```json
{
  "version": 1,
  "command": "review.prime",
  "status": "error",
  "exit_code": 2,
  "error": {
    "kind": "precondition",
    "message": "No workflow state for DE-108"
  }
}
```

**Rules:**
- `version`: always `1`; incremented if envelope shape changes
- `status`: `"ok"` | `"error"`
- `error.kind`: `"precondition"` | `"guard_violation"` | `"validation"` | `"unexpected"`
- In JSON mode, nothing goes to stderr — all output is structured stdout
- Error JSON is emitted before `typer.Exit(code)` — autobahn always gets parseable output
- Human-readable output remains default (no `--format` or `--format table`)

### Envelope helper (DEC-108-001)

Located in `supekku/cli/common.py`. ~30 lines:

```python
def review_json_success(command: str, data: dict) -> dict:
    return {"version": 1, "command": command, "status": "ok", "exit_code": 0, "data": data}

def review_json_error(command: str, exit_code: int, kind: str, message: str) -> dict:
    return {"version": 1, "command": command, "status": "error", "exit_code": exit_code,
            "error": {"kind": kind, "message": message}}

def emit_json_and_exit(payload: dict) -> Never:
    import json, typer
    typer.echo(json.dumps(payload, indent=2))
    raise typer.Exit(payload["exit_code"])
```

## 6. Command Payloads

### 6.1 `review prime`

```json
{
  "delta_id": "DE-108",
  "action": "created",
  "bootstrap_status": "warm",
  "judgment_status": "not_started",
  "review_round": 1,
  "index_path": ".spec-driver/deltas/DE-108-.../workflow/review-index.yaml",
  "bootstrap_path": ".spec-driver/deltas/DE-108-.../workflow/review-bootstrap.md"
}
```

**`action` enum** (DEC-108-006):
- `created` — first-ever prime, no prior review-index.yaml existed
- `rebuilt` — re-prime after teardown, invalid, or stale cache
- `refreshed` — reusable cache with incremental update (warm → warm)

**SHA handling** (DEC-108-002): `staleness.cache_key.head` in review-index.yaml uses full 40-char SHA. Human-readable output uses `short_sha()` for display. `get_head_sha()` already returns full SHA; current code truncates only for display.

### 6.2 `review complete`

```json
{
  "delta_id": "DE-108",
  "round": 2,
  "outcome": "approved",
  "previous_state": "reviewing",
  "new_state": "implementing",
  "findings_path": ".spec-driver/deltas/DE-108-.../workflow/review-findings.yaml",
  "teardown": true
}
```

**`outcome` enum**: `approved` | `changes_requested`

### 6.3 `review teardown`

```json
{
  "delta_id": "DE-108",
  "removed": ["review-index.yaml", "review-findings.yaml", "review-bootstrap.md"]
}
```

### 6.4 `review status` (new command)

```json
{
  "delta_id": "DE-108",
  "bootstrap_status": "warm",
  "judgment_status": "in_progress",
  "workflow_status": "reviewing",
  "active_role": "reviewer",
  "round": 1,
  "findings_summary": {
    "blocking": 2,
    "non_blocking": 1,
    "resolved": 0,
    "waived": 0
  },
  "staleness": {
    "phase_id": "IP-108-P01",
    "head": "abc123def456...",
    "triggers": []
  }
}
```

**Naming** (per DR-109 DEC-109-021):
- `bootstrap_status` — derived 5-state enum (cold/warm/stale/reusable/invalid), top level
- `judgment_status` — command-driven 4-state enum (not_started/in_progress/approved/changes_requested)
- `staleness` — derivation inputs only (phase_id, full head SHA, triggers), not the derived conclusion

**Human output**: table/summary format via `typer.echo()`.

### 6.5 `review finding` subgroup (DEC-108-003)

#### `review finding add`

**Arguments**: `DELTA`, `--severity blocking|non_blocking`, `--title TEXT`, `--summary TEXT` (optional)

```json
{
  "delta_id": "DE-108",
  "finding": {
    "id": "R1-003",
    "round": 1,
    "title": "Missing error handling in review_io",
    "severity": "blocking",
    "status": "open",
    "created_at": "2026-03-22T14:30:00+00:00"
  }
}
```

Returns full server-assigned finding, not just input echo.

#### `review finding dispose`

**Arguments**: `DELTA`, `FINDING_ID`, `--action fix|defer|waive|supersede`, `--authority user|agent`, `--rationale TEXT` (optional), `--backlog-ref TEXT` (optional), `--resolved-at TEXT` (optional)

```json
{
  "delta_id": "DE-108",
  "finding": {
    "id": "R1-001",
    "action": "fix",
    "authority": "user",
    "previous_status": "open",
    "new_status": "resolved"
  }
}
```

Guard enforcement per DR-109 §3.4: waive requires user authority + rationale; defer requires user authority + backlog_ref; fix accepts resolved_at SHA.

#### `review finding list`

**Arguments**: `DELTA`, `--round N` (optional filter)

```json
{
  "delta_id": "DE-108",
  "findings": [
    {
      "id": "R1-001",
      "round": 1,
      "title": "Missing error handling",
      "status": "open",
      "severity": "blocking",
      "disposition": null
    }
  ]
}
```

- Default: all findings across all rounds (what `can_approve()` needs) (DEC-108-005)
- `--round N`: filter to single round
- Each finding carries its `round` number

## 9. Design Decisions

| ID | Decision | Rationale |
|---|---|---|
| DEC-108-001 | Envelope helper in `common.py`, not a new module | POL-001 — reuse existing common utilities; ~30 lines |
| DEC-108-002 | Full SHA in JSON, short SHA in human output | autobahn needs unambiguous cache keys; human display stays readable |
| DEC-108-003 | `review finding` as subgroup with `add`/`dispose`/`list` | Mirrors DR-109 §3.4 disposition model; distinct operations, not overloaded |
| DEC-108-004 | Finding-level mutation helpers in `review_io.py` | IO layer as single authority for findings; atomic write guarantee preserved |
| DEC-108-005 | `--round N` optional filter on `finding list`, default all-rounds | `can_approve()` needs cross-round view; per-round useful for humans |
| DEC-108-006 | `action` enum: `created`/`rebuilt`/`refreshed` | `created` = first-ever prime (cold), `rebuilt` = re-prime after teardown/invalid/stale, `refreshed` = reusable with incremental update. autobahn uses to decide session reset |
| DEC-108-007 | Error JSON to stdout before `typer.Exit(code)` | Guarantees parseable output on every invocation; autobahn never gets partial/empty stdout |
| DEC-108-008 | Exit codes 0/1/2/3 in `common.py` | Available to all future commands; not review-specific constants |
