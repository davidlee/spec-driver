## 1. Summary

- **Delta**: [DE-108](./DE-108.md)
- **Design Revision**: [DR-108](./DR-108.md)
- **Specs Impacted**: PROD-010 (CLI UX), PROD-011 (Implementation Execution Workflow)
- **Desired Outcome**: All review CLI commands produce structured JSON output via `--format json` with versioned envelope, granular exit codes, and two new command surfaces (`review status`, `review finding`). autobahn can programmatically orchestrate reviewer sessions.

## 2. Context & Constraints

- **Current Behaviour**: 3 review commands (prime/complete/teardown) with human-only output, binary exit codes
- **Target Behaviour**: 6 commands (+ status, finding add/dispose/list) with `--format json`, exit codes 0/1/2/3, consistent envelope
- **Dependencies**: DR-109 terminology (bootstrap_status, judgment_status); existing review_io.py / review_state_machine.py
- **Constraints**: Must not break existing interactive CLI; POL-001 (reuse); POL-002 (no magic strings); STD-002 (lint clean)

## 6. Testing & Verification Plan

- **Updated Suites**: `supekku/cli/workflow_review_test.py`, `supekku/scripts/lib/workflow/review_io_test.py`
- **New Cases**:
  - JSON envelope structure validation (version, command, status, exit_code, data/error)
  - Each exit code path: 0 success, 1 unexpected, 2 precondition, 3 guard violation
  - `review status` — bootstrap_status, judgment_status, findings_summary, staleness inputs
  - `review finding add` — ID assignment (R{round}-{seq}), cold start, severity
  - `review finding dispose` — disposition, status derivation, authority guards, not-found error
  - `review finding list` — all-rounds default, --round N filter
  - Full SHA in JSON vs short SHA in human output
  - Stderr silence in JSON mode
- **Rollback Plan**: N/A — additive changes, existing behaviour preserved

## 7. Risks & Mitigations

| Risk                                                               | Mitigation                                                              | Owner |
| ------------------------------------------------------------------ | ----------------------------------------------------------------------- | ----- |
| Finding-level IO helpers break existing round-level write patterns | Tests for both round-level and finding-level IO; atomic write preserved | Dev   |
| `--format` convention diverges from `--json` on create commands    | DEC-108-009 documents the choice; review subgroup is self-consistent    | Dev   |
| Stderr leakage in JSON mode from early validation                  | DEC-108-010 pattern: early format check + try/except routing            | Dev   |
