## 1. Summary & Context

- **Product Spec(s)**: PROD-012 (Code Contract Documentation)
- **Implementation Plan**: [IP-110](./IP-110.md)
- **Change Drivers**: autobahn DE-001/DR-001 OQ-002 — autobahn needs canonical workflow YAML fixtures and enum values exported by spec-driver for cross-project contract testing

## 2. Motivation

- autobahn defines provisional pydantic models for spec-driver's workflow YAML schemas (`supekku:workflow.*`)
- These models need validation against authoritative JSON Schema — autobahn can call `spec-driver show schema workflow.* --format=json-schema` at test time
- 9 of 15 registered block schemas currently lack `--format=json-schema` support: 7 workflow schemas (have `BlockMetadata` but aren't wired into the display path) and 2 spec schemas (have no `BlockMetadata` at all)
- The root cause is a hardcoded lookup dict in `schema.py` that was never updated for workflow schemas
- Enum values are already available via `show schema enums.*` — no additional work needed

## 3. Scope & Objectives

- **Primary Outcomes**:
  - All 15 registered block schemas support `show schema <type> --format=json-schema` (JSON Schema Draft 2020-12)
  - All 15 support `--format=yaml-example` with representative examples
  - Hardcoded metadata lookup dict in `schema.py` eliminated — metadata attached directly to `BlockSchema` registry records
  - autobahn can call `spec-driver show schema workflow.state --format=json-schema` and receive valid schema
- **Operational Constraints**: No new commands, no committed fixture files — the tool IS the contract
- **Dependencies**:
  - None blocking. DE-109 (review state machine) enums already present in `workflow_metadata.py`.

## 4. Out of Scope

- Shared Python package between repos
- Bulk export command (autobahn calls `show schema` per-type)
- Pydantic model export (DE-107 concern)
- Workflow schema validation engine wiring (separate from documentation)
- autobahn-side contract test implementation

## 5. Approach Overview

- **System Touchpoints**: `schema_registry.py`, `schema.py`, `workflow_metadata.py`, `relationships.py`, new `spec_metadata.py`
- **Key Changes**:
  - Add `metadata: BlockMetadata | None` to `BlockSchema` dataclass
  - Wire `metadata=` at all registration sites
  - Replace hardcoded dict in `schema.py` with `schema.metadata` lookup
  - Create `spec_metadata.py` with `BlockMetadata` for `spec.relationships` and `spec.capabilities`
  - Add `examples` to all 7 workflow `BlockMetadata` objects

## 6. Verification Strategy

- **Acceptance Criteria**:
  - All 15 `show schema <type> --format=json-schema` produce valid JSON Schema
  - All 15 `show schema <type> --format=yaml-example` produce valid YAML
  - Existing 6 types produce output consistent with pre-change behaviour
  - `spec-driver list schemas` unchanged
