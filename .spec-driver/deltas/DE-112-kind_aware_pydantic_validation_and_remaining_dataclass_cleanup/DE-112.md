## 1. Summary

Three items:

1. Convert `FrontmatterValidationResult` and `Relation` to Pydantic (last 2 dataclasses)
2. Wire Pydantic models into validator for kind-aware frontmatter validation (memory, backlog, drift)
3. Fix historical validation noise (AUD-012 vocabulary, legacy phase frontmatter)

## 2. Motivation

DE-107 created Pydantic models for all artifact types. They're used for construction but not yet for validation — `spec-driver validate` only checks phases via PhaseSheet. Wiring the models into the validator catches malformed frontmatter earlier and completes the Pydantic adoption story.

The validation noise (11 errors, ~14 warnings from historical data) obscures real issues.

## 3. Scope

- Convert 2 frozen dataclasses in `frontmatter_schema.py`
- Add 3 validation methods to `WorkspaceValidator` + traversal wiring
- Fix AUD-012 status vocabulary, legacy phase frontmatter gaps, IP-109 status format

## 4. Out of Scope

- Per-invocation validation hooks (future — designed for extraction)
- Spec model changes (DEC-111-001: stays as dataclass)

## 5. Verification

- `spec-driver validate` shows fewer errors/warnings after cleanup
- New validation catches intentionally malformed test fixtures
- Existing validator tests pass
- All model tests pass
