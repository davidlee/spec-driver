## 1. Summary

- **Delta**: DE-112 — Kind-aware Pydantic validation and remaining dataclass cleanup
- **Specs Impacted**: SPEC-116 (frontmatter schema), SPEC-132 (validation), SPEC-159 (models)
- **Backlog**: IMPR-024 (kind-aware frontmatter validation)
- **Desired Outcome**: Validator catches malformed memory/backlog/drift frontmatter via existing Pydantic models; last 2 dataclasses converted; historical validation noise eliminated.

## 2. Context & Constraints

- **Current Behaviour**: Validator only validates phases via PhaseSheet. Memory, backlog, drift frontmatter unchecked. `FrontmatterValidationResult` and `Relation` are the last frozen dataclasses in the model layer. `spec-driver validate` emits 11 errors + ~14 warnings from historical data issues.
- **Target Behaviour**: Kind-aware validation for all artifact types with Pydantic models. Clean validate output reflecting real issues only.
- **Dependencies**: DE-107 (Pydantic models exist), DE-111 (PhaseSheet validation pattern)
- **Constraints**: POL-001 (reuse existing models), STD-002 (lint compliance). Validation is permissive (warnings, not errors).
