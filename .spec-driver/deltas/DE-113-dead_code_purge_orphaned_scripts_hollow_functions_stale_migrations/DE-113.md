## 1. Summary & Context

- **Technical Spec(s)**: SPEC-110 (supekku/cli), SPEC-112 (supekku/scripts/cli)
- **Change Drivers**: Code quality audit identifying 12+ orphaned scripts, hollow display functions, stale migration code, and a duplicate doc generator.
- **Governance**: POL-001 (maximise code reuse, minimise sprawl)

## 2. Motivation

The unified CLI (`supekku/cli/`) fully supersedes the original `supekku/scripts/*.py` argparse wrappers, but 12+ orphaned scripts remain in-tree. Several contain hollow function bodies (14 `pass` statements in `complete_delta.py` where output should appear; 17 in `decision_registry.py`). A one-time migration script (`migrate_stub_status.py`) bypasses project frontmatter primitives. Two doc generators exist where one is unreachable. This dead weight confuses the `scripts/` namespace, inflates test surface, and misleads contributors.

## 3. Scope & Objectives

- **Primary Outcomes**:
  - Delete 12 orphaned script wrappers: `list_specs.py`, `list_changes.py`, `list_deltas.py`, `change_registry.py`, `create_delta.py`, `create_new_spec.py`, `create_revision.py`, `create_requirement.py`, `decision_registry.py`, `validate_workspace.py`, `requirements.py`, `backlog/{append,create_entry}.py`
  - Delete orphaned tests: `list_specs_test.py`, `list_changes_test.py`
  - Delete unreachable `scripts/cli/ast_doc_generator.py` (duplicate of `deterministic_ast_doc_generator.py`)
  - Delete `migrate_stub_status.py`
  - Delete `validate_revision_blocks.py` (orphaned argparse wrapper over `lib/blocks/revision`)
  - Rename `registry_migration.py` → `registry_v2.py` (live code misnamed as migration)
  - Move `complete_delta.py` business logic into `scripts/lib/changes/completion.py`; update `cli/complete.py` import; delete or reduce script to thin shim
  - Remove hollow `pass`-body display functions or replace with real implementations
  - Clean stale TODOs (`blocks/__init__.py`)
- **Operational Constraints**: Zero behaviour change. All tests must continue to pass.
- **Dependencies**: None. This is the first delta in the quality remediation sequence.

## 4. Out of Scope

- Refactoring `complete_delta.py` business logic (just relocate it)
- Implementing the hollow display functions if they're no longer needed (delete instead)
- Structural splits of remaining god files (DE-114)

## 5. Approach Overview

- **System Touchpoints**: `supekku/scripts/`, `supekku/cli/complete.py`, test files
- **Key Changes**:
  - Phase 1: Delete orphaned scripts and their tests. Verify no imports break.
  - Phase 2: Move `complete_delta.py` logic to lib. Update CLI import.
  - Phase 3: Rename `registry_migration.py`. Update all importers.
  - Phase 4: Clean hollow functions and stale TODOs.

## 6. Verification Strategy

- **Acceptance Criteria**:
  - `just check` passes (full test suite + lint)
  - `grep -rn` confirms no imports of deleted modules
  - No `pass`-only function bodies remain in active code paths
  - `registry_migration` name no longer appears in imports

## 7. Risks & Mitigations

- **Risk**: An orphaned script is actually invoked externally (CI, user scripts) – _Likelihood_: low – _Impact_: medium – _Mitigation_: Verify pyproject.toml entry points; grep for script names in `.agents/`, CI configs, and Justfile.
- **Risk**: `complete_delta.py` relocation breaks `cli/complete.py` – _Likelihood_: low – _Impact_: low – _Mitigation_: Import path update is mechanical; tests cover the CLI path.

## 8. Follow-ups & Tracking

- **Dependency of**: DE-114 (god-file splits benefit from smaller surface)
- **Sequence**: Sprint 1 of quality remediation. Do first.
