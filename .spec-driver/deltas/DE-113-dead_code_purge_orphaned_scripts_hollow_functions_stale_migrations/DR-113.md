# DR-113 – Dead code purge: orphaned scripts, hollow functions, stale migrations

## 1. Executive Summary

- **Delta**: [DE-113](./DE-113.md)
- **Status**: accepted
- **Last Updated**: 2026-03-22
- **Synopsis**: Delete 15 orphaned script wrappers, relocate `complete_delta` business logic to `scripts/lib/changes/completion.py`, rename `registry_migration.py` → `registry_v2.py`, clean stale TODO.

## 2. Problem & Constraints

- **Current Behaviour**: 15 top-level scripts in `supekku/scripts/` have zero live importers — they were superseded by the unified CLI (`supekku/cli/`). `complete_delta.py` has live business logic but also 7 hollow `pass`-body display functions. `registry_migration.py` is live production code misnamed as a migration. An unreachable duplicate doc generator exists alongside the canonical one.
- **Drivers**: STD-004 (orphan prevention), POL-001 (minimise sprawl), code quality audit.
- **Constraints**: Zero behaviour change. All tests must continue to pass.
- **Out of Scope**: Implementing hollow display functions (delete instead), refactoring `complete_delta` logic (just relocate), structural splits of remaining god files (DE-114).

## 3. Architecture Intent

- **Target Outcomes**:
  - `supekku/scripts/*.py` contains only scripts with live entry points or CLI imports (STD-004 compliant)
  - `complete_delta` business logic lives in `scripts/lib/changes/completion.py` alongside `complete_revision`
  - `registry_migration.py` renamed to `registry_v2.py` to reflect its actual role
- **Guiding Principles**: Pure deletion where possible. Mechanical relocation where not. No new abstractions.

### 4b. Relocation: `complete_delta.py` → `scripts/lib/changes/completion.py`

- **Current**: `complete_delta.py` (612 lines) contains business logic + argparse wrapper + 7 hollow display functions.
- **Target**: Move `complete_delta()` and its supporting functions (`validate_delta_status`, `collect_requirements_to_update`, `update_delta_frontmatter`, `update_requirements_status`, `update_requirements_in_revision_sources`, `handle_already_completed_delta`) into `scripts/lib/changes/completion.py`.
- **Hollow functions** (`display_preview`, `display_actions`, `display_dry_run_requirements` and other `pass`-only display stubs): Delete. They produce no output today.
- **Residual**: `complete_delta.py` either deleted (if argparse `main()` is unused) or reduced to thin shim re-exporting from `completion.py`. The CLI path (`cli/complete.py`) already imports `complete_delta` directly — update that import.
- **Test**: `complete_delta_test.py` imports updated to new location.
- **Importers to update**: `cli/complete.py` (line 10).

## 6. Design Decisions

- **DEC-113-01**: Relocate `complete_delta` business logic to `scripts/lib/changes/completion.py` — already holds `complete_revision`, natural home, avoids creating new modules.
- **DEC-113-02**: Delete hollow `pass`-body display functions rather than implement them — they produce no output today, zero behaviour change.
- **DEC-113-03**: Add `validate_revision_blocks.py` to kill list — orphaned, no recent activity, no importers.

## 7. Risks & Mitigations

- **Risk**: Orphaned script invoked externally → **Mitigated**: `pyproject.toml [project.scripts]` has only `spec-driver = supekku.cli.main:main`. Justfile grep clean. No CI refs.
- **Risk**: `complete_delta` relocation breaks CLI → **Mitigated**: Mechanical import update, existing test coverage on CLI path.
