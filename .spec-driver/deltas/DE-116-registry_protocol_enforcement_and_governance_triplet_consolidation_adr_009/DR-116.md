# DR-116 – Registry protocol enforcement and governance triplet consolidation (ADR-009)

## 1. Executive Summary

- **Delta**: [DE-116](./DE-116.md)
- **Status**: accepted (2026-06-02)
- **Owners / Team**: unassigned
- **Last Updated**: 2026-06-01
- **Synopsis**: Formalise the ADR-009 registry read surface as `RegistryProtocol[T]`, collapse the
  decision/policy/standard triplet onto a `FrontmatterFileRegistry[T]` base, hoist backlink
  composition up to `Workspace`, and migrate the resulting records + registries into
  `spec_driver.domain` (records/registries layers) with legacy re-export shims.

## 2. Problem & Constraints

- **Current Behaviour**: Three governance registries (`DecisionRegistry`, `PolicyRegistry`,
  `StandardRegistry`, ~330 lines each) independently implement `collect/_parse_file/iter/find/filter/
  write/sync/load/to_dict` with ~80% duplicated bodies, differing only in prefix strings, glob
  patterns, YAML root keys, and backlink topology. ADR-009's read-surface convention is enforced
  only by documentation. The registries lazily import `build_backlinks` from `domain.relations` — an
  upward edge that passes today only because legacy `supekku/` is unlinted (the import-linter
  blindspot), and which becomes a layer violation the moment the registries migrate.
- **Drivers / Inputs**: ADR-009 §6 names **DE-116 as the designated vehicle** for the
  `typing.Protocol` formalisation (revisit trigger met: 10 registries, ≥8 conforming). POL-001
  (reuse, minimise sprawl). The layered-migration program (DE-125/127/128, complete) established the
  `spec_driver.domain` internal-layer contract.
- **Constraints / Guardrails**:
  - External CLI behaviour (stdout) unchanged.
  - Generated registry YAML (`decisions/policies/standards.yaml`) must be byte-identical after the
    backlink hoist.
  - ADR-002 remains in force: no derived backlinks in frontmatter.
  - Domain-internal layer order (lower may not import higher): `lifecycle < records < registries <
    relations < validation`. **`domain.registries` must not import `domain.relations`.**
  - New domain modules import **`spec_driver.core.*`** (the canonical helpers already exist:
    `dates`, `paths`, `repo`, `spec_utils`) — **not** legacy `supekku.scripts.lib.core` (ER-6).
    "Legacy-core tolerated" does **not** apply here because the canonical core is available.
- **Out of Scope**:
  - Updating the **legacy `supekku` consumers** to canonical paths + deleting shims (tracked
    follow-up). **In scope this delta**: the ≤3 `spec_driver`-internal consumers
    (`orchestration/artifact_view.py`, `domain/relations/{backlinks,query}.py` where they import the
    symbols) migrate to canonical paths now — internal `spec_driver` code must not route through the
    legacy facade (ER-4).
  - `RequirementsRegistry` / `CardRegistry` (different domains, separate deltas).
  - Migrating `creation`/`Workspace` themselves into `spec_driver.orchestration` (kept legacy).
  - Persistence **naming** unification (`sync`/`write`/`save`) — ADR-009 §5 defers it.

## 3. Architecture Intent

- **Target Outcomes**:
  - `RegistryProtocol[T]` formalised per ADR-009 §6 covering **`find`/`collect`/`iter` only**
    (typed, ty-verifiable, exact signature match). **`filter` is deliberately excluded from the
    typed generic protocol** (ER-1): its keyword-only, per-registry-divergent signature cannot be a
    structural subtype of any single generic `filter` signature. `filter` presence is asserted by
    convention/test, per ADR-009 §5 ("the method exists; params stay domain-specific"). This is a
    design fact, not a spike outcome.
  - One `FrontmatterFileRegistry[T]` ABC; decision/policy/standard registries reduced to ~40 lines
    of genuine variation (prefix, root key, record construction, backlink declaration, filter fields).
  - Registries **pure**: no `relations` import, no backlink building, migration-ready.
  - Records + registries live in `spec_driver.domain.{records,registries}`; legacy paths shimmed.
  - Creation collapses to one shared private engine **impl-only** (P3): public `create_X` /
    `*Options` / `*Result` / `*AlreadyExistsError` and exact CLI messages preserved as wrappers.
- **Guiding Principles**: ADR-009 read surface is the only mandated contract; persistence stays
  domain-specific but its *mechanical* duplication is removed; **orchestration (Workspace) owns
  cross-registry composition** (migration principle #3); backlink topology expressed as data, not code.
- **State Transitions / Lifecycle Impact**: none. Record status semantics, ADR status-symlink
  directories, and lifecycle values are unchanged. ADR status-from-directory inference is preserved
  as a `_resolve_status` override.

### Base class shape

```python
# Imports: spec_driver.core.{dates,paths,repo,spec_utils} + spec_driver.domain.records + yaml + stdlib.
# NO supekku.* import, NO spec_driver.domain.relations import.
#
# RegistryProtocol[T] (find/collect/iter ONLY — filter excluded, see §3 / ER-1):
#   @runtime_checkable
#   class RegistryProtocol(Protocol[T_co]):
#     def find(self, id: str) -> T_co | None: ...
#     def collect(self) -> dict[str, T_co]: ...
#     def iter(self, status: str | None = None) -> Iterator[T_co]: ...

class FrontmatterFileRegistry(ABC, Generic[T]):
  _prefix: str            # 'ADR' | 'POL' | 'STD'
  _yaml_root_key: str     # 'decisions' | 'policies' | 'standards'
  backlink_inputs: list[tuple[str, str]] = []   # (sibling_category, source_field) — data, consumed by Workspace

  def __init__(self, *, root: Path | None = None) -> None:
    self.root = find_repo_root(root)
    self.directory = self._artifact_dir(self.root)
    self.output_path = get_registry_dir(self.root) / f"{self._yaml_root_key}.yaml"

  def collect(self) -> dict[str, T]: ...                          # glob f'{_prefix}-*.md', swallow, key by id
  def iter(self, status: str | None = None) -> Iterator[T]: ...   # positional-or-keyword (ADR-009 §1)
  def find(self, id: str) -> T | None: ...
  def filter(self, *args, **kwargs) -> list[T]: ...               # overridden per registry; see _filter
  def write(self, path: Path | None = None, *, records: dict[str, T] | None = None) -> None: ...  # PURE dump
  def sync(self) -> None: ...                                     # write(self.collect()); no backlinks

  def _parse_file(self, path: Path) -> T | None:
    fm, content = load_markdown_file(path); fm = fm or {}
    m = re.match(rf'{self._prefix}-(\d+)', path.name)
    if not m: return None
    record_id = fm.get('id', f'{self._prefix}-{m.group(1)}')
    title = fm.get('title') or extract_h1_title(content, f'{self._prefix}-') or path.stem.replace('-',' ').title()
    status = self._resolve_status(fm, path)
    return self._build_record(fm, content, record_id=record_id, title=title, status=status, path=path)

  def _resolve_status(self, fm: dict, path: Path) -> str:
    return (fm.get('status') or '').lower() or 'draft'

# DecisionRegistry override MUST short-circuit (AR-1) — `super() or infer` never reaches infer,
# because super() always returns at least 'draft':
#   def _resolve_status(self, fm, path):
#     status = (fm.get('status') or '').lower()
#     if status: return status
#     return self._infer_from_dirs(path) or 'draft'

  def _matches_common(self, r, *, tag, spec, delta, requirement) -> bool: ...
  def _filter(self, *, common: dict, extra: dict) -> list[T]:
    return [r for r in self.iter()
            if self._matches_common(r, **common)
            and all(v in getattr(r, f) for f, v in extra.items() if v)]

  @abstractmethod
  def _artifact_dir(self, root: Path) -> Path: ...
  @abstractmethod
  def _build_record(self, fm, content, *, record_id, title, status, path) -> T | None: ...
```

### Workspace backlink composition (orchestration owns it)

```python
def _sync_governance(self, registry):
  records = registry.collect()
  groups = [([(r.id, getattr(r, field)) for r in self._registry_for(cat).collect().values()], cat)
            for cat, field in registry.backlink_inputs]
  if groups:
    build_backlinks_multi(records, groups)        # the ONE relations call, at orchestration layer
  registry.write(records=records)

def sync_policies(self):  self._sync_governance(self.policies)
def sync_standards(self): self._sync_governance(self.standards)
def sync_decisions(self): self.decisions.sync_with_symlinks()   # unchanged; ADR has no backlinks
```

### Creation unification — impl-only (P3, public surface preserved)

Shared **private** engine; the 3 public modules stay as thin typed wrappers. `label` (not `prefix`)
drives the error message so wording is unchanged.

```python
@dataclass(frozen=True)
class _GovernanceArtifactSpec:
  prefix: str; label: str; template_name: str; render_var: str
  build_frontmatter: Callable[[str, Any], dict]

def _create_governance_artifact(spec, registry, options, *, sync_registry=True) -> tuple[str, Path]:
  artifact_id = next_sequential_id(registry.collect(), spec.prefix)
  record_artifact(artifact_id)                                    # now ALL three (behaviour change on STD — OQ)
  path = registry.directory / f"{artifact_id}-{slugify(options.title)}.md"
  if path.exists():
    raise _AlreadyExists(f"{spec.label} file already exists: {path}")   # 'Policy'/'Standard'/'ADR' — UNCHANGED text
  fm = spec.build_frontmatter(artifact_id, options)               # per-type shaping retained
  content = Template(extract_template_body(get_templates_dir(registry.root) / spec.template_name)
                    ).render(**{spec.render_var: artifact_id, 'title': options.title})
  path.write_text(f"---\n{yaml.safe_dump(fm, sort_keys=False)}---\n\n{content}", encoding='utf-8')
  if sync_registry: registry.sync()
  return artifact_id, path

# Public wrappers KEPT verbatim in signature/type/message (no break):
def create_adr(registry, options, *, sync_registry=True) -> ADRCreationResult: ...    # wraps engine, maps _AlreadyExists -> ADRAlreadyExistsError
class ADRAlreadyExistsError(Exception): ...                                           # preserved
# ditto create_policy/create_standard, *CreationOptions, *CreationResult, *AlreadyExistsError
```

`cli/create.py` is **unchanged** (`result.adr_id`, `ADRAlreadyExistsError`, messages all intact).

## 7. Design Decisions & Trade-offs

- **DEC-116-1** — `RegistryProtocol` = ADR-009 read surface only (find/collect/iter/filter).
  Persistence excluded (ADR-009 §5 defers it). *Consequence*: write/sync are base implementation
  detail, not contract.
- **DEC-116-2** — Backlink composition hoisted to `Workspace`; registries pure. *Rationale*:
  removes the illegal `registries → relations` edge; aligns with "orchestration owns cross-registry
  composition". *Trade-off*: re-works the DE-125 Phase-4 `decision_sources` seam (acceptable — the
  seam's hard part, passing records down, is reused).
- **DEC-116-3** — Full migration into `spec_driver.domain` now, legacy shims. *Rationale*: user
  directive ("do it properly"); avoids planting a new misplaced integration hub in `core/`.
  *Trade-off*: larger blast radius than the original card; shim debt incurred (tracked follow-up).
- **DEC-116-4** — `iter()` stays positional-or-keyword (ADR-009 §1). **Corrects** DE-116.md's
  "keyword-only" instruction, which over-rides the governing ADR and would break Protocol
  conformance.
- **DEC-116-5** — Base is an ABC template method. *Rationale*: typed, ty-checkable, natural
  `self`-access for ADR's symlink status scan; enforces the `_build_record` contract.
- **DEC-116-6** — Creation unified **impl-only** (shared private engine; public `create_X`/
  `*Options`/`*Result`/`*AlreadyExistsError` + exact messages preserved as wrappers). Per-type
  frontmatter builders retained. Sequenced as an isolated final phase **P3** after the registry core
  + migration land (risk isolation per ER-5/ER-7).
- **Withdrawn drift claims**: POL/STD "ignore author_email" — *not a bug* (no email field in their
  schema). ADR "omits print_schema_hints" — *left as-is* (external stdout constraint).

## 11. Adversarial Review Findings (integrated)

- **AR-1 (fixed)** — `DecisionRegistry._resolve_status` cannot be `super() or infer`: the base
  default always returns at least `'draft'`, so the symlink-dir inference would be unreachable. Must
  short-circuit (corrected sketch in §4).
- **AR-2 (verification strengthened)** — the **live governance corpus has zero backlinks** (no
  decision forward-references a policy/standard; `related_policies` is not a backlink source). A
  golden-YAML diff over the live repo would therefore pass even if the backlink hoist were broken.
  The VT **must run over a fixture corpus** with known decision→policy and decision/policy→standard
  cross-references, asserting non-empty `backlinks:` blocks identical before/after.
- **AR-3 (overclaim corrected)** — `isinstance(reg, RegistryProtocol)` with `runtime_checkable`
  verifies **method presence only**, not signatures. `uv run ty check` is the real conformance gate.
- **AR-4 (POL-002, follow-up)** — `backlink_inputs` category/field literals
  (`'decisions'`/`'policies'`/`'standards'`) and `Workspace._registry_for` dispatch use bare string
  literals → magic strings. Prefer a small shared constant set / enum for category names. Track as a
  minor cleanup; not a blocker.
- **AR-5 (known debt, tolerated)** — `domain.registries` → legacy `core.spec_utils`
  (`load_markdown_file`, `extract_h1_title`) is a legacy-core import; `spec_utils` itself carries an
  upward edge to `orchestration.templates`. import-linter (root=`spec_driver`) does not trace into
  unlinted `supekku`, so this passes — but it is real transitive debt cleared when core migrates.
  Consistent with `mem.fact.architecture.import-linter-supekku-blindspot`.
- **AR-6 (sequencing note)** — DE-120 (Workspace facade) also modifies `Workspace`. DE-116 is a
  dependency-of DE-120, so DE-116's backlink-hoist lands first; flag the co-modification when
  sequencing.
