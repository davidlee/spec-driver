## 1. Summary

- **Delta**: DE-116 — Registry protocol enforcement and governance triplet consolidation (ADR-009).
- **Specs Impacted**: SPEC-117 (decisions), SPEC-126 (policies), SPEC-127 (standards). No new FRs — architectural refactor; verification is gate-based, not requirement-coverage-based.
- **Design Revision**: [DR-116](./DR-116.md) (review-integrated: internal AR-1..6, external codex ER-1..7).
- **Desired Outcome**: `RegistryProtocol[T]` (ADR-009 read surface) formalised; decision/policy/standard triplet collapsed onto one `FrontmatterFileRegistry[T]` ABC; backlink composition hoisted to `Workspace`; records + registries migrated into `spec_driver.domain` with legacy re-export shims; creation impl-unified behind preserved public symbols. External CLI stdout + public creation surface byte-identical.

## 2. Context & Constraints

- **Current Behaviour**: 3 governance registries (~330L each) duplicate `collect/_parse_file/iter/find/filter/write/sync/load/to_dict` (~80% identical). Registries lazily import `build_backlinks` from `domain.relations` — an upward edge tolerated only by the import-linter supekku blindspot, illegal once migrated.
- **Target Behaviour**: DR-116 §3 (Architecture Intent) / §4 (Code Impact Summary). One ABC base; ~40L per registry; pure (no relations import); canonical `spec_driver.domain.{records,registries}` homes.
- **Dependencies**: DE-115 (helpers — done), DE-125/127/128 (migration program — done). Dependency-of DE-120 (Workspace facade — co-modifies `Workspace`, see AR-6: DE-116 lands first).
- **Constraints**:
  - External CLI stdout unchanged; public creation symbols (`create_X`/`*Options`/`*Result`/`*AlreadyExistsError`) + exact error messages preserved.
  - Generated registry YAML byte-identical after backlink hoist (R1 safety net).
  - ADR-002 in force (no derived backlinks in frontmatter).
  - Domain layer order: `lifecycle < records < registries < relations < validation`. `domain.registries` MUST NOT import `domain.relations`.
  - New domain modules import `spec_driver.core.*` (canonical), NOT `supekku.scripts.lib.core` (ER-6).
