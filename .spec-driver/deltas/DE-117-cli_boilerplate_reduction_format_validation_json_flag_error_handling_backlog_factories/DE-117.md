## 1. Summary & Context

- **Technical Spec(s)**: SPEC-110 (supekku/cli)
- **Change Drivers**: Code quality audit identified pervasive boilerplate across CLI commands.
- **Governance**: POL-001 (maximise code reuse)

## 2. Motivation

The CLI layer has accumulated significant mechanical repetition:

| Pattern                                                              | Occurrences | Lines wasted |
| -------------------------------------------------------------------- | ----------- | ------------ |
| `if format_type not in ["table", "json", "tsv"]` validation          | 13×         | ~52          |
| `if json_output: format_type = "json"` override                      | 17×         | ~17          |
| Status filter pipe (`parse_multi_value_filter` → normalize → filter) | 7×          | ~28          |
| Backlog list shortcut commands (passthrough to `list_backlog`)       | 4 commands  | ~452         |
| Triple-catch error handling in `edit.py`                             | 16×         | ~48          |
| Backlog create commands (identical except kind string)               | 4 commands  | ~120         |
| `find.py` commands bypassing `find_artifacts()`                      | 6 commands  | ~80          |
| Backlog `show_*` ad-hoc lambda formatters                            | 4 commands  | ~40          |

## 3. Scope & Objectives

- **Primary Outcomes**:
  - Extract `validate_format(format_type)` helper or use Typer enum constraint
  - Extract `resolve_json_flag(json_output, format_type) -> str` helper
  - Extract `filter_by_status(items, status_raw)` helper
  - Factory-register backlog list shortcuts (eliminate 452 lines of passthrough)
  - `@handle_errors` decorator for edit commands (encapsulate `typer.Exit` guard)
  - Factory for `create_issue/problem/improvement/risk` commands
  - Wire `find.py` commands through `find_artifacts()` (remove 6 direct registry imports)
  - Add proper `format_*_details` for backlog subtypes in formatters layer
- **Operational Constraints**: CLI contract (command names, flags, output format) unchanged.
- **Dependencies**: DE-114 (splits make files small enough to work on cleanly). Can run parallel with DE-115.

## 4. Out of Scope

- Changing CLI command names or flag semantics
- Restructuring the Typer app hierarchy
- Domain logic extraction from `list_requirements --implemented-by` (that's DE-114 territory in the requirements split)

## 5. Approach Overview

- Extract helpers into `cli/common.py` (post-split, this is slim enough)
- For backlog shortcuts: use Typer's dynamic command registration or a factory function
- For `@handle_errors`: decorator that catches `ArtifactNotFoundError` and `RuntimeError` (but re-raises `typer.Exit`)
- For `find.py`: replace direct registry iteration with `find_artifacts()` calls

## 6. Verification Strategy

- **Acceptance Criteria**:
  - `just check` passes
  - Net line reduction ≥ 600 lines
  - No format validation inline in command functions
  - No `if json_output: format_type = "json"` inline in command functions
  - `find.py` imports no registry classes directly

## 7. Risks & Mitigations

- **Risk**: Typer dynamic registration changes `--help` output – _Likelihood_: medium – _Impact_: low – _Mitigation_: Verify help text in CI; Typer preserves docstrings on dynamically registered commands.
- **Risk**: `@handle_errors` masks unexpected exceptions – _Likelihood_: low – _Impact_: medium – _Mitigation_: Decorator only catches the two known exception types; all others propagate.

## 8. Follow-ups & Tracking

- **Depends on**: DE-114 (file splits)
- **Sequence**: Sprint 3 of quality remediation. Parallel with DE-115.
