# DR-117 – CLI boilerplate reduction

## 1. Executive Summary

- **Delta**: [DE-117](./DE-117.md)
- **Status**: draft
- **Last Updated**: 2026-04-10
- **Synopsis**: Extract 7 repeated CLI patterns into shared helpers, enums, a decorator, and two factory loops — eliminating ~700 lines of mechanical repetition while preserving the CLI contract.

## 2. Problem & Constraints

- **Current Behaviour**: The CLI layer has accumulated pervasive boilerplate through organic growth: format validation (13×), JSON flag override (21×), triple-catch error handling (17×), near-identical backlog passthrough commands (4×110 lines), backlog create commands (4×33 lines), find commands bypassing `find_artifacts()` (6×), and inline lambda formatters (3×). Total waste: ~800 lines.
- **Drivers / Inputs**: POL-001 (extraction threshold: ≥2 non-test callers ⇒ extract), STD-001 §4-5 (shared error handling and helpers required), code quality audit.
- **Constraints / Guardrails**: CLI contract (command names, flags, output format) must be unchanged. `--help` output must remain correct. All existing tests must pass.
- **Out of Scope**: Changing CLI command names or flag semantics. Restructuring the Typer app hierarchy. Domain logic extraction. Workspace enforcement (DE-120).

## 3. Architecture Intent

- **Target Outcomes**:
  - No format validation inline in command functions (SPEC-110 FR-003)
  - No `if json_output: format_type = "json"` inline in command functions
  - `find.py` imports no registry classes directly
  - `edit.py` has no per-command try/except blocks
  - Net line reduction ≥ 600
- **Guiding Principles**: Extract to `cli/common.py` (STD-001 §5). Pure functions where possible. No premature abstraction — factories only where 4+ identical commands exist. Formatters in formatters layer, not CLI.
- **State Transitions / Lifecycle Impact**: None — no status values or lifecycle logic touched.

## 7. Design Decisions & Trade-offs

- **DEC-117-01 — StrEnum for format validation**: Typer natively validates enum options, producing clear error messages with valid choices listed. Eliminates 13 inline checks. `StrEnum` means `==` comparisons with string literals still work — no downstream changes needed in format dispatch logic.

- **DEC-117-02 — Closure-based factory for backlog commands**: A `_make_backlog_*_command(kind)` function generates and registers commands in a loop. Preserves `--help` output because Typer introspects the closure's signature. Lower risk than dynamic Typer registration or metaclass approaches. `__name__` and `__doc__` are set explicitly for correct help identity.

- **DEC-117-03 — Fixed exception set in `handle_cli_errors`**: The decorator catches exactly `typer.Exit` (re-raises), `RuntimeError`, `FileNotFoundError`, `ValueError`, `KeyError`. This is the union of what the 17 edit commands currently catch. No configurable exception lists — that would be speculative generality. `ArtifactNotFoundError` inherits from `RuntimeError`, so it's already covered.

- **DEC-117-04 — `BACKLOG_KINDS` duplicated, not shared**: The 4-element tuple `("issue", "problem", "improvement", "risk")` appears in both `list/backlog_items.py` and `create.py`. Sharing it would create a coupling path between unrelated modules. The duplication cost (one line) is lower than the coupling cost.

## 9. Rollout & Operational Notes

- **Phasing**:
  - **Phase 1**: Helper extractions + simple consumer rewrites (Sections 1-3, 6-7). `OutputFormat` enum, `resolve_format()`, `handle_cli_errors()`, find.py consolidation, backlog show formatters.
  - **Phase 2**: Factory rewrites (Sections 4-5). Backlog list factory, backlog create factory.
- **Migration**: No data migration. Pure code refactoring.
- **Recovery / Rollback**: Each phase is a single commit. Revert commit to rollback. No state changes to undo.
