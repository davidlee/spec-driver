## 1. Summary

- **Delta**: DE-118 — Block schema unification: retire hand-rolled validators, single metadata-driven validation layer.
- **Specs Impacted**: SPEC-114 (blocks/metadata), SPEC-115 (changes/blocks), SPEC-116 (frontmatter_metadata pattern reference).
- **Desired Outcome**: 7 hand-rolled `*Validator` classes retired in favour of `MetadataValidator` driven by existing `BlockMetadata` declarations, with verdict-level behaviour preservation. Mechanism only — cross-cutting strictness governance (CLI flag, alias autocorrect, migrations) remains DE-137.

## 2. Context & Constraints

- **Current Behaviour**: Two parallel validation systems for the same YAML block schemas (hand-rolled + metadata-driven), divergent strictness, active regex bugs in `REVISION_BLOCK_JSON_SCHEMA`, and structural gaps (`_entry_shape` sentinel, `_placeholder_renderer`).
- **Target Behaviour**: Single metadata-driven validation surface for the 7 listed block types; explicit `strict_unknown_keys` opt-in at retirement call sites; ID-kwarg wrapper helpers preserve external-state checks for `RelationshipsBlockValidator` and `DeltaRelationshipsValidator`.
- **Dependencies**: Loosely DE-114 (workflow_metadata split), DE-115 (block pattern factory). Sequenced before DE-137 per IP-136 Phase 2.
- **Constraints**: Behaviour-equivalence required; `spec-driver validate` baseline-diff clean at every commit; consumer-repo behaviour for workflow.* blocks unchanged (DEC-006 deferral to DE-137 via IMPR-035).
