## 1. Summary & Context

- **Technical Spec(s)**: SPEC-161 (workflow), SPEC-124 (sync/adapters)
- **Change Drivers**: Code quality audit found duplicated state between workflow modules, a broken sync engine interface, and domain logic embedded in CLI commands.
- **Governance**: POL-001

## 2. Motivation

Targeted fixes across the workflow and changes subsystems:

1. **`BootstrapStatus` + `_extract_cached_files` duplicated** between `staleness.py` and `review_state_machine.py`. The former uses a plain class; the latter a `StrEnum`. State sets have diverged (`WARMING` present in one, absent in the other).

2. **`SpecSyncEngine` has a broken interface**: `engine.py` line 211 calls `adapter.generate(unit, check=check)` but `LanguageAdapter.generate()` requires `variant_outputs: dict[str, Path]` as mandatory kwarg. Would raise `TypeError` at runtime. Tests pass only via `Mock`. CLI bypasses the engine entirely.

3. **Domain logic in `workflow.py` CLI**: `_generate_bootstrap_markdown` (92 lines) and `_build_domain_map` (68 lines) are pure domain functions with no CLI dependency. `_build_required_reading` logic duplicated between `create_handoff_command` and `phase_complete_command`.

4. **`create_completion_revision`** does direct string manipulation (`content.replace("status: draft", ...)`) bypassing `load_markdown_file`/`dump_markdown_file`.

5. **TypeScript `supports_identifier()` fallback** returns `True` for nearly any string, making adapter ordering load-bearing.

## 3. Scope & Objectives

- **Primary Outcomes**:
  - Unify `BootstrapStatus` as one `StrEnum` in a shared module; extract `_extract_cached_files` to shared location; close `WARMING`/`delta_updated` divergence
  - Fix or delete `SpecSyncEngine`: either pass `variant_outputs` through the call chain, or delete the engine class (it's dead code for production paths)
  - Extract `_generate_bootstrap_markdown` and `_build_domain_map` from `workflow.py` to `lib/workflow/`
  - Extract `_build_required_reading()` helper to deduplicate handoff/phase-complete
  - Fix `create_completion_revision` to use frontmatter primitives
  - Fix TypeScript `supports_identifier()` to require positive signal (extension or tsconfig.json)
- **Operational Constraints**: Each fix is independent and can be landed separately.
- **Dependencies**: None strict. Independent of other deltas.
