## 1. Summary & Context

- **Implementation Plan**: [IP-123](./IP-123.md) – draft
- **Change Drivers**: Large number of completed deltas makes the default `list deltas` output noisy.

## 2. Motivation

The `spec-driver list deltas` command currently shows all deltas in the repository. As a project grows, the number of completed deltas increases, making it difficult to see active work. Hiding completed deltas by default aligns with the behavior of `list backlog` and `list specs` (which hides unit specs).

## 3. Scope & Objectives

- **Primary Outcomes**:
  - `list deltas` hides deltas with status `completed` (or legacy `complete`) by default.
  - A new `--all` (or `-a`) flag is added to `list deltas` to include completed deltas.
  - The behavior of existing status filters is preserved (e.g., `-s completed` should still work).

## 4. Out of Scope

- Changing other list commands (unless they already follow this pattern and need adjustment).
- Implementing pagination for `list deltas` (out of scope for now).

## 5. Approach Overview

- **System Touchpoints**: `supekku/cli/list.py`
- **Key Changes**:
  - Add `--all` option to `list_deltas`.
  - Update filtering logic in `list_deltas` to apply default status filtering when `status` is None and `--all` is False.

## 6. Verification Strategy

- **Requirements Coverage**:
  - Verify `list deltas` hides completed deltas by default.
  - Verify `list deltas --all` shows all deltas.
  - Verify `list deltas -s completed` still works.
  - Verify `list deltas -s draft` still works.
- **Planned Artefacts**:
  - VA-123-01: Agent verification of CLI behavior.
