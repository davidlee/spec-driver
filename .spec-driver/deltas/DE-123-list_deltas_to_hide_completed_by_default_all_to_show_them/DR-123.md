# DR-123 – list deltas to hide completed by default; --all to show them

## 1. Current Behavior

The `spec-driver list deltas` command displays all deltas in the repository regardless of their status. As the project matures, the output becomes cluttered with numerous `completed` deltas, making it harder to track active work.

## 2. Target Behavior

- `spec-driver list deltas` will hide deltas with status `completed` (or legacy `complete`) by default.
- A new flag `--all` (shorthand `-a`) will be introduced to override this default and show all deltas.
- If an explicit status filter is provided (e.g., `-s completed` or `-s draft`), the default "hide completed" behavior is bypassed.

## 3. Code Impact

### `supekku/cli/list.py`

- Modify `list_deltas` function signature to include `show_all: bool`.
- Update filtering logic in `list_deltas` to implement the new default.

## 4. Design Decisions

### 4.1. Default Filtering Logic

The default filter will only apply if:

1. No explicit `--status` filter is provided.
2. The `--all` flag is not set.
3. The `--unaudited` flag is not set (as it already forces `-s completed`).

If these conditions are met, deltas with `normalize_status(artifact.status) == STATUS_COMPLETED` will be excluded.

### 4.2. Compatibility with `--unaudited`

The `--unaudited` flag is an alias for `--not-referenced-by audit -s completed`. Since it explicitly sets the status to `completed`, it should naturally bypass the default "hide completed" filter.

### 4.3. Interaction with Positional IDs

If specific delta IDs are provided as positional arguments (e.g., `list deltas DE-001`), the default status filter will still apply unless `--all` is used. This is consistent with how other filters currently interact with positional IDs in `list_deltas`.

## 5. Verification Plan

### 5.1. Automated Tests (VA-123-01)

Verify the following scenarios via CLI:

1. `spec-driver list deltas` -> Does NOT show completed deltas.
2. `spec-driver list deltas --all` -> DOES show completed deltas.
3. `spec-driver list deltas -s completed` -> DOES show completed deltas (explicit override).
4. `spec-driver list deltas -s draft` -> Shows only draft deltas (default bypass).
5. `spec-driver list deltas --unaudited` -> Shows completed deltas without audits.

## 6. Adversarial Review

### 6.1. Review Findings

- **Question**: What if a delta has an unknown status?
  - **Response**: `normalize_status` returns the original string if not in the map. If it's not `STATUS_COMPLETED`, it will be shown by default. This is safe.
- **Question**: Should this apply to `list changes`?
  - **Response**: The request specifically targeted `list deltas`. `list changes` is often used for a broader view and its current behavior (showing all) is acceptable unless it also becomes too noisy. We will keep it out of scope for now to minimize surprise.
- **Question**: Is `-a` available for `list deltas`?
  - **Response**: Yes, it is currently unused in `list deltas`.
