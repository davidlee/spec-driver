## 1. Summary & Context

- **Product Spec(s)**: PROD-010 (CLI UX), PROD-011 (Implementation Execution Workflow)
- **Implementation Plan**: [IP-124](./IP-124.md)
- **Design Revision**: [DR-124](./DR-124.md)
- **Change Drivers**: autobahn DE-004 needs to call spec-driver's review/workflow operations as typed Python functions. The CLI commands contain load-bearing orchestration logic (guards, state transitions, staleness evaluation, write ordering) that cannot be replicated by importing building blocks alone. A public API must expose composed operations, not just primitives.

## 2. Motivation

- autobahn is a Python library in the same uv workspace. Subprocess + JSON envelope is unnecessary overhead for same-process Python consumers.
- The CLI review commands (`review prime`, `review complete`, finding disposition) contain 120–150 lines of orchestration logic each — guard checks, state transition validation, staleness cascade, write ordering. This logic is load-bearing, not presentational.
- Currently this orchestration lives in the CLI layer (`supekku/cli/workflow.py`), violating the skinny CLI pattern (mem.pattern.cli.skinny). Extracting it into domain functions simultaneously: (a) makes the CLI genuinely thin, (b) gives autobahn the operations it needs, (c) ensures guards/transitions exist in exactly one place.
- `supekku` is an internal codename. A new `spec_driver` top-level package with an explicit `orchestration` layer provides the right namespace for public API.

## 3. Scope & Objectives

- **Primary Outcomes**:
  - Extract review orchestration from CLI into domain functions in `spec_driver/orchestration/operations.py`:
    - `resolve_delta_dir` — delta ID → directory resolution
    - `prime_review` — full prime orchestration (staleness, index, bootstrap, transition)
    - `complete_review` — review completion with opt-in teardown
    - `disposition_finding` — typed kwargs, domain validation, write
    - `teardown_review` — file cleanup
    - `summarize_review` — read-only query (findings counts, approval gate, judgment status)
  - Refactor CLI commands to thin wrappers around domain functions
  - New top-level package `spec_driver/` with `spec_driver.orchestration` re-exporting 52 symbols: operations, result types, enums, models, state machine functions, exceptions
  - Typed result dataclasses with enum fields (not raw strings)
  - Disposition domain validation (waive→rationale, defer→backlog_ref, etc.)
  - Import smoke tests for the public surface
  - Existing CLI behaviour unchanged (same exit codes, same JSON envelope, same human output)
- **Operational Constraints**: Existing CLI tests must continue to pass. No new dependencies. No changes to workflow file formats.
- **Dependencies**: None — builds on completed DE-108, DE-109

## 4. Out of Scope

- Renaming `supekku` → `spec_driver` internally (future, driven by DE-114–DE-122 refactoring arc)
- Public API for non-workflow subsystems (memories, specs, registries) — add when consumers exist
- Fixing the god-init transitive import cascade in `supekku/scripts/lib/__init__.py` (DE-114 scope)
- Handoff I/O or bridge re-exports — add when autobahn needs them

## 5. Approach Overview

- **Key Changes**:
  - `spec_driver/orchestration/operations.py` (new) — composed domain functions:
    - `resolve_delta_dir(delta_id, root) → Path`
    - `prime_review(delta_dir, repo_root) → PrimeResult`
    - `complete_review(delta_dir, repo_root, *, status, summary, auto_teardown) → CompleteResult`
    - `disposition_finding(delta_dir, finding_id, *, action, authority, ...) → DispositionResult`
    - `teardown_review(delta_dir) → TeardownResult`
    - `summarize_review(delta_dir) → ReviewSummary`
  - `supekku/cli/workflow.py` — refactor review commands to delegate to `operations.py`
  - `spec_driver/__init__.py` (new) — package root
  - `spec_driver.orchestration/__init__.py` (new) — re-exports (52 symbols: operations, types, exceptions)
  - `pyproject.toml` — include `spec_driver` in packages

## 6. Verification Strategy

- **Acceptance Criteria**:
  - Extracted domain functions are independently testable (unit tests for each operation)
  - CLI review commands delegate to domain functions — no orchestration logic remaining in CLI
  - All existing CLI tests pass without modification (behaviour preservation)
  - `from spec_driver.orchestration import prime_review, BootstrapStatus, StateNotFoundError` works
  - All 52 `__all__` symbols importable and identity-equal to sources
  - Disposition validation: waive without rationale raises `DispositionValidationError`
  - `summarize_review` returns correct counts and `all_blocking_resolved` for fixture data
  - `just check` passes

## 7. Risks & Mitigations

- **Risk**: Extraction introduces subtle behaviour differences — _Likelihood_: medium — _Impact_: medium — _Mitigation_: Existing CLI integration tests are the safety net; run before and after extraction
- **Risk**: Internal module refactoring breaks re-exports — _Likelihood_: medium — _Impact_: low — _Mitigation_: Import tests catch breakage; re-exports updated in same PR as internal moves
- **Risk**: Transitive import cascade (~77 modules) via `supekku.scripts.lib.__init__.py` — _Likelihood_: certain — _Impact_: low (~100ms startup) — _Mitigation_: Known limitation, documented; resolves when DE-114 lands
