# DR-124 – Public Python API facade

## 1. Executive Summary

- **Delta**: [DE-124](./DE-124.md)
- **Status**: draft
- **Last Updated**: 2026-03-23
- **Synopsis**: Create a `spec_driver` top-level package as a curated public API facade over `supekku` internals, starting with workflow/review domain types and functions.

## 2. Problem & Constraints

- **Current Behaviour**: All domain logic lives under `supekku.scripts.lib.*` — a deep, internal namespace that reflects historical structure. External consumers (autobahn) have no stable import path.
- **Drivers**: autobahn DE-004 needs typed Python access to review/workflow domain functions. Subprocess + JSON envelope is unnecessary for a same-workspace Python consumer.
- **Constraints**:
  - Zero changes to existing `supekku/` code
  - Must ship in the same distribution (single `pip install`)
  - Public surface must be narrow and auditable
  - Must not create import cycles
- **Out of Scope**: Internal rename, non-workflow API surface, stability versioning

## 3. Architecture Intent

- **Target Outcomes**:
  - `from spec_driver.workflow import X` is the public contract for programmatic consumers
  - Internal refactoring of `supekku/` does not break consumers — re-export file absorbs the change
  - The namespace anticipates the eventual `supekku` → `spec_driver` migration without requiring it now
- **Guiding Principles**:
  - Facade, not wrapper — re-exports only, no new logic
  - Consumer-driven surface — export what autobahn needs, not everything available
  - Single point of truth for "what is public" — if it's not in the re-export file, it's not public

## 4. Code Impact Summary

| Path | Current State | Target State |

## 6. Design Decisions

**DEC-124-001**: New `spec_driver` package rather than `supekku.public` or `supekku.api`.
- The product name is "spec-driver", not "supekku". Public imports should use the product namespace.
- `spec_driver` anticipates the eventual full migration without coupling to it.
- Same pattern as `_pytest`/`pytest`, `_collections_abc`/`collections.abc`.

**DEC-124-002**: Re-export facade, not wrapper layer.
- No new classes, no delegation objects, no adapter pattern.
- `spec_driver.workflow.BootstrapStatus is supekku.scripts.lib.workflow.review_state_machine.BootstrapStatus` — identity, not copy.
- If a wrapper is ever needed (e.g. to simplify a complex multi-step operation), it's a future delta.

**DEC-124-003**: Flat export from `workflow/__init__.py` rather than `workflow.review`, `workflow.state`, etc.
- The consumer surface is small (~30 symbols). Sub-modules add import path depth without value.
- If the surface grows beyond ~50 symbols, split into submodules then.
