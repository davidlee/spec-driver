## 1. Summary

- **Delta**: DE-124 — Public Python API facade for programmatic consumers
- **Specs Impacted**: PROD-010 (CLI UX), PROD-011 (Implementation Execution Workflow)
- **Desired Outcome**: Review orchestration extracted into domain functions, exposed via `spec_driver.workflow` with typed operations, enums, models, and exceptions. CLI becomes a thin wrapper. autobahn can import directly.

## 2. Context & Constraints

- **Current Behaviour**: Review orchestration (prime, complete, disposition, teardown) lives in CLI command functions (120–150 lines each) with load-bearing guards, transitions, and write ordering.
- **Target Behaviour**: Domain functions in `operations.py` encapsulate all orchestration. CLI delegates. `spec_driver.workflow` re-exports 52 symbols.
- **Dependencies**: None — builds on completed DE-108, DE-109
- **Constraints**: Zero changes to workflow file formats. All existing CLI tests must pass unmodified. No new external dependencies.

## 6. Testing & Verification Plan

- **Existing suites (must pass unmodified)**:
  - `supekku/cli/workflow_review_test.py` — CLI integration tests for review commands
  - `supekku/cli/workflow_integration_test.py` — broader workflow integration tests
  - `supekku/scripts/lib/workflow/review_io_test.py` — review I/O unit tests
  - `supekku/scripts/lib/workflow/review_state_machine_test.py` — state machine tests
- **New suites**:
  - `supekku/scripts/lib/workflow/operations_test.py` — unit tests for all 6 operations
  - `spec_driver/workflow/test_public_surface.py` (or similar) — import smoke tests
- **Key scenarios**:
  - `prime_review`: cold start, rebuild (stale/invalid), refresh (reusable)
  - `complete_review`: approve with/without auto_teardown, changes_requested, approval guard violation
  - `disposition_finding`: each action type, validation failures per DR-109 matrix, finding-not-found
  - `summarize_review`: various finding states, empty findings, no findings file
  - `resolve_delta_dir`: found, not found, case-insensitive
  - Import smoke: all 52 `__all__` symbols are importable and `is`-identical to sources

## 7. Risks & Mitigations

| Risk | Mitigation | Owner |
