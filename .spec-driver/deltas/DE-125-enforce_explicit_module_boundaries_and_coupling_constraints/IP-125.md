## 6. Testing & Verification Plan

- **Architecture Contracts**
  - verify the existing top-level `import-linter` contract with `just lint-imports`
  - draft and then verify a `Domain Internal Layers` contract before registry moves
- **Import Graph Sampling**
  - pilot selection is now based on the current graph:
    `relations.query` and `relations.manager` are the lowest-coupling migration targets
- **Test Layout**
  - keep the current `pytest` `testpaths` for the seam-extraction phase
  - use targeted tests while modules still live under `supekku/*`
  - move tests alongside code only once the first `spec_driver/domain/relations/*`
    pilots exist
- **Validation**
  - run `spec-driver validate` after structured doc updates
  - run targeted checks for each seam:
    `uv run pytest supekku/scripts/lib/relations/query_test.py supekku/scripts/lib/relations/manager_test.py -v`
  - when policy backlink extraction begins:
    `uv run pytest supekku/scripts/lib/policies/registry_test.py -v`
  - use `uv run pytest supekku/scripts/lib/requirements/registry_test.py -v`
    only when the requirements seam work starts
