# DE-126 – Fix pyright typechecking errors (389 errors)

## 1. Summary & Context

Pyright 1.1.408 reports **389 errors** across `supekku/` and `spec_driver/`.
This delta addresses all of them to establish a clean typecheck baseline.

## 2. Motivation

Type errors erode confidence in refactors and hide real bugs behind noise.
A clean pyright run is a prerequisite for adding typechecking to CI.

## 3. Scope & Objectives

**Primary Outcomes**:
- `uv run pyright supekku spec_driver` exits 0 with 0 errors.

**Root causes** (4 categories, ~95% of errors):

| # | Category | Est. errors | Fix approach |
|---|----------|-------------|--------------|
| 1 | Frontmatter `__setitem__` typed `str→str` but receives `list`/`dict` | ~55 | Widen value type or use a richer container |
| 2 | `object` not narrowed before use (dict gets, kwargs unpacking) | ~120 | Add casts / type narrowing at extraction sites |
| 3 | Optional not narrowed (test code accesses `.attr` on `T | None`) | ~60 | Add `assert x is not None` guards in tests |
| 4 | Test-specific type mismatches (`.plain` on `str`, mock attrs) | ~30 | Proper typing for Rich Text, mock patches |

Remaining ~124 errors are variations of the above or one-offs (missing imports, literal mismatches, operator issues).

## 4. Out of Scope

- Adding pyright to CI (follow-up).
- Introducing `py.typed` marker or strict mode.
- Refactoring APIs — fixes should be minimal-invasive type annotations/guards.

## 5. Approach Overview

Work in 4 passes matching the root-cause categories above.
No API changes; fixes are annotation-only or test-guard additions.

**Key touchpoints**:
- `supekku/scripts/lib/policies/` — frontmatter typing
- `supekku/scripts/lib/standards/` — frontmatter typing (mirrors policies)
- `supekku/scripts/lib/memory/` — model construction in tests
- `supekku/scripts/lib/specs/` — optional narrowing in tests
- `supekku/scripts/lib/sync/` — literal status types, adapter typing
- `supekku/scripts/lib/formatters/` — Rich Text vs str
- `supekku/tui/` — optional narrowing, Rich Text, mock attrs

## 6. Verification Strategy

- **Gate**: `uv run pyright supekku spec_driver` exits 0.
- **Regression**: `uv run pytest` passes (no test breakage from annotation changes).

## 7. Risks & Mitigations

- **Risk**: Widening a type hides a real bug — _Likelihood_: low — _Impact_: medium — _Mitigation_: Prefer narrowing (casts/guards) over widening; review each `Any` addition.

## 8. Follow-ups

- Add pyright to `just check` / CI.
- Consider enabling strict mode incrementally.
