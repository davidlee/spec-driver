# DR-128 ‚Äì Migrate genuine core modules to spec_driver.core

## 1. Executive Summary

- **Delta**: [DE-128](./DE-128.md)
- **Status**: draft
- **Synopsis**: Move 22 confirmed-foundational modules from
  `supekku/scripts/lib/core/` to `spec_driver/core/` in dependency-tier order,
  using the established re-export shim pattern.

## 2. Design

### 2.1 Dependency tiers

Migration proceeds bottom-up. Each tier can be migrated as a batch; modules
within a tier have no mutual dependencies.

#### Tier 0 ‚Äî True leaves (13 modules, ~1,200 lines)

Zero internal `core/` dependencies. Can move independently.

| Module | Lines | Third-party | Consumers | Notes |
|--------|-------|-------------|-----------|-------|
| `repo.py` | 32 | ‚Äî | 21 | `find_repo_root()` ‚Äî foundational |
| `strings.py` | 39 | ‚Äî | 1 | `slugify()` |
| `filters.py` | 40 | ‚Äî | 5 | CLI filter parsing |
| `relation_types.py` | 40 | ‚Äî | 1 | Relation/reference constants |
| `artifact_ids.py` | 140 | ‚Äî | 4 | ID normalisation; unblocks `domain/relations/graph.py` |
| `version.py` | 19 | ‚Äî | 3 | Version constant |
| `cli_utils.py` | 78 | yaml | 0 | CLI argument helpers (currently unused outside core) |
| `editor.py` | 122 | ‚Äî | 2 | External editor subprocess |
| `git.py` | 145 | ‚Äî | 4 | Git operations |
| `go_utils.py` | 116 | ‚Äî | 1 | Go toolchain |
| `npm_utils.py` | 187 | ‚Äî | 2 | npm toolchain |

### 2.2 Import update rules

- **Within `spec_driver/core/`**: use relative imports (e.g.
  `from .repo import find_repo_root`).
- **From `spec_driver/domain/`**: use absolute imports (e.g.
  `from spec_driver.core.frontmatter_schema import Relation`).
- **Legacy shims**: re-export everything from canonical location.

### 2.3 Domain import debt cleanup

After tier 0 + 1 land, update three files:

| File | Old import | New import |
|------|-----------|------------|
| `spec_driver/domain/relations/manager.py` | `supekku.scripts.lib.core.frontmatter_schema` | `spec_driver.core.frontmatter_schema` |
| `spec_driver/domain/relations/manager.py` | `supekku.scripts.lib.core.spec_utils` | `spec_driver.core.spec_utils` |
| `spec_driver/domain/relations/graph.py` | `supekku.scripts.lib.core.artifact_ids` | `spec_driver.core.artifact_ids` |

This eliminates the "domain imports legacy" debt from DE-125.

## 3. Verification

- `uvx import-linter lint` after each tier
- `uv run pytest supekku spec_driver -x` ‚Äî full pass
- `uv run ruff check` ‚Äî clean
- Domain import audit: no `supekku.scripts.lib.core` imports remain in
  `spec_driver/domain/`

## 4. Consequences

### Positive

- `spec_driver.core` becomes the real foundation with 23 modules (22 migrated +
  `file_ops.py`).
- The outer layer contract becomes truthful ‚Äî core actually contains core.
- DE-125 legacy-core import debt eliminated.
- Future domain/orchestration migration can depend on `spec_driver.core`
  directly.

### Costs

- 22 re-export shims in `supekku/scripts/lib/core/`.
- Shim maintenance burden until consumers migrate.

### Not changed

- Consumer import paths (all go through shims).
- CLI behaviour.
- Test locations.
