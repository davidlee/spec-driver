## 1. Summary

- **Delta**: [DE-128](./DE-128.md)
- **Design Revision**: [DR-128](./DR-128.md)
- **Desired Outcome**: All 26 genuine core modules live in `spec_driver/core/`
  with re-export shims at legacy locations; `spec_utils`/`config` decoupled from
  upward deps; domain modules import from `spec_driver.core`.

## 2. Context & Constraints

- **Current** (2026-05-31): `spec_driver.core` has 3 modules (`file_ops`,
  `string_utils`, `yaml_emit`); 26 genuine core modules live in
  `supekku/scripts/lib/core/`.
- **Target**: ~28 core modules; outer layer contract truthful and
  linter-provably free of upward deps.
- **Pattern**: proven in DE-125/DE-127 (module → `spec_driver.core`, **test
  moves with it** per POL-003, legacy shim re-exports public names).
- **Re-validated** against DR-128 §2.4–2.5: inventory 22→26; spec_utils + config
  decoupled in-scope; enums/artifact_view/frontmatter_metadata out.

## 3. Phase Overview

| Phase | Tier | Units | Key outcome |
|-------|------|-------|-------------|
| P01 | 0 | 16 | Leaves land (incl. dates/ids/io; `slugify`→`string_utils`); `artifact_ids` + `frontmatter_schema` unblock domain |
| P02 | 1 | 4 | `paths`, `spec_utils` (RELOCATE create + new `write_markdown_file`), `frontmatter_writer`, `events`; **domain debt eliminated** |
| P03 | 2 | 2 | `config` (DI `known_kinds`), `templates` |
| P04 | 3 | 3 | `agent_docs`, `preboot`, `sync_preferences`; migration complete |

Each phase, per module: copy module **and its `*_test.py`** to `spec_driver/core/`
(repoint test imports, delete legacy test) → set internal imports relative →
create re-export shim at legacy module path → verify (lint, tests, contracts).
Decoupling work lands inside its tier: spec_utils relocation in P02, config DI in
P03 (DR-128 §2.4).

> **2026-05-31 note**: `registry_migration.py` was planned as a tier-2 module
> (added by DE-146) but does not exist in the source tree and has zero consumers.
> Excluded from P03. If it lands later, it follows the same pattern.

After P02: update `spec_driver/domain/relations/` imports (`manager.py`,
`graph.py`) to `spec_driver.core` — eliminates DE-125 legacy-core debt.
(`manager.py` needs only pure `load_markdown_file` + `dump_markdown_file_update`.)

## 4. Testing & Verification

- `uvx import-linter lint` after each phase — actively guards decoupling (core→
  orchestration / core→domain now fails the build).
- `uv run pytest supekku spec_driver -x` after each phase.
- `uv run ruff check` after each phase.
- After P02: no `supekku.scripts.lib.core` imports in `spec_driver/domain/`;
  migrated `core/spec_utils.py` imports no orchestration/domain symbol.
- After P03: migrated `core/config.py` imports no `frontmatter_metadata`; VT
  pins strict-kind warning fires with `known_kinds` injected, silent without.
- **Out of scope (follow-on)**: `spec_driver/orchestration/` legacy-core imports
  (`operations.py`, `artifact_view.py`, ~20 mock-patch refs in
  `operations_test.py`) — distinct task.

## 5. Risks

| Risk | Mitigation |
|------|------------|
| 40 consumers of `paths.py` break | Re-export shim; proven pattern |
| Relative import cycles within `spec_driver.core` | Tier ordering prevents cycles |
| Third-party imports (pydantic, jinja2, yaml) differ by location | Python resolution is path-independent |
| spec_utils relocate breaks 12 `dump_markdown_file_create` call sites | Legacy `core/spec_utils.py` shim re-exports it from new orchestration home; callers unchanged (P02) |
| config DI silently drops strict-kind warning everywhere | Single validating entrypoint (`validate/file.py:263`) passes `known_kinds`; `core/config_test.py` updated + VT pins both paths (P03) |
| Test-with-module migration repoints many `*_test.py` imports | Mechanical; linter + full pytest after each phase catch misses |
