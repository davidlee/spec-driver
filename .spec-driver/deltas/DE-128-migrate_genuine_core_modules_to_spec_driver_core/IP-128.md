## 1. Summary

- **Delta**: [DE-128](./DE-128.md)
- **Design Revision**: [DR-128](./DR-128.md)
- **Desired Outcome**: All 22 genuine core modules live in `spec_driver/core/`
  with re-export shims at legacy locations. Domain modules import from
  `spec_driver.core` instead of legacy paths.

## 2. Context & Constraints

- **Current**: `spec_driver.core` has 1 module (`file_ops.py`); 22 genuine core
  modules live in `supekku/scripts/lib/core/`.
- **Target**: `spec_driver.core` has 23 modules; outer layer contract is truthful.
- **Pattern**: proven in DE-125 (domain relations) and DE-127 (artifact_view,
  enums).

## 3. Phase Overview

| Phase | Tier | Modules | Lines | Key outcome |
|-------|------|---------|-------|-------------|
| P01 | 0 | 13 | ~1,200 | Leaves land; `artifact_ids` + `frontmatter_schema` unblock domain imports |
| P02 | 1 | 4 | ~640 | `paths`, `spec_utils` land; domain import debt eliminated |
| P03 | 2 | 2 | ~590 | `config`, `templates` land |
| P04 | 3 | 3 | ~300 | `agent_docs`, `preboot`, `sync_preferences` land; migration complete |

Each phase: copy modules → update internal imports to relative → create shims →
verify (lint, tests, contracts).

After P02: update `spec_driver/domain/relations/` imports to use
`spec_driver.core` — eliminates DE-125 legacy-core debt.

## 4. Testing & Verification

- `uvx import-linter lint` after each phase
- `uv run pytest supekku spec_driver -x` after each phase
- `uv run ruff check` after each phase
- After P02: no `supekku.scripts.lib.core` imports in `spec_driver/domain/`
- Note: `spec_driver/orchestration/` also has legacy-core imports
  (`operations.py`: config, paths, git; `artifact_view.py`: paths; plus ~20
  mock-patch references in `operations_test.py`). Updating those is desirable
  but out of scope — the test mock paths alone make it a distinct task.
  Tracked as follow-on.

## 5. Risks

| Risk | Mitigation |
|------|------------|
| 40 consumers of `paths.py` break | Re-export shim; proven pattern |
| Relative import cycles within `spec_driver.core` | Tier ordering prevents cycles |
| Third-party imports (pydantic, jinja2, yaml) differ by location | Python resolution is path-independent |
