## 1. Summary

- **Delta**: DE-129 — Requirement authoring and validation hardening
- **Specs Impacted**: SPEC-122 (requirements package), SPEC-125/SPEC-158 (validator/diagnostics — collaborators)
- **Problems / Issues**: ISSUE-051 (stale pruning), ISSUE-050 (tagging — related), IMPR-012 (ID deduplication)
- **Desired Outcome**: Requirement authoring errors are caught at sync/validate time with actionable diagnostics, not discovered months later during painful migrations.

## 2. Context & Constraints

- **Current Behaviour**: Parser silently ignores frontmatter requirements, misparsing compound IDs, no stale pruning. Validator doesn't check bare IDs or implements-target-kind. See DR-129 §1.1–§1.9.
- **Target Behaviour**: Nine changes across parser, sync, validator, show, and memory. See DR-129.
- **Dependencies**: None blocking. DE-119 (validator → diagnostics migration) is parallel.
- **Constraints**: New validator checks go in `WorkspaceValidator` (DE-119 hasn't landed). Parser functions receive `SyncStats` for warning counting.
