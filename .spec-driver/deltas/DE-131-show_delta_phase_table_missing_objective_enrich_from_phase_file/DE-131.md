## 1. Summary & Context

- **Product Spec**: PROD-006 – Phase Management
- **Implementation Plan**: [IP-131](./IP-131.md)
- **Change Driver**: User-reported: `show delta` phase table always displays `-` for Objective column

## 2. Motivation

`show delta` renders a phase table with Phase / Status / Objective columns. The Objective column always shows `-` because the data flow has two gaps:

- **Gap A (display)**: `_enrich_phase_data()` in `change_formatters.py` reads the phase file for tracking data but never extracts `objective` from frontmatter or overview block.
- **Gap B (creation)**: `_update_plan_overview_phases()` in `phase_creation.py` appends only `{"id": phase_id}` to the IP phases array — no objective propagated back.

The objective data exists in phase files (frontmatter and/or markdown body) but is never surfaced to the display path. Detailed root cause (including canonical phases that skip `phase.overview` at load time): [DR-131 §2](./DR-131.md).

## 3. Scope & Objectives

- **Primary Outcome**: Phase objective appears in `show delta` output when present in the phase file
- **Secondary Outcome**: `create phase` propagates objective into IP phases array when available

## 4. Out of Scope

- Parsing objective from markdown body (§ heading content) — only structured sources (frontmatter, overview block)
- Retroactively backfilling existing IP files

## 5. Approach Overview

- **System Touchpoints**: `formatters/change_formatters.py`, `changes/phase_creation.py`
- **Key Changes**:
  - `_enrich_phase_data()`: when enriched dict lacks `objective`, extract it from phase file frontmatter (primary) or `phase.overview` block (fallback)
  - `_update_plan_overview_phases()`: include `objective` in the appended phase entry when `phase_frontmatter` contains one

## 6. Verification Strategy

- **VT**: Unit test `_enrich_phase_data` returns objective from phase frontmatter
- **VT**: Unit test `_enrich_phase_data` returns objective from overview block when not in frontmatter
- **VT**: Unit test `_enrich_phase_data` preserves plan-level objective (no clobber)
- **VA**: `show delta` on a real delta with phase objectives displays them
- **Acceptance**: `show delta DE-115` (or similar) shows non-`-` objective values

## 7. Risks & Mitigations

- **Risk**: Phase files without objective in any structured location – _Likelihood_: high (many existing phases) – _Impact_: low (graceful `-` fallback already works) – _Mitigation_: No change needed, existing default handles it

## 8. Follow-ups & Tracking

- Consider a future improvement: parse objective from `## 1. Objective` markdown heading as a tertiary fallback
- Consider `plan-phases` skill update to encourage writing objective into frontmatter
