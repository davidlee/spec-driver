# DR-133 – Agent sync in installer

## 1. Executive Summary

- **Delta**: [DE-133](./DE-133.md)
- **Status**: draft
- **Synopsis**: Add agent definition sync to `spec-driver install`, copying `supekku/agents/*.md` → `.claude/agents/`

## 2. Problem & Constraints

- **Current Behaviour**: Agent definitions (e.g. `dispatch-worker.md`) are manually committed to `.claude/agents/`. Package updates don't propagate.
- **Drivers**: IMPR-031
- **Constraints**: Must be idempotent. Must not touch user-created agents. Must support `--dry-run`.

## 3. Architecture Intent

- **Target Outcome**: Agent definitions ship with the package and are synced on install, same as skills/memories/hooks.
- **Pattern**: Follow `_install_claude_config` — direct copy, installer-owned, overwrite on install.

## 4. Code Impact Summary

| Path | Current State | Target State |
| --- | --- | --- |
| `supekku/agents/` | does not exist | new directory; package source for agent `.md` files |
| `supekku/agents/dispatch-worker.md` | n/a | moved from `.claude/agents/dispatch-worker.md` |
| `supekku/scripts/install.py` | no agent sync | `_install_agents()` added; called from `initialize_workspace()` |

## 5. Design

### `_install_agents(package_root, target_root, *, dry_run)`

1. Locate `package_root / "agents"` — return early if absent
2. Glob `*.md` files from source
3. For each source file:
   - Copy to `.claude/agents/<name>` (overwrite if exists and differs)
   - Skip if content identical
4. Dry-run: print planned actions without writing

No pruning of user-created agents (they're not in the package source, so they're ignored).

### Reporting

Print summary consistent with existing install output:
- `Agents: installed N` / `Agents: up to date`
- Dry-run prefixed with `[DRY RUN]`
