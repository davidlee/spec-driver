## 3. Scope & Objectives

- **Primary Outcomes** (DR-136 §5 deliverables):
  1. **Y — Template generation from metadata** (DR-136 §5.1): `BlockMetadata.render_template_frontmatter()`, comment-preserving emitter (ruamel.yaml), `spec-driver admin regenerate-templates`, `spec-driver validate templates` CI gate, split-aware template copy in installer.
  2. **Z — Actionable validation + alias autocorrect** (DR-136 §5.2): `FieldMetadata.aliases` (permanent) and `FieldMetadata.tolerated_aliases` (sunset-bounded), did-you-mean utility in `core/string_utils.py`, `MetadataValidator.validate(strict: bool)`, `validate --fix`, retire `normalize_status()`.
  3. **X — `schema enums` CLI** (DR-136 §5.3): `spec-driver schema enums [kind[.field]]` with rich-table output via Workspace (ADR-011).
  4. **`validate file <path>`** (DR-136 §5.4): single-file validation surface, `path:line:col: severity: message` format, non-zero exit on error — closes ISSUE-054 authoring-time gap.
  5. **`validate --kind` filter** (DR-136 §5.4a): per-kind validation sweep flag composing with `--strict`, `--no-tolerated-aliases`, `--fix`.
  6. **Skill tuning** (DR-136 §5.5): verbatim gate insertions into `execute-phase`, `close-change`, `audit-change`, `notes`, `update-delta-docs` skills.
  7. **`admin migrate` orchestrator** (DR-136 §5.6): `--check` / `--list` / `<kind>` / `<kind> --dry-run` / `<kind> --strict` / `all`; dispatches to `migrations/v0X_*.py`; per-kind state in `workflow.toml [validation.strict]` + `[schema_version]`; per-run logs in `.spec-driver/run/migrations/`.
  8. **`workflow.toml` schema additions**: `[validation.strict]` per-kind table + `[schema_version]` per-kind table; new-install strict-on default (DR-136 §11.3 design constraint 2).
