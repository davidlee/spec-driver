## 3. Scope & Objectives

- **Primary Outcomes** (DR-136 §5 deliverables — DR-137 locks the residual mechanics):
  1. **Y — Template generation from metadata** (DR-136 §5.1; DR-137 §5.1): shared `spec_driver/orchestration/templates.py:render_frontmatter_for_kind` consumed by both the regenerator and `dump_markdown_file_create` (POL-001); comment-preserving emit via custom `spec_driver/core/yaml_emit.py` (stdlib yaml; OQ-137-01 covers ruamel-fallback); `spec-driver admin regenerate-templates`, `spec-driver validate templates` CI gate. **`dump_markdown_file` is split** into `*_create(..., kind=)` (emit comments) and `*_update(...)` (preserve existing comments) — every caller migrates explicitly.
  2. **Z — Actionable validation + alias autocorrect** (DR-136 §5.2; DR-137 §5.2): two-axis alias mechanism (DEC-137-23 / F-30) — `BlockMetadata.field_aliases` for field-NAME parse-time key rename (e.g. `annotation → nature`) and `FieldMetadata.aliases` + `FieldMetadata.tolerated_aliases` for field-VALUE normalisation (e.g. `complete → completed`). `ValidationError.fix_kind ∈ {rename_key, rewrite_value}` routes `--fix`. Did-you-mean via stdlib `difflib.get_close_matches(cutoff=0.6)` in `spec_driver/core/string_utils.py`. `MetadataValidator.validate(*, strict, accept_tolerated)`. Retire `normalize_status()` for `normalize_field(kind, field, value)`. `ENUM_REGISTRY` splits (Category A artefact-frontmatter ⇒ derived view; Category B verification.kind/backlog.status-umbrella/drift.status/command.format/requirement.kind/spec.kind ⇒ hardcoded or relocate per-block).
  3. **X — `schema enums` CLI** (DR-136 §5.3; DR-137 §5.3): `spec-driver schema enums [kind[.field]]` with rich-table output via Workspace (ADR-011).
  4. **`validate file <path>`** (DR-136 §5.4; DR-137 §5.4): single-file validation surface; diagnostic shape `<path>: <severity>: <dotted.field.path>: <message>` for semantic errors and `<path>:<line>:<col>: parse-error: ...` for YAML-parse-stage errors; non-zero exit on error.
  5. **`validate --kind` filter** (DR-136 §5.4a; DR-137 §5.4): under the new `validate workspace` named peer of the Typer group. Bare `spec-driver validate` prints help (no default dispatch). Composes with `--strict`, `--no-tolerated-aliases`, `--fix`.
  6. **Skill tuning** (DR-136 §5.5; DR-137 §5.5): verbatim gate insertions bracketed by `<!-- validate-gate:<skill> begin/end -->` anchors into `execute-phase`, `close-change`, `audit-change`, `notes`, `update-delta-docs` skills.
  7. **`admin migrate` orchestrator** (DR-136 §5.6; DR-137 §5.6): `--check` / `--list` / `<kind>` / `<kind> --dry-run` / `<kind> --strict` / `all`; dispatches to migrations in `spec_driver/migrations/v<M>_<m>_<p>_<NNN>_<slug>/`; single `[migrations] last_applied` watermark; per-kind `[validation.strict]` toggle; lockfile-guarded single-process invariant; per-run logs in `.spec-driver/run/migrations/`. Migration steps are singular-kind (multi-kind forbidden); imports limited to `_protocol`, `_helpers`, `_folder`, stdlib, pinned pyyaml.
  8. **`workflow.toml` schema additions**: `[migrations] last_applied` + per-kind `[validation.strict]` table + optional `[schema_version]` derived cache; strict-on-default trigger keyed to `.spec-driver/` workspace absence at install time (NOT `workflow.toml` presence).
- **Inherited scope-notes from DE-136 Phase 1** (resolved during DR-137 drafting):
  - **F-B**: relations key naming resolved — `nature` is canonical (DEC-137-05); `annotation` becomes a permanent alias on the relations FieldMetadata. Locked in DR-137.
