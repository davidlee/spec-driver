## 3. Scope & Objectives

- **Primary Outcomes** (DR-136 §6):
  1. **Field placement** (§6.1): `applies_to` becomes a derived in-memory property from `supekku:delta.relationships@v1`; `context_inputs` → `supekku:delta.context_inputs@v1` block; `risk_register` → `supekku:delta.risk_register@v1` block; `outcome_summary` → prose body §X (appended as `## Outcome` section before `## Implementation Notes`); `audit_gate` + `audit_gate_rationale` stay FM.
  2. **Body reshape** (§6.2): delete §7 Risks (block canonical); keep §§1–6, 8–9.
  3. **`list deltas` enrichment** (§6.3): default columns ID, Name, Status, Specs (count + first ID), Audit gate (non-default only), Audit (yes/no glyph), Phases (`N/M`).
  4. **`applies_to` derivation** (§6.4): `ChangeArtifact.applies_to` is computed from parsed block; transition-window fallback reads block-first then frontmatter-second; `changes/coverage_check.py` refactored to consume the property.
  5. **Migration** (§6.5): `migrations/v02_delta_blocks.py` — §4 universals cut, FM `applies_to` removed (block synthesised if missing), `risk_register` FM → block, `context_inputs` FM → block, `outcome_summary` FM → body. Drift entries for irreconcilable disagreements.
- **Operational Constraints**:
  - DE-137 must close first — provides the orchestrator and per-kind state tracking.
  - Tolerant-on-read held until per-kind strict flip (DR-136 §11.3 design constraint 1).
  - PROD-004 FR-007 is *first exercised here*; verify DE-036 compaction infrastructure under realistic delta corpus.
- **Dependencies**:
  - **Hard**: DE-118 (closed), DE-137 (closed).
  - **Independent of**: per-artefact siblings (spec, requirements-in-spec, audit, revision); no cross-kind coupling.
