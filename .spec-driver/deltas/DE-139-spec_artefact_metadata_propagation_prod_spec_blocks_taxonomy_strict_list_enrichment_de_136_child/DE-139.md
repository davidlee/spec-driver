## 3. Scope & Objectives

- **Primary Outcomes** (DR-136 §7 + F-A PROD extension):
  1. **Tech SPEC field placement** (§7.1): `category` (`unit|assembly`) FM strict, tolerated `unknown`; `c4_level` FM strict, tolerated `unknown`; `scope` → prose body §2; `concerns` → `supekku:spec.concerns@v1` block; `hypotheses` → `supekku:spec.hypotheses@v1` block; `decisions` → `supekku:spec.decisions@v1` block; `responsibilities`/`guiding_principles`/`assumptions`/`constraints` stay FM (body duplicates deleted); `verification_strategy` delegated to existing `supekku:verification.coverage@v1` block in tests companion; `packages` CUT.
  2. **PROD field placement (F-A extension authored in DR-139)**: PROD's `hypotheses` (id/statement/status objects) → `supekku:prod.hypotheses@v1` block (or shared schema if compatible); `decisions` → `supekku:prod.decisions@v1`; `product_requirements` → block; `category` (freeform) — placement decision (keep FM as freeform tag? CUT? DR-139 resolves); `value_proposition`/`scope`/`problems`/`guiding_principles`/`assumptions` — FM; no `c4_level` field exists for PROD.
  3. **Body reshape** (§7.2): tech SPEC delete §§3 (Responsibilities), 4 (Concerns), 5 (Hypotheses/Decisions); PROD body reshape mirrors per F-A.
  4. **`list specs` enrichment** (§7.3): default columns ID, Name, Status, Category, C4 (one-letter glyph), Sources (count + first language).
  5. **Strict taxonomy migration** (§7.4 — burden realism): ~60 in-tree specs, plausible 50–80 drift entries; tolerated_alias `unknown` added by sweep; manual reconciliation tracked but **not gating spec-delta close**; reconciliation is a follow-up batch (or small cleanup delta) sequenced before umbrella close.
  6. **Migration** (§7.5): `migrations/v03_spec_blocks.py` — §4 universals + `packages` + `verification_strategy` FM duplication cut; concerns/hypotheses/decisions FM → blocks; `category`/`c4_level` set to `unknown` if missing (tolerated_alias) with drift; body cleanup deferred to interactive `validate --prune-duplicates`. Migration step covers BOTH `kind: spec` and `kind: prod` (or split into v03a/v03b — DR-139 decides).
- **Operational Constraints**:
  - DE-137 closed first.
  - Spec-delta close gate excludes manual unit/assembly reconciliation (per §7.4 burden-realism note).
  - PROD coverage: F-A extension is **authoritative** for PROD placement; do not silently treat PROD as "out of scope" — that defeats DE-136's PROD-004 closure target.
- **Dependencies**:
  - **Hard**: DE-118, DE-137, DE-138 (sequence-precedent for sweep procedure).
