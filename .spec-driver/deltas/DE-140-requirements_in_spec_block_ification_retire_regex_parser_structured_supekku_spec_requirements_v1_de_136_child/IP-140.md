## 1. Summary

- **Delta**: DE-140 — Requirements-in-spec block-ification
- **Design**: DR-140 (16 design decisions, adversarially reviewed)
- **Specs Impacted**: PROD-004, SPEC-114
- **Desired Outcome**: Requirements in spec/product files are defined via structured `supekku:spec.requirements@v1` YAML blocks. Regex parsing is fallback-only for unmigrated specs. Strict enforcement via schema_version flip after full corpus migration.

## 2. Context & Constraints

- **Dependencies**: DE-118 (completed), DE-137 (completed), DE-139 (completed — strict-flip pattern)
- **Migration isolation**: DEC-138-12 — migration steps import only stdlib + helpers + protocol + pyyaml
- **POL-001**: block-first, regex fallback, never both. No competing parsers.
- **POL-003**: blocks/ (infrastructure), requirements/ (domain), validation/ (wiring)
- **Corpus**: 15 PROD specs with regex-format requirements. No tech specs with requirements.
