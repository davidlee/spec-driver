## 3. Scope & Objectives

- **Primary Outcomes** (DR-136 §9 + F-C/F-D resolution):
  1. **Field placement** (§9.1): `mode` (`conformance|discovery`) FM strict; `delta_ref`/`spec_refs`/`prod_refs`/`code_scope` FM; `audit_window` (start/end) FM **optional** (F-C); `findings` rich list-of-objects → `supekku:audit.findings@v1` **block** with strict-validated per-finding `outcome` enum (F-D).
  2. **Body reshape** (§9.2): keep Observations, Recommendations, optional Evidence; delete prose Findings section (block canonical).
  3. **Strict enforcement** (§9.3): per-finding `outcome` enum strictly enforced (`aligned | drift | risk`); AUD-012's `outcome: pass` per-finding becomes strict-mode error; disposition `status × kind` and `outcome × kind` matrices tighten warn → error; completed audit with undisposed finding fails strict mode; `closure_override.effect` must be less strict than derived gate; `rationale` required.
  4. **`list audits` enrichment** (§9.4): default columns ID, Name, Status, Mode (C/D glyph), Delta (truncated `delta_ref`), Findings (count + outcome breakdown e.g. `7 (3a/2d/2r)`), Disposed (e.g. `5/7`).
  5. **Migration** (§9.5): `migrations/v05_audit_findings.py` — §4 cuts; `findings` FM → block (lossless); AUD-012-style invalid enums NOT auto-fixed (drift entry; manual disposition); strict mode flips after sweep clean.
- **Operational Constraints**:
  - DE-137 closed first.
  - AUD-012 explicitly excluded from auto-fix (per §9.5); needs manual disposition before strict flip.
  - Per-finding outcome canonicalisation lives in block schema, not FM.
- **Dependencies**:
  - **Hard**: DE-118, DE-137, DE-138.
