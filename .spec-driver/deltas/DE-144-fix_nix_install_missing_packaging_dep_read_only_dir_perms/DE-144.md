## 1. Summary & Context

- **Technical Spec**: SPEC-129 – sync installer-component (skills install/prune).
- **Change Drivers**: Two crash-on-`sdr install` defects found installing the post-DE-136 (umbrella) `0.10.0` build into a real project.

## 2. Motivation

Nix-built `spec-driver-0.10.0` could not `install`:

1. `ModuleNotFoundError: No module named 'packaging'` — DE-136 added `from packaging.version import Version` (`spec_driver/migrations/_folder.py`) and `packaging>=23` to `pyproject.toml`, but `flake.nix` hand-maintains its Python dep list and was never updated, so the Nix closure omitted `packaging`.
2. `PermissionError: [Errno 13]` removing `.spec-driver/skills/audit-change` — `copytree_with_write_permission` made copied *files* writable but not *directories*. Skill dirs copied from the read-only Nix store stayed read-only; `shutil.rmtree` unlinks children via the parent dir, which needs write.

## 3. Scope & Objectives

- **Primary Outcomes**:
  - `flake.nix` dep list matches `pyproject.toml` runtime deps (adds `packaging`).
  - `copytree_with_write_permission` makes destination directories writable, not just files.
  - New `remove_tree` helper force-removes read-only trees; skills-sync `rmtree` sites route through it, self-healing prior bad installs.
- **Operational Constraints**: No lint regressions; no `pyproject.toml` rule changes.
- **Dependencies**: None.

## 4. Out of Scope

- Automated guard that `flake.nix` deps ⊇ `pyproject.toml` deps (follow-up, see §7).
- Deduplicating the `supekku` → `spec_driver` shim layer.

## 5. Approach Overview

- **System Touchpoints**: `flake.nix`, `spec_driver/core/file_ops.py`, `supekku/scripts/lib/file_ops.py` (shim re-export), `supekku/scripts/lib/skills/sync.py`.
- **Key Changes**:
  - Add `packaging` to `flake.nix` `dependencies`.
  - `copytree_with_write_permission`: walk and `chmod u+w` directories as well as files.
  - Add `remove_tree(path)`; replace 3 `shutil.rmtree` sites in `sync.py` with it.
- **Migration / Rollout Notes**: Requires `nix build` / reinstall to take effect (immutable store path). Re-install self-cleans existing read-only skill dirs via `remove_tree`.

## 6. Verification Strategy

- **Acceptance Criteria**: `sdr install` succeeds on a Nix build into a project with prior read-only skill dirs; no `ModuleNotFoundError` / `PermissionError`.
- **Verification Artefacts (VT)**:
  - `test_copytree_with_write_permission_readonly_source_dirs` — proven red on old code, green on fix.
  - `test_remove_tree_readonly_dirs`, `test_remove_tree_missing_path_is_noop`.
  - Full suites: 25 `file_ops` + 62 skills-sync tests pass; ruff clean; no new pylint warnings.

## 7. Follow-ups & Tracking

- **Future work**: guard test asserting `flake.nix` runtime deps ⊇ `pyproject.toml` `[project].dependencies` so dep drift can't recur silently.

## 8. Implementation Notes

- Verify build-affecting changes with an actual `nix build` + run, not just `uv run` (uv reads `pyproject`; Nix does not). See memory `flake-deps-mirror-pyproject`.
