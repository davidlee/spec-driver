## 1. Context

- **Why**: Unit tech specs were introduced to disambiguate generated, code-bound stubs from human-authored assembly specs (ADR-003), but operationally they still add complexity and low signal in human workflows. The practical value has converged on contracts-first plus assembly specs.
- **Source Specs**: PROD-012, PROD-015
- **Destination Specs**: PROD-012, PROD-015
- **Requirements Affected**:
  - PROD-015.FR-002 (unit defaults) -> revised to assembly-only creation behavior
  - PROD-015.FR-003 (CLI taxonomy filtering) -> revised to assembly-first filtering with legacy compatibility
  - PROD-015.FR-004 (category views) -> revised to assembly/unknown-focused navigation
  - PROD-015.FR-005 (taxonomy validation) -> revised to deprecate `category: unit` instead of promoting it

## 3. Summary of Changes

- **Specs Updated**:
  - PROD-015: shift from "unit vs assembly split" to "assembly-first with no new unit-spec creation".
  - PROD-012: remove expectation that code units map to generated unit specs; keep contracts corpus as primary code-facing artifact.
- **Requirements**:
  - Unit creation defaults are deprecated as a product goal.
  - Category/level filters remain for navigation, but unit as a first-class target is removed from default workflows.
- **Responsibilities / Relations**:
  - Clarify that contracts represent observed code truth.
  - Clarify that tech specs represent intent/constraints at assembly scope.
  - Keep backward compatibility for existing `category: unit` metadata during transition.

## 4. Consequences

- **Positive**:
  - Reduces conceptual surface area and onboarding burden.
  - Aligns tooling behavior with actual usage: contracts + assembly specs.
  - Removes incentives for large-scale low-value spec generation in legacy repos.
- **Trade-offs**:
  - Existing docs/tests that assume unit-spec taxonomy must be updated.
  - ADR-003 terminology and consequences require explicit follow-up (supersede or amendment).
- **Impact on audits/testing**:
  - Update VT coverage tied to FR-002/003/004/005 under PROD-015 and FR-005 under PROD-012.
  - Add migration tests for legacy `category: unit` handling.
