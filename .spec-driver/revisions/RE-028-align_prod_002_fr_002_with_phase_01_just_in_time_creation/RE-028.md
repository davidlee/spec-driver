## 1. Context

- **Why**: PROD-011 FR-001 explicitly reversed PROD-002 FR-002's phase-01
  auto-creation behavior. ADR-004 (now accepted) canonically states that phase
  sheets are created just-in-time and interleaved with execution, not scaffolded
  up front. PROD-002 was never revised to reflect this, so agents reading it in
  isolation implement the wrong behavior.
- **Source Specs**: PROD-011 (FR-001: phase-01 MUST NOT auto-create during delta creation)
- **Destination Specs**: PROD-002 (FR-002: default creates delta + DR + plan + first phase)
- **Requirements Affected**: PROD-002.FR-002
- **Drift Ledger**: DL-047.001, DL-047.011

## 2. Related Artefacts

- **Decisions / ADRs**: ADR-004 (canonical core workflow loop, section 1: phases created just-in-time)
- **Drift Ledger**: DL-047 entries .001 and .011
- **Delta**: DE-047 (spec corpus reconciliation)

## 3. Summary of Changes

- **PROD-002 FR-002**: Remove "first phase" from the default delta bundle.
  Default bundle becomes: delta file + design revision + implementation plan
  (with `--skip-plan` to omit plan). Phase sheets are created explicitly and
  just-in-time per ADR-004.
- **PROD-002 assumptions**: Remove "Default workflow creates plan + first phase".
- **PROD-002 capabilities**: Update `frictionless-delta-creation` summary and
  `integrated-validation` responsibilities to remove phase-01 auto-creation
  references.
- **PROD-002 verification coverage**: VT-002 updated to reflect three-artifact
  bundle (delta + DR + plan), not four.
- **PROD-002 journeys/flows**: Update Journey 1, Flow 1 to remove auto-created
  phase from output. Update Flow 3 (lightweight) to clarify delta + DR only.

## 4. Consequences

- Positive: PROD-002 aligns with PROD-011, ADR-004, and runtime behavior.
  Agents reading PROD-002 will no longer attempt to auto-create phase-01.
- Positive: Eliminates a blocking contradiction in the spec corpus.
- Neutral: No code changes required — runtime already behaves this way.
