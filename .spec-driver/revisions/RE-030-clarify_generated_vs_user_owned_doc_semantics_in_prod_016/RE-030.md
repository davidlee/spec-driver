## 1. Context

- **Why**: PROD-016 describes generated guidance and doctrine hooks but does not
  explicitly distinguish which files are generated (overwritten on re-install)
  vs user-owned (seeded once, never overwritten). This caused DL-047.014 to flag
  a missing decision. The distinction is already clear in implementation
  (`.spec-driver/hooks/README.md`) but not in the spec itself.
- **Destination**: PROD-016
- **Requirements Affected**: FR-007 (customization hooks), FR-008 (generated guidance)
- **Drift Ledger**: DL-047.014

## 2. Related Artefacts

- **Delta**: DE-047 (spec corpus reconciliation)

## 3. Summary of Changes

- **PROD-016 section 5.2**: Add explicit ownership semantics for installer-managed files:
  - **Generated (overwritten)**: `.spec-driver/agents/*.md` — config-tailored guidance
    generated from `workflow.toml`; safe to overwrite on re-install.
  - **User-owned (seeded once)**: `.spec-driver/hooks/doctrine.md` — bespoke project
    instructions; seeded by installer with defaults, never overwritten.
- **FR-007 note**: Clarify that hooks directory is user-owned space.
- **FR-008 note**: Clarify that agents directory is generated space.

## 4. Consequences

- Positive: Agents and users understand which files they can safely edit vs which
  will be regenerated.
- Neutral: No code or behavior changes — this is documentation precision.
