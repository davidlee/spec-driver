## 1. Context

- **Why**: ADR-008 (accepted) establishes that specs own normative lifecycle
  truth and the requirements registry is a derived view. PROD-008 already aligns
  with this in spirit (FR-001: specs are authoritative) but needs precision:
  clarify that audits trigger explicit reconciliation rather than silently
  overriding spec truth, and that the registry is derived, not authoritative.
- **Source**: ADR-008
- **Destination**: PROD-008
- **Requirements Affected**: FR-001 (add normative/observed distinction), FR-003 (audits trigger reconciliation, not override)
- **Drift Ledger**: DL-047.009, DL-047.017

## 2. Related Artefacts

- **Decisions / ADRs**: ADR-008, ADR-004
- **Delta**: DE-047

## 3. Summary of Changes

- **FR-001**: Add clarification that specs own *normative* lifecycle truth;
  evidence overlays record *observed* truth; the requirements registry is a
  derived projection, not an authoritative source.
- **FR-003**: Clarify that audits surface drift for explicit reconciliation —
  they do not silently override spec truth by timestamp or precedence.
- **Guiding principles**: Strengthen "single canonical source" to "normative
  source; observed evidence triggers reconciliation".
- **Solution outline**: Clarify registry sync contracts as derived views.

## 4. Consequences

- Positive: PROD-008 aligns with ADR-008 and ADR-004 doctrine.
- Neutral: No behavioral changes — PROD-008 already leans this way.
