## 1. Context

- **Why**: ADR-008 (accepted) establishes that drift between normative and
  observed truth is resolved explicitly, never by timestamp precedence. PROD-009
  currently describes a merge model where "newest evidence wins" and audits
  outrank deltas by timestamp. This contradicts ADR-008 and must be revised.
  Additionally, ADR-008 formalizes `asserted` and `legacy_verified` — PROD-009
  introduces these statuses but lacks definitions.
- **Source**: ADR-008
- **Destination**: PROD-009
- **Requirements Affected**: FR-001 (add status definitions), FR-002 (replace timestamp precedence with explicit reconciliation)
- **Drift Ledger**: DL-047.009, DL-047.017, DL-047.018

## 2. Related Artefacts

- **Decisions / ADRs**: ADR-008, ADR-004
- **Delta**: DE-047

## 3. Summary of Changes

- **Guiding principles**: Replace "Later evidence overlays earlier declarations
  but never silently discards them" with language about explicit reconciliation.
- **FR-001**: Add formal definitions for `asserted` and `legacy_verified` per
  ADR-008 section 6.
- **FR-002**: Replace timestamp precedence model with explicit reconciliation
  model — evidence surfaces drift for human resolution, does not silently
  determine effective status.
- **Solution outline / Status Model**: Remove "newest timestamp wins" and
  "audits outrank deltas" precedence rules; replace with drift surfacing model.
- **Flows**: Update "Evidence overlay" and "Observation drift" flows to describe
  explicit reconciliation rather than automatic authority transfer.
- **Capability summary**: Replace "most recent authoritative signal" with
  "explicit reconciliation after evidence surfaces drift".

## 4. Consequences

- Positive: PROD-009 aligns with ADR-008 and eliminates the core contradiction
  with PROD-008.
- Negative: Some existing "effective status" implementation may need follow-up
  code changes to surface drift warnings instead of silently applying precedence.
