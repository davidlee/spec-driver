## 1. Context

- **Why**: FR-006 was revised to make `.contracts/` canonical, but behavioral
  flows, examples, directory structure diagrams, and CLI examples throughout
  PROD-012 still reference the old `specify/tech/SPEC-*/contracts/` write path
  and `by-language/by-package` symlink trees as primary navigation. ADR-007
  (accepted) settles this: mirror-of-source at `.contracts/` is primary;
  by-language/by-package are derived navigation indexes.
- **Source**: ADR-007
- **Destination**: PROD-012
- **Requirements Affected**: FR-006 (already correct), FR-009 (MUST→MAY for
  by-\* trees, clarified as derived), FR-016 (symlink→navigation structures)
- **Drift Ledger**: DL-047.002, DL-047.004, DL-047.012, DL-047.013

## 2. Related Artefacts

- **Decisions / ADRs**: ADR-007
- **Delta**: DE-047

## 3. Summary of Changes

- **Capability descriptions**: spec-bundle storage → canonical corpus; symlink
  trees → mirror-of-source corpus + optional derived indexes
- **FR-009**: MUST create symlink trees → MAY create derived navigation indexes
- **FR-016**: Symlink trees → navigation structures (mirror-of-source + indexes)
- **Flows 1, 5, 6, 7, 8, 9**: Updated contract write/read paths from
  `specify/tech/SPEC-*/contracts/` to `.contracts/`
- **Journey 2**: Updated contract organization reference
- **Directory structure example**: Replaced spec-bundle contracts and by-\*
  symlinks with mirror-of-source corpus diagram
- **CLI examples**: Updated to use `.contracts/` paths

## 4. Consequences

- Positive: All flows and examples now consistent with FR-006 and ADR-007.
- Positive: Resolves 4 drift ledger entries.
- Neutral: FR-009 relaxed from MUST to MAY — by-\* trees remain supported but
  are not mandatory.
