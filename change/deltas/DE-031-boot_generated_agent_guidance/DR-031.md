# DR-031 – boot_generated_agent_guidance

## 1. Executive Summary
- **Delta**: [DE-031](./DE-031.md)
- **Status**: draft (update when approved)
- **Owners / Team**: <names or teams accountable>
- **Last Updated**: 2026-03-01
- **Synopsis**: Generate `.spec-driver/agents/*.md` modules from templates using `.spec-driver/workflow.toml` so `/boot` can load token-cheap, project-correct guidance.

## 2. Problem & Constraints
- **Current Behaviour**: Installer copies static agent docs; they cannot embed correct project paths or hide disabled primitives.
- **Drivers / Inputs**:
  - `specify/product/PROD-016/PROD-016.md` (FR-008 `.spec-driver/agents/**` generated guidance)
  - `docs/commands-workflow.md` (ceremony modes and closure contract)
- **Constraints / Guardrails**:
  - Keep generated modules short and composable.
  - Do not make skills parse config dynamically; they should `@reference` generated modules.
- **Out of Scope**: OpenSkills allowlist exposure (DE-032).

## 3. Architecture Intent
- **Target Outcomes**:
  - Generate `.spec-driver/agents/exec.md`, `workflow.md`, `glossary.md`, `policy.md`.
  - Tailor output to configured paths and enabled primitives to conserve tokens.
  - Keep `initialize_workspace()` idempotent and safe to re-run.
- **Guiding Principles**:
  - Templates are tiny and focused; use doctrine as the escape hatch for bespoke text.
  - Default to legacy-friendly assumptions when config is incomplete.

## 4. Code Impact Summary
| Path | Current State | Target State |
| --- | --- | --- |
| `supekku/scripts/install.py` | Copies static `agents/` dir | Renders `.spec-driver/agents/*.md` from templates using workflow TOML |
| `supekku/templates/` | No agent-doc templates | Add `agents/*.md` templates for the four modules |
