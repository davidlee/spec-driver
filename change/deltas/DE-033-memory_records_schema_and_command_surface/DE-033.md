## 2. Motivation
- We need a memory system that is deterministic and practical, but aligned with spec-driver conventions.
- The current brief uses a metadata contract and command shape that will fragment the UX unless normalized now.
- This delta defines and implements the normalization so memory becomes a first-class, schema-validated artifact family.

## 3. Scope & Objectives
- **Primary Outcomes**:
  - Introduce memory artifact schema that fits existing frontmatter and block validation infrastructure.
  - Introduce memory command surface following verb-noun conventions.
  - Provide deterministic filtering/ranking behavior for hook-like context surfacing.
- **Operational Constraints**:
  - Keep memory pointers aligned with canonical artifacts to avoid competing truths.
  - Keep CLI files thin and delegate formatting/selection to library modules.
- **Dependencies**:
  - None hard-blocking; this delta establishes the baseline.

## 4. Out of Scope
- Embeddings/vector retrieval.
- Automatic rewriting of ADR/spec/design docs from memory.
- Multi-repo or remote memory synchronization.

## 5. Approach Overview
- **System Touchpoints**:
  - `supekku/cli/main.py`
  - `supekku/cli/{create,list,show,view,edit,find}.py`
  - `supekku/scripts/lib/core/frontmatter_metadata/`
  - `supekku/scripts/lib/formatters/`
  - `supekku/scripts/lib/<memory domain package>`
- **Key Changes**:
  - Add memory artifact model + registry with deterministic selection/filtering.
  - Add memory frontmatter + optional YAML block metadata definitions and validation tests.
  - Add CLI commands as verb-noun pairs for creation, discovery, inspection, and lifecycle actions.
  - Define companion memory skills (`capturing-memory`, `retrieving-memory`, `maintaining-memory`; optional `reviewing-memory`) as external deliverables in `spec-driver-skills`.
- **Migration / Rollout Notes**:
  - Existing `change/revisions/RE-017-memory_schema_and_cli_taxonomy_alignment/mem.brief.md` becomes reference input and test fixture, not canonical artifact format.

- **Risk**: Command sprawl and naming inconsistency – *Likelihood*: medium – *Impact*: high – *Mitigation*: keep operations in existing verb groups and add CLI snapshot tests.
