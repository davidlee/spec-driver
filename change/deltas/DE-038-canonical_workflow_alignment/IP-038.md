## 1. Summary
- **Delta**: DE-038 - canonical workflow alignment
- **Specs Impacted**: none directly in this delta (documentation/memory/skill bundle only)
- **Problems / Issues**:
  - competing workflow narratives (delta-first vs revision-first);
  - stale lifecycle/status claims in high-visibility docs/memories;
  - weak discoverability of the coverage close-out prerequisite.
- **Desired Outcome**: implementation-ready documentation contract for `strict_mode` plus aligned memory/skill guidance with explicit adoption gaps.

## 2. Context & Constraints
- **Current Behaviour**:
  - runtime is permissive by default;
  - primary close-out gate is coverage verification in parent specs during `complete delta`;
  - ceremony remains advisory (no ceremony-based runtime branch).
- **Target Behaviour**:
  - DE/DR/research define a single coherent canonical narrative and strict-mode contract;
  - strict mode baseline is hard-fail for non-canonical paths;
  - recommended v1 coverage precedence and mixed-status warning policy are documented.
- **Dependencies**:
  - DE-038, DR-038, and `workflow-research.md` in this bundle;
  - source-of-truth scripts/docs listed in `workflow-research.md`.
- **Constraints**:
  - no runtime behavior change in this delta;
  - documentation/design/memory/skill updates only.
