## 1. Summary & Context

`supekku/templates/` currently conflates three distinct concerns:

| Concern | Examples | Installed to | User-serviceable? |
|---------|----------|-------------|-------------------|
| **Agent rendering** | `boot.md`, `agents/*.md` | Rendered dynamically by installer | No |
| **Artifact creation** | `ADR.md`, `delta.md`, `spec.md` | `.spec-driver/templates/` | Yes |
| **Hook files** | `doctrine.md` | Not installed (bug) | Yes |

## 2. Motivation

- `doctrine.md` is referenced by `/boot` as `@.spec-driver/doctrine.md` but never installed there
- `boot.md` (agent template) leaks into `.spec-driver/templates/` alongside user artifact templates
- No mechanism exists to install user-customizable hook files to a stable path

## 3. Scope & Objectives

- **O1**: Move `boot.md` into `supekku/templates/agents/` so it stops being copied as an artifact template
- **O2**: Create `supekku/templates/hooks/` for user-customizable files installed to `.spec-driver/hooks/`
- **O3**: Move `doctrine.md` from `supekku/templates/` to `supekku/templates/hooks/`
- **O4**: Add install step: copy `hooks/*` to `.spec-driver/hooks/` (create-if-missing semantics, like seeds)
- **O5**: Exclude subfolders (`agents/`, `hooks/`) from the artifact template copy step

## 4. Out of Scope

- Refactoring artifact template rendering (stays as-is)
- Changing agent template rendering pipeline (`_render_agent_docs`)

## 5. Approach

### File moves

```
supekku/templates/boot.md        → supekku/templates/agents/boot.md
supekku/templates/doctrine.md    → supekku/templates/hooks/doctrine.md
```

### Install changes (`install.py`)

1. Template copy step: already uses `pattern="*.md"` (top-level only) — verify subfolders excluded
2. New step: copy `hooks/*` to `.spec-driver/hooks/` with create-if-missing (never overwrite user edits)
3. Update `_render_boot_md` path if needed (currently uses `render_template("boot.md", ...)`)

### Skill updates

- `/boot` reference changes from `@.spec-driver/doctrine.md` to `@.spec-driver/hooks/doctrine.md`

## 6. Verification

- VT: existing install tests still pass (template copy excludes subfolders)
- VT: new test — hooks installed to `.spec-driver/hooks/`
- VT: new test — existing hook files not overwritten on reinstall
- VH: `spec-driver install --yes` in clean dir produces `.spec-driver/hooks/doctrine.md`

## 7. Risks

- **Risk**: `pattern="*.md"` glob might match subfolder files on some platforms — *Likelihood*: low — *Mitigation*: verify in test
