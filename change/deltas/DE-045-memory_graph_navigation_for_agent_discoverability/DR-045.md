# DR-045 — Memory graph navigation for agent discoverability

## 1. Executive Summary

- **Delta**: [DE-045](./DE-045.md)
- **Status**: draft
- **Synopsis**: Add runtime graph navigation (backlinks, depth expansion, body-only output) to the memory CLI, building on existing pure link infrastructure and the shared `emit_artifact` pattern.

## 2. Problem & Constraints

- **Current Behaviour**: Memory graph is forward-only. Agents can follow `[[links]]` from a memory but cannot discover what links *to* it, nor expand a link neighbourhood without sequential `show` calls.
- **Drivers**: ISSUE-035, agent onboarding friction, token waste during bulk exploration.
- **Constraints**: ADR-002 — no backlinks in frontmatter. All reverse-edge computation must be runtime.

## 3. Architecture Intent

- **Target Outcomes**:
  1. `compute_backlinks()` — pure function returning `{target_id: [source_ids]}` by parsing all memory bodies
  2. `expand_link_graph()` — BFS from a root node to depth N, returning `list[LinkGraphNode]` with cycle protection
  3. `emit_artifact` extended with `body_only` mode — available to all show commands

- **Guiding Principles**:
  - Pure functions in domain layer, thin CLI wiring
  - Reuse `parse_links()` (existing) as the shared parser
  - `emit_artifact` is the single output dispatcher — don't create parallel paths

## 7. Design Decisions & Trade-offs

- **DEC-045-01**: Parse bodies at query time — no cache for v1. Pure function isolation means cache slots in later without API change.
- **DEC-045-02**: Compact table default for `--links-depth`, `--tree` as option. Agents prefer scannable tabular output; tree conveys structure for humans.
- **DEC-045-03**: Migrate `show_memory` to `emit_artifact` first, then add `--body-only` to the shared helper. One-time consistency cost, permanent reuse gain across all show commands.
